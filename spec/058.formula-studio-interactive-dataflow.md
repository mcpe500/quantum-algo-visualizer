## Status: COMPLETED

**Project:** quantum-algo-visualizer  
**Area:** Formula Studio (`/formulas`)  
**Scope:** Membuat skenario Studio menjadi flow data interaktif, bukan sekadar canvas statis satu arah.

---

## 1. Prompt

User menegaskan bahwa implementasi skenario sebelumnya belum menangkap maksud utama. Kebutuhan sebenarnya bukan hanya menambahkan daftar skenario di Studio, melainkan membuat skenario tersebut memiliki **flow data yang terlihat**, interaktif, detail, dan mudah dipahami manusia. Skenario harus terasa hidup di tab **Studio**, bukan hanya kumpulan node yang diletakkan satu arah.

---

## 2. Tujuan

Tujuan perubahan adalah meningkatkan Formula Studio agar skenario yang dimuat memiliki alur data yang dapat diikuti langkah demi langkah. Pengguna harus dapat melihat node mana yang sedang aktif, koneksi mana yang sedang mengalirkan nilai, nilai apa yang dibawa oleh input, serta bagaimana rumus menghasilkan keluaran numerik. Semua ini tetap berada di tab **Studio**.

---

## 3. Masalah Sebelumnya

Implementasi sebelumnya hanya memuat skenario ke canvas. Walaupun benar secara lokasi karena berada di Studio, pengalaman pengguna masih terasa statis. Node dan koneksi sudah muncul, tetapi tidak ada pemandu flow, tidak ada highlight per langkah, tidak ada animasi arah aliran, dan hasil formula tidak cukup terlihat pada node formula. Akibatnya, pengguna harus menafsirkan sendiri hubungan antar node, sehingga fitur belum terasa sebagai data flow interaktif.

- `engine/evaluator.ts` menyediakan evaluator ekspresi simbolik.

---

## 5. Perubahan Logis

Perubahan logis yang diterapkan:

1. Setiap skenario kini memiliki `flowSteps`, bukan hanya `CanvasState`.
2. Setiap flow step menyimpan:
   - judul langkah,
   - deskripsi manusiawi,
   - daftar node aktif,
   - daftar koneksi aktif.
3. Studio menyimpan skenario aktif dan indeks flow aktif.
4. Flow aktif menyorot node dan koneksi yang relevan.
5. Koneksi aktif memiliki animasi titik bergerak di sepanjang garis agar arah data terlihat.
6. Label koneksi data menampilkan nilai dari node sumber, misalnya `n = 4`.
7. Formula node menampilkan hasil komputasi atau error kecil langsung di kartu node.
8. Canvas memakai `computeDataflowGraph()` sehingga hasil lebih terkait dengan koneksi, bukan hanya scope global.
9. Connection port ditambahkan lewat `toPort` agar edge input dapat membawa nilai ke parameter formula yang benar.
10. Evaluator ditambah fungsi `mod()` agar skenario oracle Deutsch-Jozsa dapat dihitung tanpa error fungsi tidak dikenal.

## 6. Pseudocode

```text
loadScenario(scenarioId):
  scenario = find scenario by id
  canvas = clone scenario.state
  dispatch LOAD_CANVAS(canvas)
  activeScenarioId = scenarioId
  activeFlowStepIndex = 0
  connectionMode = idle

render Studio:
  varScope = buildVarScope(nodes)
  dataflowGraph = computeDataflowGraph(nodes, connections, varScope)
  activeStep = scenario.flowSteps[activeFlowStepIndex]
  activeNodeIds = Set(activeStep.nodeIds)
  activeConnectionIds = Set(activeStep.connectionIds)

render node:
  if node.id in activeNodeIds:
    show cyan ring and glow
  if result available:
    show result badge

render connection:
  if connection.id in activeConnectionIds:
    draw cyan stroke
    animate moving dot along bezier path
  if connection carries data:
    show label with source value

FlowPanel:
  show scenario title and description
  show current step title and explanation
  allow prev / next / direct step click
```

## 8. Data Flow Scenario Coverage

### 8.1 Deutsch-Jozsa

Flow menampilkan input `n`, `x`, `y`, `f`, dan `t`, lalu mengarahkan pengguna ke ancilla, oracle unitary, phase kickback, dan batas query klasik. Nilai `f` membuat hasil phase kickback terlihat secara numerik.

### 8.2 QFT

Flow menampilkan transisi DFT ke QFT, input indeks `j`, `k`, dan `n`, pembentukan twiddle factor, pemetaan ke controlled rotation, dan perhitungan jumlah gerbang.

### 8.3 VQE

Flow menampilkan sumber masalah energi, koefisien Hamiltonian, ekspektasi Pauli, perhitungan energi variational, target ground state, dan baseline FCI.

### 8.4 QAOA

Flow menampilkan spin graf `zi` dan `zj`, cost Max-Cut, cost Hamiltonian, mixer, state QAOA, probabilitas acceptance, dan nilai edge numerik.
