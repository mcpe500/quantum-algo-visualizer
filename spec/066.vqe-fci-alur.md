# SPEC 066: VQE-FCI Computational Trace

## Prompt

Implementasikan tracing VQE-FCI di `quantum-algo-visualizer` dengan gaya penelusuran komputasi seperti dokumen `vqe-fci-computational-trace.md`: alur harus memperlihatkan input molekul, konstruksi ruang FCI, Hamiltonian, diagonalization, konfigurasi VQE, ansatz, expectation value, optimasi, evaluasi shots, dan perbandingan FCI vs VQE.

## Purpose

Menambahkan trace komputasi yang lebih akademik dan rinci untuk VQE. Trace lama hanya menunjukkan urutan gate ansatz, sedangkan kebutuhan sekarang adalah trace konseptual dan numerik dari awal sampai akhir, termasuk sisi klasik FCI dan sisi kuantum VQE.

## Why

1. VQE adalah algoritma hybrid, sehingga tracing harus menunjukkan dua jalur: FCI sebagai referensi klasik dan VQE sebagai proses variational quantum.
2. Dokumen sumber berisi trace naratif langkah demi langkah; aplikasi perlu menampilkan bentuk serupa agar reviewer dapat membaca alur perhitungan, bukan hanya melihat grafik konvergensi.
3. Trace gate ansatz tetap berguna, tetapi belum menjawab bagaimana input H₂/STO-3G berubah menjadi energi ground state.
4. Nilai numerik harus tetap konsisten dengan runtime project, bukan menyalin nilai statis dari dokumen sumber jika berbeda dengan canonical Hamiltonian aktif.

## Source Conflict / Resolution

Dokumen sumber memakai angka Hamiltonian dan energi referensi tertentu, sedangkan project saat ini memakai canonical Hamiltonian aktif:

- VQE-01: 2-qubit Z₂ tapered Hamiltonian dengan energi FCI runtime sekitar `-1.915284 Ha`.
- VQE-02: 4-qubit Jordan-Wigner canonical runtime dengan energi FCI sekitar `-0.539423 Ha`.

Keputusan implementasi: struktur trace mengikuti dokumen sumber, tetapi seluruh nilai energi, Pauli terms, matrix size, eigenvalue preview, error, accuracy, dan shot result diambil dari hasil benchmark runtime. Ini menjaga trace tetap sinkron dengan aplikasi.

## Implemented Changes

### 1. Backend Computational Trace Builder

Fungsi `_build_vqe_fci_computational_trace(...)` dibuat untuk menyusun payload trace berbasis runtime benchmark. Payload dibagi menjadi tiga bagian besar:

1. `fci.steps` — trace FCI:
   - Membaca molekul dan basis set.
   - Membangun orbital molekul.
   - Menyusun determinan Slater.
   - Membangun Hamiltonian qubit sebagai Pauli sum.
   - Melakukan diagonalization.
   - Menetapkan FCI sebagai referensi klasik.

2. `vqe.steps` — trace VQE:
   - Parsing konfigurasi VQE.
   - Mapping Hamiltonian ke Pauli sum.
   - Membangun ansatz parameterized circuit.
   - Menghitung expectation value.
   - Optimasi parameter COBYLA.
   - Evaluasi shot-based.
   - Membandingkan VQE terhadap FCI.

3. `iteration_trace` — checkpoint optimizer:
   - Dipilih dari history evaluasi VQE menggunakan checkpoint merata.
   - Berisi iteration, energy, dan parameter vector.

### 2. Runtime-Numeric Policy

Trace tidak memakai angka statis dari dokumen sumber. Semua nilai berikut berasal dari runtime:

- Pauli terms canonical.
- Matrix preview dari `SparsePauliOp.to_matrix()`.
- Eigenvalue preview dari `np.linalg.eigvalsh()`.
- `E_FCI`.
- `E_VQE`.
- Error dan accuracy.
- Shot-based energy, standard deviation, dan actual shots jika estimator berhasil.

### 3. API Route

Route baru:

```text
GET /api/vqe/computational-trace/<case_id>?shots=1024
```

Route ini mengembalikan computational trace penuh. Karena trace VQE memerlukan hasil optimasi, route ini menjalankan pipeline benchmark untuk kasus terkait.

### 4. Frontend Trace Panel

Komponen `VQEComputationalTracePanel` menampilkan:

- Header trace dan numerical policy.
- Strip metrik perbandingan: `E_FCI`, `E_VQE`, error, accuracy, matrix size, dan jumlah Pauli terms.
- Dua kolom utama:
  - FCI (biru).
  - VQE (ungu).
- Setiap step menampilkan:
  - Nomor step.
  - Phase badge.
  - Ringkasan.
  - Formula.
  - Kalkulasi baris demi baris.
  - Value pills.
  - Pauli terms jika relevan.
  - Matrix preview jika relevan.
  - Result box.
- Tabel checkpoint optimizer.

### 5. Existing Trace Preserved

Trace ansatz lama dari `GET /api/vqe/trace/<case_id>` tetap digunakan dan tetap tampil sebagai `Ansatz Circuit Trace`. Implementasi baru bersifat additive dan tidak memutus fitur lama.

## Logic / Pseudocode

### Backend

```text
run_vqe_payload(case_id, shots):
  load canonical case
  load raw case
  build SparsePauliOp from canonical Hamiltonian

  FCI:
    matrix = H.to_matrix()
    eigenvalues = eigvalsh(matrix)
    fci_energy = eigenvalues[0]

  VQE:
    run statevector optimization
    collect history: [{params, energy}, ...]
    generate iteration snapshots
    evaluate optimal params with BackendEstimatorV2 if available

  computational_trace = build_trace(
    raw_case,
    canonical_case,
    matrix,
    eigenvalues,
    fci_energy,
    vqe_result,
    shot_eval
  )

  return benchmark payload + computational_trace
```

### Trace Builder

```text
build_trace(...):
  extract molecule spec: H2, R, basis, charge, multiplicity
  compute electron_count = proton_count - charge
  compute spin_orbitals = 2 * spatial_orbitals
  compute determinant_count = C(spin_orbitals, electron_count)
  build FCI steps
  build VQE steps
  select optimizer checkpoints from VQE history
  return structured payload
```

### Frontend

```text
VQEQuantumTab:
  if result.computational_trace exists:
    render VQEComputationalTracePanel
  keep old TraceTable for ansatz gate trace
```
