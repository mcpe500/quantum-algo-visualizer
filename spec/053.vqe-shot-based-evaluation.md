# SPEC 053: VQE Shot-Based Evaluation

## Prompt

Implementasi shot-based measurement simulation untuk VQE menggunakan BackendEstimatorV2 (Qiskit 2.x). Parameter `shots` yang sebelumnya dead code kini digunakan untuk evaluasi energi pada parameter optimal, sesuai proposal line 261 ("mengukur energinya") dan line 353-354 (1024 shots untuk VQE/QAOA). Optimasi tetap menggunakan Statevector (eksak) untuk konvergensi yang stabil.

## Purpose

Menutup gap antara "VQE menggunakan Statevector eksak" dan "proposal mensyaratkan pengukuran dengan shots". Pendekatan hierarchical: Statevector untuk optimasi (analisis time complexity yang bersih), BackendEstimatorV2 untuk evaluasi akhir (simulasi quantum hardware).

## Why

1. Proposal line 261 secara eksplisit menyebut "mengukur energinya" — implies shot-based measurement.
2. Proposal line 353-354 menetapkan 1024 shots untuk eksperimen VQE/QAOA.
3. Parameter `shots` sudah diterima dari API route tetapi tidak digunakan di computation (dead code).
4. Tanpa shot evaluation, VQE tidak merepresentasikan bagaimana algoritma bekerja di quantum hardware nyata.
5. Shot-based evaluation menunjukkan perbedaan antara simulasi ideal dan pengukuran stochastic — penting untuk pembahasan akademik.

## Design Decisions

### 1. Hierarchical Framework: Statevector + Shots

**Optimasi (Statevector)**:
- Konvergensi deterministik, tidak terpengaruh shot noise
- Bersih untuk analisis time complexity (O(n_params * iter))
- Gradient-based optimizer (COBYLA) bekerja optimal tanpa noise

**Evaluasi Akhir (BackendEstimatorV2)**:
- Menggunakan parameter optimal dari Statevector optimization
- Mensimulasikan pengukuran pada quantum hardware
- Menghasilkan energy, standard deviation, dan shot count
- Proposal-compliant: "mengukur energinya dengan N shots"

### 2. BackendEstimatorV2 API (Qiskit 2.x)

```text
AerSimulator → BackendEstimatorV2
PUB = (circuit, observable, parameter_values, precision)
precision = 1 / sqrt(shots) → actual shots = N
Result: .data.evs (energy), .data.stds (std), .metadata['shots']
```

BackendEstimatorV2 menangani Pauli basis rotation secara internal — tidak perlu manual H/Rx gates untuk X/Y terms.

### 3. Shots Configurable via UI

Dropdown selector: 1024 (default), 2048, 4096, 8192. Default sesuai proposal.

### 4. extraControls Pattern

Prop baru `extraControls?: ReactNode` pada `AlgorithmPageShell` memungkinkan setiap halaman menambah kontrol khusus tanpa memodifikasi shared layout secara invasive. VAE menggunakannya untuk shots selector. Halaman lain (DJ, QFT, QAOA) tidak terdampak.

## Logic / Pseudocode

### Backend Shot Evaluation

```text
setelah VQE optimization selesai:
  qc_eval, params_eval = build_ansatz_no_measure(n_qubits, ansatz_type, n_layers)
  backend = AerSimulator()
  estimator = BackendEstimatorV2(backend=backend)
  precision = 1.0 / sqrt(shots)
  pub = (qc_eval, hamiltonian_op, optimal_parameters, precision)
  job = estimator.run([pub])
  result = job.result()
  
  shot_energy = float(result[0].data.evs)
  shot_std = float(result[0].data.stds)
  actual_shots = int(result[0].metadata['shots'])
  shot_error = abs(shot_energy - fci_energy)
  
  return shot_evaluation ke quantum section
```

### Frontend Data Flow

```text
user pilih shots via dropdown → setShots(state)
user klik "Jalankan" → handleRun() mengirim { case_id, shots } ke backend
backend: Statevector optimization → BackendEstimatorV2 evaluation → response
frontend: VQEQuantumTab render Shot-Based Evaluation section
```

## Notes

- BackendEstimatorV2 menggunakan precision = 1/sqrt(shots) untuk mengontrol jumlah shots. Actual shots mungkin sedikit berbeda dari requested, tetapi metadata menunjukkan jumlah aktual.
- Silent exception pada shot evaluation by design — jika BackendEstimatorV2 gagal, VQE tetap mengembalikan semua data eksak. Frontend hanya menampilkan shot section jika data tersedia.
- ConvergenceChart tidak diubah — kurva konvergensi tetap dari Statevector (eksak). Shot evaluation hanya single-point evaluation di parameter optimal.
- extraControls pattern memungkinkan halaman lain (DJ, QFT, QAOA) menambah kontrol serupa tanpa perubahan ke AlgorithmPageShell.
