# 028. DJ Right Panel Redesign — OracleConstructionPanel Enhancement + Panel Consolidation

## Context

User memberikan redesign concept untuk DJ quantum animation page. Redesign menunjukkan:
- Header compact dengan timeline pills di水平 row
- Active Column info sebagai sleek bar而非 separate card
- Left column: visualizer + controls
- Right column: fixed 380px dengan 4 panels only

User also feedback bahwa:
1. OracleConstructionPanel penjelasannya 'kurang' — perlu richer content dan adaptif ke active step
2. Panel di sisi kanan 'banyak banget sampe ada empty space' — perlu konsolidasi dari 6 ke 4 panels

## Analisis Masalah

### OracleConstructionPanel — 'kurang' penjelasannya

**Current implementation problems:**

1. `buildOracleConstructionModel` hanya return 4 generic steps yang sama untuk semua balanced case
2. Tidak ada visualisasi gate sequence (X→MCX→X)
3. Tidak ada breakdown sub-phase (init → flip → MCX → restore)
4. Tidak adaptif terhadap `activeBits` dari step animasi aktif
5. Row content terlalu text-heavy tanpa structure visual

**Redesigned approach:**

1. Enhanced `OracleConstructionModel` dengan:
   - `overviewBlock`: ringkasan cepat dengan gate count
   - `subSteps` per row: 4-phase breakdown (init/flip/mcx/restore)
   - `gateSequence` label: visual notation seperti `[X q0,q1] → [MCX] → [X q0,q1]`
   - `isActive` per row untuk highlight dinamis

2. Panel UI berubah:
   - Compacter styling matching redesign concept
   - Gate sequence displayed sebagai inline badge
   - When active: show 4-column sub-step grid (init/flip/mcx/restore)
   - Zero-controls displayed sebagai compact pills

### Right-side panel consolidation — 'banyak banget'

**Before:** 6 panels
- ReadingGuideCard
- StateSummaryPanel
- TruthTablePanel
- OracleConstructionPanel
- FinalAmplitudePanel
- MeasurementPanel

**After:** 4 panels matching redesign concept
- ReadingGuideCard (compact)
- TruthTablePanel (compact table style)
- OracleConstructionPanel (enhanced)
- MeasurementPanel (merged amplitude data + measurement)

**Changes:**
- StateSummaryPanel: removed, terlalu redundant dengan info di ReadingGuideCard
- FinalAmplitudePanel: content merged ke MeasurementPanel
- Panel order: ReadingGuide → TruthTable → OracleConstruction → Measurement

## Implementation Detail

### A. oracle-construction.ts — Enhanced Model

New interfaces:
```typescript
interface OracleGateBlock {
  inputBits: string;
  zeroControls: string[];
  flipQubits: string[];
  mcxControls: string[];
  restoreQubits: string[];
  isActive: boolean;
}

interface OracleConstructionRow {
  inputBits: string;
  stepNumber: number;
  intro: string;
  subSteps: {
    phase: 'init' | 'flip' | 'mcx' | 'restore';
    label: string;
    detail: string;
    gate: string;
    affected: string;
  }[];
  gateSequence: string;
  zeroControls: string[];
}

interface OracleConstructionModel {
  profile: 'constant-zero' | 'constant-one' | 'balanced';
  intro: string;
  overviewBlock: {
    label: string;
    content: string;
    gateCount: number;
  } | null;
  steps: string[];
  gateSummary: string;
  rows: OracleConstructionRow[];
  activeBits: string | null;
}
```

Key changes:
- `buildBalancedRows`: now builds `subSteps` array with 4 phases per row
- `buildOracleConstructionModel`: now accepts `activeBits` parameter, sets `isActive` on matching row
- Gate sequence computed as `[X flipQubits] → [MCX] → [X restoreQubits]` or `[MCX]` if no flips
