# SPEC 050: QAOA Adjacency-Matrix-Only JSON Schema (Matrix-Centric)

## Prompt

JSON dataset QAOA diubah menjadi HANYA berisi `adjacency_matrix` (tanpa `nodes` dan `edges`). Semua komponen yang mengonsumsi data harus diupdate: backend harus derive nodes/edges dari matrix, frontend components harus matrix-native, dan validasi strict diterapkan.

## Purpose

Menjadikan `adjacency_matrix` sebagai **single source of truth** untuk data graph QAOA. Nodes dan edges di-derive dari matrix, bukan didefinisikan terpisah.

## Why

1. JSON lebih bersih - hanya satu representasi graph
2. Tidak ada inkonsistensi antara nodes, edges, dan matrix
3. Validasi strict memastikan data selalu valid
4. Matrix-native components lebih resilient

## Codebase

**Backend:** `backend/services/qaoa_service.py`, `backend/api/routes/qaoa.py`
**Frontend Types:** `frontend/src/types/qaoa.ts`
**Frontend Utils:** `frontend/src/utils/qaoaGraph.ts` (baru)
**Frontend Components:** `AdjacencyMatrixGrid.tsx`, `MiniGraph.tsx`, `QAOADatasetVisualizer.tsx`

## Changes Implemented

### 1. Backend - Strict Validation
- `_validate_adjacency_matrix()`: Square, symmetric, diagonal=0, values 0/1
- `_normalize_problem()`: Matrix-centric, derives nodes/edges/edges_weighted from matrix
- `enrich_case_graph()`: Injects derived nodes/edges ke API response untuk backward compatibility

### 2. Backend - Routes
- `qaoa_cases()`: Enrich setiap case dengan derived nodes/edges
- `qaoa_dataset()`: Enrich single case

### 3. Frontend Utils
- `deriveNodesFromMatrix()`: `[0, 1, ..., N-1]`
- `deriveEdgesFromMatrix()`: Pairs where `matrix[i][j] === 1`
- `deriveGraphFromMatrix()`: Combines both

### 4. Frontend Types
- `QAOAGraph`: `adjacency_matrix` required, `nodes`/`edges` optional

### 5. Frontend Components (Matrix-Native)
- `AdjacencyMatrixGrid`: Hanya accept `adjacencyMatrix` prop
- `MiniGraph`: Hanya accept `adjacencyMatrix` prop, derive nodes/edges internally
- `QAOADatasetVisualizer`: Derive dari matrix, pass ke children
