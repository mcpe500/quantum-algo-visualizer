# SPEC 007: Dynamic Trace Table dari truth_table JSON

## Prompt

Trace table harus lahir dari JSON case aktif secara nyata. Bukan 5/7 baris tetap. Panjang tabel harus sesuai dengan kompleksitas oracle pada sirkuit.
Kalau truth_table berubah, trace harus berubah. Tambah JSON baru tanpa ubah kode manual.

## Purpose

Trace table yang benar-benar dinamis: jumlah baris menyesuaikan strukturgate aktual. Ini melengkapi phase-boxed circuit image,
di mana gambar sudah actual, tetapi tabel sebelumnya masih ringkasan tetap.

## Why

1. Gambar sirkuit sudah actual (phase boxes mengikuti struktur gate aktual).
2. Tabel sebelumnya masih 7 baris固定 - bukan actual.
3. Request pengguna: "circuit panjang tapi table pendek, seharusnya tablenya seusi dengan talbenya dong."
4. Setiap JSON baru harus langsung bekerja tanpa ubah kode manual.

## Changes Made

### 1. Backend trace now DYNAMIC

`build_trace_from_circuit(n_qubits, truth_table)` creates actual trace:

- **Init**: 1 row - Inisialisasi awal
- **Prep**: 3 rows - X anc, H inputs, H anc
- **Oracle**: variable rows based on truth_table:
  - constant-zero: 1 row (identity oracle)
  - constant-one: 1 row (X oracle)
  - balanced: 3 rows per input where f(x)=1 (pre-X, MCX, post-X)
- **Interference**: 1 row - H final
- **Measure**: n rows - measurement per qubit

Row count formula:
```
DJ-01 (constant-zero, n=3): 1 + 3 + 1 + 1 + 3 = 9 rows
DJ-02 (balanced, 4 inputs → 1): 1 + 3 + (3×4) + 1 + 3 = 18 rows
DJ-03 (constant, n=4): 1 + 4 + 1 + 1 + 4 = 11 rows
DJ-04 (balanced, n=4): 1 + 4 + (3×6) + 1 + 4 = 24+ rows
```

### 2. Frontend data model updated

`DJTraceStage` new structure:
```typescript
interface DJTraceStage {
  step: number;
  operation: string;
  wire_markers: Record<string, string>;  // q0, q1, ... per step
  ancilla_marker: string;
  phase: 'init' | 'prep' | 'oracle' | 'interference' | 'measure';
}
```

`wire_markers` values: `|0⟩`, `|1⟩`, `|+⟩`, `|−⟩`, `H`, `X`, `ctrl`, `target`, `M`, `I`, or `-`.

### 3. Frontend table updated

`QuantumTraceTable.tsx` now:
- Iterates `stages` directly (no parsing)
- Displays `wire_markers` per column
- Colors rows by `phase` for visual grouping
- Shows step count and "Dynamic dari truth_table" note in header

### 4. Partitions computed dynamically

Partition boundaries now derived from phase labels in stages, not hardcoded indices.

## Logic / Pseudocode

### New trace builder

```python
def build_trace_from_circuit(n_qubits, truth_table):
    stages = []
    anc = n_qubits
    step = 0

    # Init
    append_stage('Inisialisasi', {i: '|0⟩', anc: '|0⟩'}, 'init')

    # Prep
    append_stage('X on anc', {i: '|0⟩', anc: 'X'}, 'prep')
    append_stage('H on inputs', {i: 'H', anc: 'H'}, 'prep')
    append_stage('H on anc', {i: 'H', anc: 'H'}, 'prep')

    # Oracle - ACTUAL per truth_table
    for input_bits where truth_table[input_bits] == 1:
        pre_X = {i: ('X' if input_bits[i]=='0' else '-'), anc: '-'}
        append_stage(f'pre-X for {input_bits}', pre_X, 'oracle')

        mcx = {i: ('ctrl' if input_bits[i]=='1' else 'target' if i==n-1 else '-'), anc: 'target'}
        append_stage(f'MCX for {input_bits}', mcx, 'oracle')

        post_X = {i: ('X' if input_bits[i]=='0' else '-'), anc: '-'}
        append_stage(f'post-X for {input_bits}', post_X, 'oracle')

    # Interference
    append_stage('H final', {i: 'H', anc: '-'}, 'interference')

    # Measure
    for i in range(n_qubits):
        m = {j: ('M' if j==i else '-'), anc: '-'}
        append_stage(f'M on q{i}', m, 'measure')

    return stages
```

### Frontend rendering

stages.map((stage) => (

## Dynamic Behavior

- **DJ-01**: constant-zero → shorter table (9 rows)
- **DJ-02**: balanced with 4 ones → longer table (18 rows)
- **DJ-04**: balanced with 6 ones → even longer table
- Adding new `DJ-05.json` with different `truth_table` → table auto-adjusts

When new JSON added:
```json
{
  "case_id": "DJ-05",
  "n_qubits": 4,
  "expected_classification": "BALANCED",
  "oracle_definition": {
    "truth_table": {"0000": 1, "0001": 1, "0010": 0, ...}
  }
}
```

Backend builds correct row count automatically. No code changes anywhere.
