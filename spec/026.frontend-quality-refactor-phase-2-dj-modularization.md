# SPEC 026: Frontend Quality Refactor Phase 2 (DJ Modularization Integration)

## Prompt

Lanjutkan Phase 2 untuk hotspot utama frontend:

1. Selesaikan modularisasi `DJQuantumAnimation`.
2. Integrasikan modul baru tanpa mengubah behavior yang sudah berjalan.
3. Tetap low-risk, tidak melakukan rewrite logika 3D secara destruktif.
4. Validasi dengan lint dan build.

## Tujuan

1. Menurunkan kompleksitas file `DJQuantumAnimation.tsx` agar lebih maintainable.
2. Memisahkan concern antara:
   - scene primitives 3D,
   - panel UI,
   - narration,
   - helper,
   - video overlay/export rendering.
3. Menjaga semua fitur playback, phase jump, dan export video tetap identik secara behavior.

## Kondisi Awal

Sebelum integrasi Phase 2, `DJQuantumAnimation.tsx` masih monolitik sangat besar:

1. Banyak fungsi utilitas, narasi, scene primitives, dan panel berada dalam satu file.
2. Sulit melakukan review/perubahan lokal tanpa menyentuh area lain.
3. Risiko regresi tinggi saat menambah fitur export atau visual behavior.

## Scope Perubahan

### 1. Integrasi modul `animation/*` ke komponen utama

File:

- `frontend/src/components/dj/DJQuantumAnimation.tsx`

Perubahan:

1. Ganti konstanta lokal menjadi import dari `animation/constants.ts`.
2. Ganti helper lokal menjadi import dari `animation/helpers.ts`.
3. Ganti panel lokal menjadi import dari `animation/panels.tsx`.
4. Ganti scene lokal menjadi import `StoryScene` dari `animation/scene-primitives.tsx`.
5. Hapus blok fungsi lokal yang sudah dipindahkan agar tidak duplikatif.

### 2. Ekstraksi video overlay rendering

File baru:

- `frontend/src/components/dj/animation/video-overlay.ts`

Isi utama:

1. `getSupportedVideoMimeType()`
2. `drawVideoFrame(...)`
3. helper internal `drawLegendPill(...)`

Alasan:

1. Isolasi concern export compositor agar logika UI utama tidak tercampur dengan drawing canvas 2D.
2. Memudahkan pengujian perubahan overlay tanpa menyentuh logic playback.

### 3. Penyesuaian scene module

File:

- `frontend/src/components/dj/animation/scene-primitives.tsx`

Perubahan:

1. Tambahkan `OrbitControls` di `StoryScene` agar parity dengan behavior sebelumnya tetap terjaga.

### 4. Deduplikasi pipeline export WebM/MP4

File:

- `frontend/src/components/dj/DJQuantumAnimation.tsx`

Perubahan:

1. Gabungkan alur export WebM dan MP4 menjadi satu pipeline `runExportPipeline(target)`.
2. Perbedaan output hanya pada tahap akhir:
   - `webm`: langsung download blob recorder.
   - `mp4`: convert via `convertWebmToMp4(...)` lalu download.
3. Pertahankan state handling:
   - `isExporting`
   - `isConverting`
   - restore step/speed/playback setelah export.

## Dampak Arsitektur

1. Boundary modular Phase 2 sekarang aktif dipakai, bukan hanya file placeholder.
2. `DJQuantumAnimation.tsx` berubah menjadi orchestrator playback + rendering shell.
3. Concern heavy rendering dipisah:
   - 3D scene: `scene-primitives.tsx`
   - 2D export overlay: `video-overlay.ts`
   - content panel: `panels.tsx`
   - narration logic: `narration.ts`
   - shared helper/constants: `helpers.ts`, `constants.ts`

## Risiko dan Mitigasi

1. Risiko pergeseran behavior kamera/orbit:
   - mitigasi: `OrbitControls` dipastikan tetap aktif di `StoryScene`.
2. Risiko perbedaan flow WebM vs MP4:
   - mitigasi: disatukan ke pipeline yang sama untuk intro/play/outro timing.
3. Risiko state tidak kembali setelah export:
   - mitigasi: restore `speed`, `currentStep`, `isPlaying`, `isConverting`, `isExporting` di blok `finally`.
