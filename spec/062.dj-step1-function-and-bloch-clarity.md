# SPEC 062: DJ Step 1 Function Label and Bloch Clarity

## Prompt

User requested fixes for all relevant video/image engines:

- Step 1 video/image must display the function.
- Constant case must clearly show constant output.
- Bloch sphere arrow direction must be clearer.
- Bloch sphere internal space can be enlarged.

## Understanding

The issue is in the Deutsch-Jozsa animation/image export path. The oracle function exists in the dataset truth table, but Step 1 did not show the function directly inside the visual. Constant cases therefore required the viewer to infer the function from later oracle/measurement labels. Bloch arrows were also too short and visually blended with the sphere body.

## Implementation Logic

1. Derive a visible function label from existing payload data.
   - `constant-zero` becomes `f(x) = 0 for all x`.
   - `constant-one` becomes `f(x) = 1 for all x`.
   - `balanced` becomes `f(x) = 1 for {...}` based on truth-table rows where output is `1`.

2. Render the function inside the Three.js scene.
   - A `FunctionBoard` appears above the circuit lanes.
   - Step 1 enlarges the board slightly so the first frame directly communicates the function.
   - No dataset JSON is changed.

3. Add the function to exported video narration for Step 1.
   - The exported bottom accent line shows `Function: ...` during the pre-init frame.

4. Improve Bloch arrow readability globally.
   - Arrow head and shaft are larger.
   - Arrow color is darker for contrast.
   - Bloch sphere radius is slightly larger and more transparent.
   - Equator ring is enlarged so the arrow has more visible internal space.
