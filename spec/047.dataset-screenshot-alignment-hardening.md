# SPEC 047: Dataset Screenshot Alignment Hardening

## 壹 Task

Perbaiki mismatch alignment pada hasil PNG fitur `take picture` untuk halaman dataset visualization DJ, QFT, dan QAOA. Fokus bukan menambah tombol lagi, tetapi membuat hasil capture identik dengan layout yang dilihat di browser dan lebih aman dipakai untuk BUKU TA.

## 貳 Goals

- Menghilangkan relayout header saat tombol kamera tidak ikut dirender pada screenshot.
- Menstabilkan pipeline `html2canvas` agar font dan layout sudah settle sebelum capture.
- Memperkuat centering angka, label, dan elemen numerik yang sebelumnya sensitif terhadap perbedaan metrik font saat screenshot.
- Menutup gap dokumentasi dari sesi sebelumnya dengan spec dan handoff final yang mencerminkan implementasi aktual.

## 叁 Why

Masalah utama bukan lagi ketiadaan tombol `take picture`, karena tombol tersebut sudah ada dari sesi sebelumnya. Masalah sebenarnya muncul pada output PNG yang dipakai di `bab4/images-context.md`: beberapa elemen tampak aman di web, tetapi bergeser atau terasa tidak benar-benar center setelah ditangkap oleh `html2canvas`. Ini paling berisiko pada komponen yang memakai layout fleksibel di header, font web eksternal, serta centering teks numerik yang bergantung pada baseline default browser.

## 肆 Root Cause yang Divalidasi

1. **Layout berubah saat capture**
   - Tombol kamera sebelumnya dihapus dari DOM ketika `isCapturing = true`.
   - Pada header yang memakai `flex justify-between`, penghapusan child memicu relayout sehingga PNG tidak lagi identik dengan tampilan live.

2. **Font dan paint belum tentu settle saat `html2canvas` berjalan**
   - Font proyek dimuat dari Google Fonts (`Inter`, `JetBrains Mono`).
   - `downloadElementAsPNG()` sebelumnya langsung menangkap elemen tanpa menunggu `document.fonts.ready` dan tanpa penenangan frame render.

3. **Centering teks numerik masih rapuh**
   - Beberapa angka masih mengandalkan baseline default atau offset manual.
   - `MiniGraph` QAOA memakai `y + 4` pada SVG text, yang mudah berbeda saat screenshot.
   - Sel matrix dan kartu DJ/QAOA/QFT belum menegaskan `leading-none`, `text-center`, dan `tabular-nums` secara konsisten.

4. **Dokumentasi sesi tertinggal dari keadaan kode terbaru**
   - Sumber yang dirujuk user masih berhenti pada fase riset tombol `take picture`.
   - Implementasi nyata sudah bergerak lebih jauh, sehingga perlu spec/handoff baru yang sesuai.

## 伍 Implementasi

### 1. Hardening Pipeline Screenshot

File: `frontend/src/utils/download.ts`

Perubahan:
- Tambah `waitForCaptureReady()`.
- Menunggu `document.fonts.ready` sebelum capture.
- Menunggu dua frame `requestAnimationFrame` agar layout dan paint benar-benar settle.
- Menjalankan `html2canvas` dengan `logging: false`.
- Memakai `onclone` untuk mematikan animation dan transition pada subtree target selama clone render.

Efek:
- Capture tidak lagi terlalu cepat terhadap web font.
- Risiko baseline shift atau state transisi setengah jalan berkurang.

### 2. Tombol Kamera Tidak Lagi Mengubah Layout Capture

Files:
- `frontend/src/components/dj/OracleTopography.tsx`
- `frontend/src/components/qft/QFTSignalTopology.tsx`
- `frontend/src/components/qaoa/QAOADatasetVisualizer.tsx`

Perubahan:
- Handler screenshot diubah menjadi async dan langsung `await downloadElementAsPNG(...)`.
- Tombol tidak lagi dihapus dari DOM saat capture.
- Tombol diberi `data-html2canvas-ignore="true"` agar tidak ikut dirender oleh `html2canvas`.
- Tombol cukup di-`disabled` sementara selama cooldown 10 detik.
- Setiap root kartu diberi marker `data-capture-root="true"` agar subtree capture mudah dikenali.

Efek:
- Struktur layout header tetap sama sebelum dan saat capture.
- PNG tidak membawa ikon kamera, tetapi komposisi layout tetap stabil.

### 3. Hardening Alignment Teks dan Angka

#### DJ Oracle Topography

File: `frontend/src/components/dj/OracleTopography.tsx`

Perubahan:
- `TruthTableCell` memakai `inline-flex`, `w-full`, `items-center`, `justify-center`, `text-center`, `leading-none`, dan `tabular-nums`.
- Marker kecil `01/10` di header ditegaskan `items-center`, `text-center`, dan `tabular-nums`.

Tujuan:
- Angka input/output pada truth table lebih konsisten center horizontal dan vertical di hasil screenshot.

#### QFT Signal Topology

File: `frontend/src/components/qft/QFTSignalTopology.tsx`

Perubahan:
- Kontrol kanan header digabung ke grup tetap agar komposisi lebih stabil.
- Label signal dan badge `Data Points` diberi `leading-none`.
- Badge `Data Points` juga diberi `tabular-nums` dan `inline-flex` center.

Tujuan:
- Header dan footer QFT tidak lagi berubah proporsi saat capture.

#### QAOA Dataset Visualizer

Files:
- `frontend/src/components/qaoa/QAOADatasetVisualizer.tsx`
- `frontend/src/components/qaoa/AdjacencyMatrixGrid.tsx`
- `frontend/src/components/qaoa/MiniGraph.tsx`

Perubahan:
- Badge header QAOA diubah menjadi slot berukuran tetap melalui helper `HeaderBadge` berbasis `inline-grid h-9 place-items-center`, bukan lagi `span` inline dengan padding biasa.
- Isi badge diberi `translate-y-[-0.02em]` agar baseline huruf kapital tidak turun di hasil capture.
- Metrik `edges/nodes` di header kanan diubah ke slot `h-9` dengan `items-center` eksplisit.
- Header matrix, row label, dan cell value pada adjacency matrix dibuat center dengan `grid place-items-center` + offset kecil untuk baseline.
- Fragment row matrix diberi key stabil melalui wrapper `className="contents"`.
- Label node pada `MiniGraph` diubah dari offset manual `y + 4` menjadi `dy="0.35em"` yang lebih stabil untuk rendering SVG di screenshot.
- Label `graph` dan deskripsi footer juga dipindah ke slot tinggi tetap dengan centering eksplisit.

Tujuan:
- Angka node graph dan isi matrix tidak lagi bergantung pada baseline manual yang rapuh.
