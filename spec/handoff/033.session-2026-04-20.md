# Handoff 033: Circuit Lab Drag-and-Drop Tab

**Date:** 2026-04-20
**Status:** COMPLETE

## Summary

Mode baru `Circuit Lab` sudah ditambahkan ke halaman `/playground` sebagai tab terpisah dari mode state-level. User sekarang bisa menyusun sirkuit kuantum dengan drag-and-drop, melihat preview Bloch/statevector secara live, membaca trace per langkah, dan memperoleh insight otomatis seperti superposition, phase rotation, entanglement, dan heuristic phase kickback.

## What Was Done

### 1. Added new reusable preview component

**`frontend/src/components/qubit-playground/QubitStatePreview.tsx`**

- Ekstraksi preview Bloch + `StatevectorBars` dari `QubitPlaygroundPage`
- Dipakai ulang oleh `State Lab` dan `Circuit Lab`
- Render 1 canvas per qubit card untuk menghindari overlap/clipping

### 2. Added new circuit builder hook

**`frontend/src/components/qubit-playground/useCircuitBuilder.ts`**

- Menyimpan placement gate per kolom/row
- Mendukung qubit 1..3 dan kolom 4..10
- Replay rangkaian dari `|00...0⟩`
- Derive final `statevector`, `blochData`, `blochCards`, `trace`, `insights`
- Mendukung:
  - `placeGate`
  - `movePlacement`
  - `removePlacementAt`
  - `updateSelectedAngle`
  - `clearCircuit`

### 3. Added new Circuit Lab tab UI

**`frontend/src/components/qubit-playground/CircuitLabTab.tsx`**

- Toolbar:
  - qubit switch
  - column controls
  - clear circuit
- Gate palette grouped by fixed / parametric / two-qubit
- HTML5 drag-and-drop grid
- Selected gate inspector
- Insights panel
- Trace panel
- Final `StateInfoPanel`

### 4. Integrated tabs into playground page

**`frontend/src/pages/QubitPlaygroundPage.tsx`**

- Page title now `Quantum Playground`
- Added segmented tabs:
  - `State Lab`
  - `Circuit Lab`
- Subtitle now changes based on active tab
- Existing state-mode preview now uses `QubitStatePreview`
- `Circuit Lab` state is preserved in page-level hook while switching tabs

## Notes / Constraints

1. Gate dua qubit saat ini selalu memakai row yang dipilih dan row tepat di bawahnya.
2. Insight phase kickback masih berupa heuristic edukatif, bukan analisis formal penuh.
3. Drag-and-drop tidak memakai dependency baru; tetap native browser API.
4. Preview state tetap memakai engine matrix/statevector yang sama dengan mode state-level agar hasil konsisten.
