# Handoff 065: Problem Statement Visualization Input-to-Output Flow

## Summary

Reworked problem statement UI because prior version emphasized dataset metadata instead of explaining the actual problem and solution flow. New design shows one clear visual per algorithm: input problem -> process/algorithm idea -> output solution.

## User Clarification

User clarified the real requirement:

1. Need one visual/gambar that explains what the problem is and what the solution is.
2. Visual must show from input to output.
3. QFT must not be blank and must show output frequency result.
4. VQE must clearly show input and output; output is ground state energy.
5. QAOA must show graph input to Max-Cut partition output, not dataset metadata.

## Implemented Changes

### Shared Shell and Hooks

1. `frontend/src/shared/components/AlgorithmPageShell.tsx`
   - Supports `problem` tab.
   - Problem tab can render before benchmark result exists.
   - Classic/quantum/animation tabs show empty state until run.

2. `frontend/src/hooks/useAlgorithmBenchmark.ts`
   - Stores dataset case objects as `caseData`.
   - Exposes `selectedCaseData` to pages.
   - Problem statements use loaded `/cases` data, no extra calls needed.

3. Algorithm hooks now support `problem` active tab:
   - `frontend/src/hooks/useDJBenchmark.ts`
   - `frontend/src/hooks/useQFT.ts`
   - `frontend/src/hooks/useVQE.ts`
   - `frontend/src/hooks/useQAOA.ts`

### Problem Statement Components

1. `frontend/src/components/dj/DJProblemStatement.tsx`
   - New flow: truth table input -> oracle question -> output classification.
   - Uses actual `oracle_definition.truth_table`, `n_qubits`, and `expected_classification`.

2. `frontend/src/components/qft/QFTProblemStatement.tsx`
   - New flow: signal waveform -> Fourier transform -> frequency spectrum.
   - Uses `signal_data` from JSON.
   - Computes DFT spectrum in frontend if benchmark result is not available.
   - Uses `result.fft.spectrum` and `result.fft.dominant_bins` after run.

3. `frontend/src/components/vqe/VQEProblemStatement.tsx`
   - New flow: molecule + basis -> Hamiltonian + ansatz -> ground state energy.
   - Shows H2 structure, distance, basis, mapping, qubit reduction, ansatz, optimizer.
   - Shows energy target formula before run.
   - Shows VQE and FCI energies after run.

4. `frontend/src/components/qaoa/QAOAProblemStatement.tsx`
   - New flow: graph -> Max-Cut objective -> partition + cut output.
   - Uses adjacency matrix from JSON.
   - Computes exact small Max-Cut in frontend for pre-run visual.
   - Uses `result.exact` after benchmark.

## Show Don't Tell Follow-Up

Applied latest spec update: `show don't tell`.

Final visual behavior:

1. DJ no longer explains with paragraphs. It shows truth table grid -> output balance -> C/B classification.
2. QFT no longer has explanatory paragraphs. It shows waveform `x[t]` -> QFT transform glyph -> spectrum `X[k]`.
3. VQE no longer explains the problem in prose. It shows H-H molecule -> Hamiltonian/ansatz -> scalar energy output.
4. QAOA no longer describes dataset metadata. It shows graph -> group split objective -> partition/cut output.

All long panel text was removed or converted into badges, formulas, charts, or icon labels.
