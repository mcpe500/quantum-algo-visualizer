# Handoff 014 — Session 2026-04-16 (DJ Gate + Story Rework)

## Summary

DJ visualization reworked again after user feedback about overlap and unclear semantics.

Final direction now:

- 3D scene = story only
- Python gate trace = separate clear timeline above scene
- oracle token data = optional and explicitly explained

## User Feedback Addressed

1. Too much overlapping / duplication in scene.
2. Gate sequence from Python was not shown clearly.
3. User did not understand what 0/1 balls meant.

## What Changed

### 1. `DJQuantumAnimation.tsx` fully rewritten

New structure:

- `GateTimeline` (HTML)
- `StoryScene` (R3F)

This avoids mixing all information in one visual layer.

### 2. Gate from Python now visible

Component now requires both:

- `data: DJAnimationPayload`
- `trace: DJQuantumTrace`

`trace.stages` is rendered into grouped stage cards by phase.

Each card shows:

- operation text
- per-wire marker badge
- active phase highlight

### 3. 3D scene simplified

- 5 permanent zones:
  - INPUT
  - SUPERPOSITION
  - ORACLE
  - INTERFERENCE
  - OUTPUT
- orb group moves only in active story zone
- result board separated into OUTPUT zone
- oracle chamber hidden unless phase = oracle

### 4. Token data clarified

- token data default OFF
- toggle button added
- note added:
  - orb = state qubit
  - token = sample oracle input/output data from dataset
