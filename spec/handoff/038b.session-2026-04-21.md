# Handoff 038b: Circuit Lab - CNOT Target Above Control Fix

**Date:** 2026-04-21
**Status:** COMPLETE

## Summary

Fixed remaining issues with CNOT target qubit selection when target is above the control qubit (e.g., q1 controlling q0).

## Issues Fixed

### Issue 1: Default targetRow when placing at bottom row

**Before:** When placing CNOT at q1 with only 2 qubits, target defaulted to `numQubits - 1` = 1, which is the same as control row.

**Fix:** Changed default target logic to use `row - 1` when at the last row:
```typescript
// Before
const actualTargetRow = definition.numQubits === 2
  ? (targetRow ?? (row < numQubits - 1 ? row + 1 : numQubits - 1))
  : undefined;

// After
let actualTargetRow: number | undefined = targetRow;
if (definition.numQubits === 2 && actualTargetRow === undefined) {
  actualTargetRow = row < numQubits - 1 ? row + 1 : Math.max(0, row - 1);
}
```

### Issue 2: Visual line rendered in wrong direction

**Before:** The connection line always drew downward from control and upward from target, assuming target is always below control.

**Fix:** Added dynamic line positioning based on whether target is above or below control:

**Primary cell (control):**
- Target below: line extends downward from center (`top: 50%`)
- Target above: line extends upward from center (`bottom: 50%`)

**Secondary cell (target):**
- Target below (secondary is below): line comes from above (`top: -24px`)
- Target above (secondary is above): line comes from below (`bottom: -24px`)

### Issue 3: Gate tile position

**Before:** Gate tile always positioned at `top-2` of primary cell.

**Fix:** When target is above control, gate tile moves to `bottom-2` so the line goes upward naturally.

### Issue 4: Tooltip text

**Before:** "Need q+1" (outdated assumption that target must be row+1)

**Fix:** "Need 2+ qubits"
