# Handoff 024: DJ Circuit 500 Fix + MP4 Anti-Hang

Date: 2026-04-17
Session focus: stabilisasi endpoint gambar sirkuit DJ dan konversi MP4

## Ringkasan Masalah User

1. Frontend spam error `500` pada:
   - `/api/dj/circuit-image/<case>`
   - `/api/dj/circuit-image-boxed/<case>`
2. Export MP4 tidak selesai (loading terus).

## Perbaikan yang Diterapkan

### A. Backend: circuit image jadi robust

File: `backend/services/dj_service.py`

- Tambah fallback ketika `circuit_drawer(..., output='mpl')` gagal.
- Fallback merender `qc.draw(output='text')` ke figure matplotlib monospace.
- Endpoint tidak lagi wajib 500 hanya karena MPL drawer gagal.
- Jika save figure MPL gagal (`ParseException` saat serialisasi), sistem fallback ulang ke text renderer.
- Payload kini bisa mengandung:
  - `render_mode`: `mpl` / `text-fallback`
  - `warning`: detail fallback/overlay failure

### B. Frontend API: tampilkan error backend asli

File: `frontend/src/services/api.ts`

- Tambah helper `readApiError(response, fallback)`.
- `getCircuitImage()` dan `getCircuitImageBoxed()` sekarang melempar message backend (`error`) jika ada.
- Debug user jadi jauh lebih jelas dibanding pesan generik "not found".

### C. MP4 conversion: fail-fast, tidak endless spinner

File: `frontend/src/utils/videoConvert.ts`

- Tambah `withTimeout(...)` wrapper.
- Timeout load ffmpeg core: 45 detik.
- Timeout exec convert: 120 detik.
- Tambah fallback CDN (jsDelivr -> unpkg).
- Tambah timeout di tahap download core (`toBlobURL`) agar proses tidak menggantung sebelum `ffmpeg.load`.
- Jika load gagal, `ffmpegLoading` di-reset supaya retry berikutnya tidak terkunci.
- Command convert dibuat lebih kompatibel:
  - dari konfigurasi `libx264` yang ketat
  - ke command sederhana `-i input.webm -pix_fmt yuv420p output.mp4`

### D. UI wording

File: `frontend/src/components/dj/DJQuantumAnimation.tsx`

- Message MP4 support disesuaikan agar tidak misleading soal SharedArrayBuffer.

## Catatan Operasional

Jika user masih melihat 500 lama setelah patch:
1. Pastikan backend process direstart.
2. Hard refresh frontend (cache).
3. Cek response body endpoint circuit-image (sekarang harus berisi error detail jika gagal).

Jika MP4 tetap gagal:
1. Cek koneksi ke CDN ffmpeg core.
2. Tunggu timeout message baru (tidak akan spinner tanpa akhir).
3. WebM export tetap bisa dipakai sebagai fallback operasional.
