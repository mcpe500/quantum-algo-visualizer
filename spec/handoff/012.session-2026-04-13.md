# Handoff 012 — Session 2026-04-13

## Ringkasan

Session ini melakukan refactor backend untuk memisahkan route layer, service/domain logic, dan helper shared tanpa mengubah kontrak endpoint yang dipakai frontend.

## Masalah Awal

Struktur backend sebelum refactor terlalu padat di modul `backend/api/*.py`, terutama `backend/api/dj.py`. Satu file memuat:
- route Flask
- akses dataset
- circuit builder
- simulator execution
- trace builder
- pseudocode generator
- plotting helper

Pola serupa juga ada di QFT, VQE, dan QAOA.

## Perubahan yang Dilakukan

### Struktur Baru
- `backend/app.py` → app factory
- `backend/run.py` → entrypoint runtime
- `backend/api/routes/*` → route HTTP tipis
- `backend/services/*` → algorithm/domain logic
- `backend/api/shared/*` → helper presentasi/API reusable

## Keputusan Penting

### 1. Endpoint Tetap
Seluruh path endpoint dipertahankan agar frontend tidak perlu diubah.

### 2. Payload Dipertahankan
Response shape dipertahankan sedekat mungkin dengan implementasi lama.

### 3. Shared Utility Dipisah
Helper trace partition DJ dan plotting phase-box dipindah agar bisa dirawat sendiri.

### 4. Config Disederhanakan
`DEBUG` kini mengikuti env `FLASK_DEBUG`, bukan hardcoded.

## Catatan Implementasi

- Route sekarang hanya handle request/response.
- Service menghasilkan payload dictionary siap `jsonify`.
- Untuk DJ boxed circuit image, helper plotting tetap mempertahankan visual grouping.

## Risiko Lanjutan

- Script eksternal yang import langsung dari `backend/api/dj.py` lama akan rusak dan perlu diarahkan ke service baru.

## Cara Pakai

Untuk local run setelah refactor:
```bash
cd backend
python run.py
```

Untuk integrasi:
1. replace file yang diubah
2. tambahkan file baru
3. hapus modul lama
4. jalankan smoke test frontend ke endpoint existing

## Status

Refactor backend selesai pada level struktur dan pemisahan concern.
