# Handoff 034: Circuit Lab Layout and Playback Refinement

**Date:** 2026-04-20
**Status:** COMPLETE

## Summary

Circuit Lab disempurnakan agar lebih dekat ke pola laboratorium sirkuit yang polished. Session ini menambahkan playback berbasis snapshot langkah, menyinkronkan trace/grid/preview ke langkah aktif, memperjelas gate dua qubit dengan connector visual, dan memperbaiki layout utama yang sebelumnya masih collapse pada screenshot review.

## What Was Done

### 1. Added playback frame model in hook

**`frontend/src/components/qubit-playground/useCircuitBuilder.ts`**

- Added `CircuitPlaybackFrame`
- Added `playbackFrames` derived from simulation
- Playback now includes explicit initial state frame (`step 0`)
- Reused existing trace snapshots instead of duplicating simulation logic in UI

### 2. Reworked Circuit Lab layout and interaction flow

**`frontend/src/components/qubit-playground/CircuitLabTab.tsx`**

- Rebuilt main layout into three deterministic columns:
  - left: gate palette + automated insights
  - center: circuit canvas + playback bar + trace
  - right: live preview + selected gate + current snapshot
- Added playback controls:
  - previous step
  - play / pause
  - next step
  - range slider
  - active step summary
- Trace is now clickable and synchronized with playback
- Grid highlights active placement / active column from playback
- Added control-target style connector for 2-qubit gates (`CNOT`, `CPhase`, `SWAP`)

### 3. Made preview component work in narrow sidebar

**`frontend/src/components/qubit-playground/QubitStatePreview.tsx`**

- Added `compact` mode
- Compact mode forces single-column preview cards and slightly shorter canvas height
- Used by right sidebar in Circuit Lab

### 4. Widened page wrapper for Circuit Lab

**`frontend/src/pages/QubitPlaygroundPage.tsx`**

- Added conditional wrapper width:
  - `State Lab` stays `max-w-[1280px]`
  - `Circuit Lab` becomes `max-w-[1600px]`

This was added after screenshot review showed the earlier layout still too narrow.

## Notes

1. Root issue from screenshot was layout space allocation, not the replay engine itself.
2. Fixed-width side columns plus flexible center column were chosen to keep canvas readable.
3. Playback UI state remains local to `CircuitLabTab`, while circuit simulation state remains in `useCircuitBuilder`.
4. Final state panel still uses `StateInfoPanel` and continues to represent full circuit replay, not the currently scrubbed step.
