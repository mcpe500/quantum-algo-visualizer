## What Was Done

### 1. Critical Bug Fixes

#### Bug 1: Auto-Layout Direction (canvasUtils.ts)
- **Problem:** `forceTo.x -= forceX` always pushed nodes apart regardless of connection direction
- **Fix:** Added directional spring force bias:
  ```typescript
  const horizontalForce = dx * thisSpringConstant * (1 - thisSpringLength / Math.sqrt(dx * dx + dy * dy + 0.01));
  const verticalForce = dy * thisSpringConstant * (1 - thisSpringLength / Math.sqrt(dx * dx + dy * dy + 0.01));
  forceTo.x += horizontalForce;  // Changed from -=
  forceTo.y += verticalForce;
  ```
- **File:** `frontend/src/components/formula-studio/studio/canvasUtils.ts`

#### Bug 2: Connection Reset (StudioCanvas.tsx)
- **Problem:** `setConnectionMode('selecting-source')` was called unconditionally, resetting even failed connections
- **Fix:** Moved reset inside success block:
  ```typescript
  if (success) {
    setConnectionMode('selecting-source');
  }
  ```
- **File:** `frontend/src/components/formula-studio/studio/StudioCanvas.tsx`

#### Bug 3: Formula Node Computation (graphEngine.ts)
- **Problem:** `FORMULA_REGISTRY.find()` and `FORMULA_COMPUTATION_MAP[]` were never used for formula nodes
- **Fix:** Added proper lookup:
  ```typescript
  const formula = FORMULA_REGISTRY.find(f => f.id === node.formulaId);
  const computation = formula?.computation ?? FORMULA_COMPUTATION_MAP[node.formulaId];
  ```
- **File:** `frontend/src/components/formula-studio/studio/graphEngine.ts`

### 2. Dataflow Engine (dataflowEngine.ts)

Created new topological execution engine for Formula Studio canvas connections.

**Key Functions:**
- `detectCycles()` - DFS with color marking (white/gray/black)
- `topologicalSort()` - Kahn's algorithm
- `computeDataflowGraph()` - Main execution engine
- `wouldCreateCycle()` - Cycle prevention for new connections

**Architecture:**
- Lazy initialization for module imports (avoids circular dependencies)
- Validation for computation steps (prevents NaN propagation)
- Cycle extraction fix (removed duplicate neighbor in path)

**File:** `frontend/src/components/formula-studio/studio/dataflowEngine.ts`

### 3. Extended Computation Presets (presets.ts)

Added FORMULA_COMPUTATION_MAP entries for:
- `oracle-unitary` - XOR operation: mod(x+y*f, 2)
- `dj-phase-kickback` - Phase factor: 2*f - 1
- `plus-minus-states` - Amplitude: 1/sqrt(2)
- `qft-twiddle-factor` - Complex phase: exp(-2πijk/2^n)

**File:** `frontend/src/components/formula-studio/computation/presets.ts`

## Key Decisions Made

1. **Dynamic require() pattern:** Used lazy module-level initialization in dataflowEngine.ts to avoid circular dependencies while ensuring require() is only called once per module

2. **Cycle detection correct:** Critic concern about "missing starting node" was incorrect - cycle B→C→B correctly detects [B, C], not [A, B, C] where A is outside the cycle

3. **Computation config approach:** Used symbolic computation (parse → substitute → simplify → evaluate) for all new formulas, which works for scalar/real expressions
