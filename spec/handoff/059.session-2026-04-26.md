# HANDOFF 059 — Formula Studio Scenario Loader in Studio

**Date:** 2026-04-26  
**Status:** COMPLETE  
**Area:** Frontend — Formula Studio

---

## Goal

User requested scenario loading inside Formula Studio **Studio** tab at `/formulas`, not in `Explore` and not only in `Stories`. The implementation adds a Studio toolbar scenario loader that populates the editable canvas with prebuilt algorithm graphs.

---

## What Was Done

### 1. Added Studio scenarios dataset

New folder:

- `frontend/src/components/formula-studio/scenarios/`

New files:

- `data.ts`
- `index.ts`

Implemented `FORMULA_STUDIO_SCENARIOS` with four editable canvas scenarios:

1. `studio-dj-phase-kickback`
   - Deutsch-Jozsa oracle, ancilla state, phase kickback, classical query bound.
2. `studio-qft-circuit-flow`
   - DFT, QFT definition, twiddle factor, controlled rotation, circuit construction, gate count.
3. `studio-vqe-energy-estimation`
   - Eigenvalue equation, Hamiltonian decomposition, variational energy, ground state, FCI comparison.
4. `studio-qaoa-maxcut-flow`
   - Max-Cut cost, cost Hamiltonian, mixer Hamiltonian, QAOA state, acceptance probability.

Each scenario includes:

- formula nodes,
- input nodes,
- expression nodes,
- directed connections,
- pan/zoom canvas state.

---

## 2. Added scenario loader to Studio toolbar

Modified:

- `frontend/src/components/formula-studio/studio/CanvasToolbar.tsx`

Changes:

- added local selected scenario state,
- added dropdown labeled `Pilih skenario Studio...`,
- added `Muat` button,
- exposed props:
  - `scenarios`
  - `onLoadScenario`

The scenario controls are in the Studio toolbar, alongside node creation tools, so user does not need to go to `Stories` or `Explore`.

---

## 3. Connected loader to canvas state

Modified:

- `frontend/src/components/formula-studio/studio/StudioCanvas.tsx`

Changes:

- imports `FORMULA_STUDIO_SCENARIOS`,
- adds `handleLoadScenario(scenarioId)`,
- clones selected scenario state with `JSON.parse(JSON.stringify(...))`,
- dispatches existing reducer action `LOAD_CANVAS`,
- resets connection mode to `idle`,
- clears `connectionSourceId`.

Reason: scenario load replaces current canvas safely and avoids stale connection state.
