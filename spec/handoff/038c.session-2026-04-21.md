# Handoff 038c: Circuit Lab - CNOT Target Direction Fix

**Date:** 2026-04-21
**Status:** COMPLETE

## Summary

Fixed the visual line direction for 2-qubit gates when the target qubit is above or below the control qubit. The direction is now computed relative to the control row, not the current rendering row.

## Change

**File:** `frontend/src/components/qubit-playground/CircuitLabTab.tsx`

**Before:**
```tsx
const targetRow = cellState?.placement.targetRow;
const targetIsAbove = targetRow !== undefined && targetRow < row;
const targetIsBelow = targetRow !== undefined && targetRow > row;
```

**After:**
```tsx
const controlRow = cellState?.placement.row;
const targetRow = cellState?.placement.targetRow;
const targetIsAbove = targetRow !== undefined && controlRow !== undefined && targetRow < controlRow;
const targetIsBelow = targetRow !== undefined && controlRow !== undefined && targetRow > controlRow;
```

## Why This Fixes It

Before, `targetIsAbove` and `targetIsBelow` were computed relative to the current rendering `row`. For the secondary cell (target cell), `row === targetRow`, so both `targetIsAbove` and `targetIsBelow` were `false`. This caused the line to render in the wrong direction (always from bottom upward).

After, the direction is computed relative to the `controlRow` (the primary cell's row). Both primary and secondary cells use the same reference frame, so the line always connects correctly:
- **Target below control** (q0→q1): Primary line goes down from center, secondary line comes from top
- **Target above control** (q1→q0): Primary line goes up from center, secondary line comes from bottom

1. The gate tile position also uses `targetIsAbove`:

2. This fix ensures the visual line connects properly regardless of whether the target is above or below the control qubit.
