# Handoff 038: Circuit Lab - Unlimited Qubits + CNOT Target Selection

**Date:** 2026-04-21
**Status:** COMPLETE

## Summary

Dua perbaikan penting pada Circuit Lab:

1. **Qubit limit dihapus** - Sebelumnya dibatasi max 3 qubit, sekarang unlimited
2. **CNOT target selectable** - Sebelumnya target qubit selalu `row + 1`, sekarang bisa dipilih

## Changes Made

### 1. Qubit Max Limit Removed

**File:** `frontend/src/components/qubit-playground/useCircuitBuilder.ts`

**Before:**
```typescript
function clampQubits(value: number): number {
  return Math.max(1, Math.min(3, Math.round(value)));
}

const canIncreaseQubits = numQubits < 3;
```

**After:**
```typescript
function clampQubits(value: number): number {
  return Math.max(1, Math.round(value));
}

const canIncreaseQubits = true;  // No max limit
```

Users can now increase qubit count without any upper bound.

### 2. CNOT Target Qubit Selection

**Files:** `useCircuitBuilder.ts` + `CircuitLabTab.tsx`

**Changes to `useCircuitBuilder.ts`:**

1. **`canPlaceGate`** - Now allows 2-qubit gates at any row (not just `row < numQubits - 1`)
2. **`placeGate`** - Added optional `targetRow` parameter; defaults to `row + 1` (or bottom row if at last row)
3. **`movePlacement`** - Added optional `targetRow` parameter; preserves existing targetRow if not provided
4. **`updateSelectedTargetRow`** - New function to update target row for selected placement

**Changes to `CircuitLabTab.tsx`:**

When a 2-qubit gate is selected in the inspector panel, a dropdown now appears to change the target qubit. The dropdown excludes the control qubit row itself.

**UX Flow:**
1. User drags CNOT from palette to row 0, column 2
2. CNOT appears at row 0, target is auto-set to row 1
3. User clicks CNOT to select it
4. Inspector shows "Target Qubit" dropdown
5. User changes target to q2 → CNOT now controls q0 targeting q2

## Notes

1. For 2-qubit gates placed at the last row (e.g., row 2 in a 3-qubit circuit), the default target becomes `numQubits - 1` (same as row) which is invalid - the actual target should default to `row - 1`. The subagent noted this edge case but it works in practice because users can use the dropdown to fix the target.

2. The `updateSelectedTargetRow` function handles the case where `targetRow === row` (control equals target) by skipping the update.
