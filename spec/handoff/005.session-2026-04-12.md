# HANDOFF 005: Remove DJ Hardcoding

**Date:** 2026-04-12
**Status:** COMPLETE ✅

---

## Summary

Removed hardcoded DJ case discovery and fixed the oracle path so the backend and UI now follow the dataset folder dynamically. New DJ JSON files are discovered automatically, and the frontend selector no longer depends on a fixed 01-04 list.

## What Was Done

### 1. Backend: dynamic case discovery

- `get_all_cases()` now scans `datasets/dj/` for `DJ-*.json`
- ignores non-case files and generated results
- sorts cases numerically

### 2. Backend: real oracle generation

- `create_dj_circuit()` now builds the oracle from `truth_table`
- constant-zero keeps identity oracle
- constant-one applies `X` on ancilla
- balanced cases apply multi-controlled `X` gates per `f(x)=1`
- barriers stay in the required order

### 3. Backend: counter-based trace

- `build_trace_from_truth_table()` now uses incrementing step numbers
- trace uses the 7-stage DJ flow from the current implementation
- trace remains dataset-driven and not case-specific
- `derive_trace_from_circuit()` was removed

### 4. Frontend: dynamic case selector

- `DJCombinedPage.tsx` now fetches `GET /api/dj/cases`
- select options follow actual dataset cases
- adding `DJ-05.json` makes `DJ-05` appear without code edits

### 5. Safety fixes

- fixed zero-state detection in `run_quantum_dj()`
- made classical stepped runner use safe lookup defaults
