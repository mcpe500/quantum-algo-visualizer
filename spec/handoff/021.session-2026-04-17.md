# Handoff 021: DJ WebM Video Export

Date: 2026-04-17
Session: Add `Export Video Quantum (.webm)` for DJ Three.js animation

## Goal

Menambahkan fitur export video pada animasi Deutsch-Jozsa tanpa mengubah arsitektur backend dan tanpa mencoba merekam seluruh DOM.

## Hasil Utama

1. `DJQuantumAnimation` sekarang bisa mengekspor canvas Three.js menjadi video `.webm`.
2. Video hasil export tidak kosong atau polos, tetapi memiliki caption penjelasan otomatis di atas scene.
3. Halaman DJ kini mengunci kontrol case/tab/run saat export berjalan agar state tidak rusak di tengah recording.

## Keputusan Teknis

### 1. Canvas-only recording

Dipilih `canvas-only recording` karena scene Three.js memang menjadi fokus visual utama. Ini lebih sehat daripada memaksa `html2canvas` mengambil seluruh DOM berkali-kali.

### 2. Overlay penjelasan dirender di compositor

Caption video tidak dirender di panel HTML asli, tetapi digambar ulang di `compositorCanvas`. Keuntungan utama:

1. video tetap berisi penjelasan,
2. UI live tidak perlu diubah khusus untuk mode export,
3. hasil video lebih stabil.

### 3. WebM dulu, MP4 nanti

User membuka opsi `webm -> mp4`, tetapi untuk sesi ini hanya `WebM` yang dikirim. Alasannya sederhana: jalur ini stabil, kecil, dan lolos build tanpa menarik dependensi encoder besar.

## UX yang Sekarang Ada

1. Tombol baru: `Export Video Quantum (.webm)`.
2. Saat export berjalan:
   - tombol berubah ke status loading,
   - play/step/reset/slider/camera toggle dikunci,
   - selector case, tab, run button, dan link navigasi luar dikunci.
3. Jika browser tidak mendukung `MediaRecorder` WebM, komponen menampilkan pesan error yang jelas.

## Hal yang Belum Dikerjakan

1. MP4 conversion.
2. Export seluruh layout quantum tab termasuk panel samping.
3. Quality presets, resolution presets, dan custom duration.

Semua item di atas bisa ditambahkan nanti tanpa membongkar fondasi WebM yang sekarang.

## Verifikasi

### PASS

1. `npm run build`
2. `python -m py_compile backend/services/dj_service.py backend/api/routes/dj.py`

### Catatan Validasi Runtime

1. Build TypeScript lolos.
2. Fitur mengandalkan `MediaRecorder` browser, jadi validasi final tetap perlu dilakukan di browser nyata, terutama Chrome/Edge.

## Risiko yang Masih Ada

1. Browser support untuk WebM tidak seragam. Implementasi sudah memberi fallback error message, tetapi fitur ini tetap paling ideal di Chrome/Edge/Firefox modern.
2. MP4 belum ada, jadi jika user membutuhkan format sangat universal, perlu fase berikutnya.
