## Critic Findings

### Bug Fixed (Critic #2 — Frontend Types)

- **Issue**: `shots` tidak ada di `handleRun` dependency array `useVQE.ts`
- **Impact**: Jika user mengubah shots lalu langsung klik "Jalankan", stale shots value digunakan
- **Fix**: Added `shots` to `[selectedCaseId, shots, loadCircuitImage, loadTrace]`

### Non-blocking Concerns (Critic #1 — Backend)

1. Silent exception swallow (`except Exception: pass`) — by design, tapi production bisa ditambah logging
2. `evs` is ndarray → `float()` works on 0-d/1-d single-element, safe
3. Precision heuristic (1/sqrt(shots)) → actual shots dari metadata, sudah benar

### No Issues (Critic #3 — AlgorithmPageShell)

- `extraControls?: ReactNode` optional → safe untuk semua 4 consumer pages
- `undefined` renders nothing in React → no layout impact

## Design Decisions

1. **Statevector untuk optimasi, BackendEstimatorV2 untuk evaluasi**: Shot noise mengganggu konvergensi COBYLA. Pisahkan concerns: time complexity analysis (Statevector) vs empirical validation (shots).
2. **BackendEstimatorV2 vs manual AerSimulator**: BackendEstimatorV2 menangani Pauli basis rotation secara internal. Manual approach memerlukan H/Rx gates per Pauli term — error-prone dan verbose.
3. **extraControls pattern**: Surgical addition ke shared shell. Tidak memaksa perubahan ke halaman lain.
4. **Conditional rendering**: Shot evaluation section hanya muncul jika data tersedia. Graceful degradation jika BackendEstimatorV2 gagal.
