# HANDOFF 007: Dynamic Trace Table

**Date:** 2026-04-13
**Status:** COMPLETE ✅

---

## Summary

Replaced fixed 7-row trace with DYNAMIC trace that scales with truth_table complexity.
For balanced cases, table now has more rows (proportional to oracle complexity).

## What Was Done

### 1. Backend: dynamic trace generator

Replaced `build_trace_from_truth_table()` with `build_trace_from_circuit()`:
- Init/prep/interference/measure phases stay similar
- Oracle phase now expanded per input where f(x)=1:
  - each balanced input adds 3 rows: pre-X, MCX, post-X
- Row count formula:
  - DJ-01 (constant-zero): 9 rows
  - DJ-02 (balanced, 4 ones): 18 rows
  - DJ-04 (balanced, 6 ones): 24+ rows

### 2. Frontend: data model update

Changed `DJTraceStage` type:
- OLD: `inputs`, `ancilla`, `classical`
- NEW: `wire_markers`, `ancilla_marker`, `phase`

### 3. Frontend: table component rewrite

`QuantumTraceTable.tsx` rewritten:
- Uses `wire_markers` directly (no parsing)
- Colors rows by phase: init=blue, prep=green, oracle=amber, interference=violet, measure=red
- Shows step count in header: "X steps - Dynamic dari truth_table"

### 4. Backend: dynamic partitions

Partition boundaries now computed from phase labels in stages.
