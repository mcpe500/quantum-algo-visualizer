Date: 2026-04-17
Session: Real-case Three.js circuit layout from unified DJ animation timeline

## Goal

Membuat animasi Deutsch-Jozsa terasa benar-benar berasal dari dataset dan circuit nyata, bukan lagi scene template dengan oracle box tetap.

## Ringkasan Hasil

1. Backend animation payload sekarang mengirim timeline langkah nyata, phase partitions, dan oracle summary.
2. Three.js scene sekarang menggambar kolom sirkuit berdasarkan timeline aktual.
3. Dataset JSON dirender penuh di panel samping dan ikut highlight saat oracle aktif.
4. Typography dibesarkan sedikit dan detail dipindah keluar canvas agar tetap rapi.

## Keputusan Arsitektur

### 1. Animation payload jadi source of truth tunggal

Masalah lama: animasi memakai `trace` dan `snapshots` yang tidak sepenuhnya sinkron. Solusi: backend membentuk `timeline` baru yang sudah berisi marker trace dan probabilitas statevector untuk langkah yang sama.

### 2. Layout kolom dinamis, bukan gate landmark tetap

Masalah lama: circuit selalu terlihat seperti punya 5 landmark besar walaupun oracle case berbeda-beda. Solusi: setiap stage timeline diberi satu kolom visual. Posisi X dihitung dari jumlah langkah aktual.

### 3. Detail real-case dipindah ke panel HTML

Masalah lama: jika semua detail dipaksa masuk ke canvas, font kecil dan mudah overlap. Solusi: canvas fokus ke struktur sirkuit dan state orb, sedangkan detail deskriptif, dataset JSON, amplitudo, dan measurement dipindah ke panel HTML.

### 4. Constant case ikut punya oracle visual

Masalah lama: constant case praktis tidak punya oracle stage yang jelas di trace. Solusi: tambahkan `Oracle I` dan `Oracle X(anc)` agar struktur tetap jujur terhadap circuit dan siap untuk dataset constant-one di masa depan.
