## Summary

Created the next three spec docs for the remaining algorithms:
- `spec/008.qft-webapp.md`
- `spec/009.vqe-webapp.md`
- `spec/010.qaoa-webapp.md`

These specs align the next implementation phase with the proposal, the existing DJ architecture, and the JSON datasets already present in the app.

## What Was Done

### 1. QFT spec

Defined the QFT page as a replacement for the `/qft` placeholder, with:
- dataset-driven case discovery
- FFT as the classical baseline
- padding to the next power of two when needed
- circuit depth, gate count, and accuracy tracking
- QFT stage/trace visualization

### 2. VQE spec

Defined the VQE page as a replacement for the `/vqe` placeholder, with:
- dataset-driven ansatz construction from JSON
- FCI as the classical reference
- hybrid optimization loop visualization
- energy error as the main accuracy metric

### 3. QAOA spec

Defined the QAOA page as a replacement for the `/qaoa` placeholder, with:
- graph-to-Hamiltonian mapping from JSON
- Simulated Annealing as the classical baseline
- exact Max-Cut reference for small graphs
- optimizer trace and coupling-map-aware circuit depth
