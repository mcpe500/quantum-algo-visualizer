# Handoff 022: DJ Video Export 1080p Fix

Date: 2026-04-17
Session: Fix blurry DJ video export by moving recording source to dedicated 1080p renderer

## Goal

Menghilangkan masalah kualitas export video DJ yang sebelumnya masih mengikuti ukuran preview di halaman.

## Ringkasan Hasil

1. Export video sekarang memakai hidden export renderer `1920x1080`, bukan preview canvas.
2. Output `.webm` sekarang punya source render yang benar untuk `1080p`.
3. Bitrate export dinaikkan ke `16 Mbps`.
4. Tombol export kini menjelaskan bahwa hasilnya adalah `1080p`.

## File yang Diubah

| File | Perubahan |
|------|-----------|
| `frontend/src/components/dj/DJQuantumAnimation.tsx` | Dedicated export canvas 1080p, wait-for-ready, bitrate bump, UI copy 1080p |
| `spec/021.dj-video-export-1080p.md` | Spec sesi fix resolusi |
| `spec/handoff/022.session-2026-04-17.md` | Handoff sesi ini |

## Keputusan Penting

### 1. Jangan upscale preview

Masalah lama adalah export merekam preview kecil lalu membesarkannya. Itu menghasilkan blur walaupun file berhasil dibuat. Perbaikannya adalah mengganti source render, bukan sekadar membesarkan kompositor atau menaikkan bitrate.

### 2. Hidden export canvas hanya aktif saat export

Canvas 1080p kedua tidak dirender terus-menerus. Ia hanya dipasang saat export aktif, sehingga beban runtime normal halaman tetap wajar.

### 3. Preview canvas tetap ringan

Setelah source export dipindahkan, preview canvas tidak lagi perlu `preserveDrawingBuffer`. Ini menjaga interaksi live tetap lebih ringan.

## Verifikasi

### PASS

1. `npm run build`
2. `python -m py_compile backend/services/dj_service.py backend/api/routes/dj.py`

## Risiko Tersisa

1. Validasi final kualitas visual tetap perlu dicek di browser nyata karena `MediaRecorder` adalah API browser.
2. Belum ada preset resolusi lain selain `1080p`.
3. `MP4` conversion belum ada.

## Follow-up Yang Paling Masuk Akal

1. Tambah preset resolusi `720p / 1080p / 1440p`.
2. Tambah metadata/resolution label di UI setelah export selesai.
3. Jika kebutuhan format universal meningkat, tambahkan fase `WebM -> MP4` terpisah.
