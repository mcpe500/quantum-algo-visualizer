# Handoff 064: Frontend + Backend JSON-Driven Refactor

## Summary

Implemented SPEC 064. Runtime case selection now follows dataset JSON via `/cases` and backend `get_*_cases()` instead of hardcoded `DJ-01`, `QFT-01`, `VQE-01`, or `QAOA-01` defaults. Removed QAOA Simulated Annealing hardcoded K3/K4 templates. Backend scripts now discover dataset cases and use portable output paths.

## No-Cheating Rule Applied

User clarified cheating means hardcoded data when dataset JSON already exists. This session removed runtime hardcoded dataset defaults and duplicated graph templates, then verified by grep audit.

## Frontend Changes

1. `frontend/src/utils/cases.ts`
   - Added `getSortedCaseIds()` and `firstCaseId()` helpers.
   - Case IDs are extracted from API data, then sorted.

2. Hooks updated to start unresolved and load case IDs from API:
   - `frontend/src/hooks/useDJBenchmark.ts`
   - `frontend/src/hooks/useQFT.ts`
   - `frontend/src/hooks/useVQE.ts`
   - `frontend/src/hooks/useQAOA.ts`

3. `frontend/src/shared/hooks/useAlgorithmBenchmark.ts`
   - `defaultCaseId` is now optional.
   - Initial state no longer requires hardcoded `*-01`.
   - Guards run/download/load operations when no dataset case is selected.

4. `frontend/src/shared/components/AlgorithmPageShell.tsx`
   - Select and run button are disabled when no dataset cases exist.
   - Empty select shows `No dataset cases`.

5. `frontend/src/components/qaoa/QAOASimulatedAnnealingFlow.tsx`
   - Removed hardcoded K3/K4 JSON templates.
   - `Dataset Aktif` now loads JSON generated from the current benchmark result.
   - Copy updated to say JSON comes from active benchmark dataset or user paste.

6. React Compiler/lint fixes required for clean verification:
   - Moved Formula Studio hook/context exports into `FormulaStudioSyncContext.ts`.
   - Moved animation ref writes behind methods in `useAnimationEngine`.
   - Updated DJ/QFT animation canvases and shared `useVideoExport` to use those methods.
   - Reworked effect-triggered state updates via microtask scheduling where lint required it.
   - Fixed small TypeScript/ESLint issues in Formula Studio, Qubit Playground, and formula printer/registry.

## Backend Changes

1. `backend/api/shared/request_helpers.py`
   - Added `json_body()`, `resolve_case_id()`, `no_cases_response()`, `parse_int()`, `parse_float()`, and `parse_bool()`.

2. Routes updated to resolve omitted `case_id` from dataset-backed case lists:
   - `backend/api/routes/dj.py`
   - `backend/api/routes/qft.py`
   - `backend/api/routes/vqe.py`
   - `backend/api/routes/qaoa.py`

3. DJ route hardening:
   - Circuit image routes no longer return traceback text to clients.
   - Tracebacks still print server-side.

4. Backend scripts/tests are now JSON-discovered and portable:
   - `backend/generate_fft_images.py` reads QFT JSON cases and no longer uses fallback signal data.
   - `backend/generate_qft_images.py` reads QFT JSON cases and derives qubits from signal size.
   - `backend/generate_vqe_qaoa_images.py` reads VQE/QAOA cases and derives circuit parameters/graph from dataset values.
   - `backend/test_classic.py` iterates `get_dj_cases()` and is read-only.
   - `backend/test_classic_run.py` iterates `get_dj_cases()` and only writes files with `--write-results`.
   - `backend/debug_dj.py` defaults to first dataset case or accepts `--case-id`.

5. `backend/services/qft_service.py`
   - Removed unreachable timeline-building block after `return timeline`.
   - Kept live QFT timeline path that derives state from dataset `signal_data`.
