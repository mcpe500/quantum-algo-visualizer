# Handoff 063: DJ Bloch Arrow Gate Occlusion

## Summary

Fixed DJ issue where Bloch sphere arrows were unclear and disappeared when the active gate overlapped the sphere.

## Changes

1. `frontend/src/components/dj/animation/scene-primitives.tsx`
   - Imported `clamp`.
   - Replaced `sphereOffset = 0` with bounded offset:
     - base offset: `clamp(gap * 0.6, 0.62, 0.86)`
     - capped before result board.
   - Result: Bloch sphere shows state after gate, not inside the gate box.

2. `frontend/src/shared/components/3d/gates/BlochQubitNode.tsx`
   - Moved node slightly forward on z-axis.
   - Enlarged arrow head/shaft.
   - Rendered arrow with `meshBasicMaterial`, `depthTest={false}`, and `depthWrite={false}`.
   - Made sphere more transparent and disabled depth writing.
   - Muted equator ring so it does not look like the state arrow.

## Critic Finding Used

Subagent critic found the key root cause: `sphereOffset = 0` placed the Bloch sphere exactly at `columnXs[currentStep]`, the same `x` coordinate as the active gate in `StageColumn`. Gate and sphere overlapped by design.
