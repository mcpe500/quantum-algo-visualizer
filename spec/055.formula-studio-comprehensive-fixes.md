## 2. Critical Bug Fixes

### 2.1 Auto-Layout Direction Bug

**Lokasi:** `frontend/src/components/formula-studio/studio/canvasUtils.ts:158-159`

**Problem:**
```typescript
// CURRENT (BUGGY) - arah gaya tarik SALAH
forceFrom.x += forceX;  // ← node dari tertarik ke kanan
forceFrom.y += forceY;
forceTo.x -= forceX;     // ← node ke TERDORONG ke kiri (salah!)
forceTo.y -= forceY;     // ← Ini membuat node saling MENGHINDAR bukan MENDEKAT
```

**Dampak:** Node yang terhubung justru **saling menjauh**, membuat auto-layout menghasilkan layout berantakan.

**Fix:**
```typescript
// FIXED - arah gaya tarik BENAR
forceFrom.x -= horizontalForce;  // ← Tarik node kiri ke kiri
forceFrom.y += verticalForce;
forceTo.x += horizontalForce;    // ← Tarik node kanan ke kanan
forceTo.y -= verticalForce;
```

**Also fix:** Tidak ada centering force dan hard boundaries yang terlalu ketat.

### 2.2 Connection Reset Bug

**Lokasi:** `frontend/src/components/formula-studio/studio/StudioCanvas.tsx:267-268`

**Problem:**
```typescript
const handleConnectionEnd = useCallback((nodeId: string) => {
  if (connectionMode === 'selecting-target' && connectionSourceId && connectionSourceId !== nodeId) {
    dispatch({ type: 'ADD_CONNECTION', fromId: connectionSourceId, toId: nodeId, relationType: 'feeds-into', label: '→' });
  }
  // BUG: SELALU reset state, bahkan jika koneksi GAGAL
  setConnectionMode('selecting-source');  // ← Salah! Harus conditional
  setConnectionSourceId(null);              // ← Salah! Harus conditional
}, [connectionMode, connectionSourceId]);
```

**Fix:**
```typescript
const handleConnectionEnd = useCallback((nodeId: string) => {
  const success = connectionMode === 'selecting-target' && connectionSourceId && connectionSourceId !== nodeId;
  
  if (success) {
    dispatch({ type: 'ADD_CONNECTION', fromId: connectionSourceId, toId: nodeId, relationType: 'feeds-into', label: '→' });
    // Reset ke selecting-source untuk chain connections
    setConnectionMode('selecting-source');
    setConnectionSourceId(null);
  } else {
    // Jika gagal (click sama node, dll) - TETAP di selecting-target
    // Jangan reset paksa
  }
}, [connectionMode, connectionSourceId]);
```

### 2.3 Formula Node Computation Bug

**Lokasi:** `frontend/src/components/formula-studio/studio/graphEngine.ts:118-121`

**Problem:**
```typescript
} else if (node.kind === 'formula') {
  // Will be handled by the StudioCanvas using formula.computation directly
  // We still provide the varScope result here for formula nodes that have computation configs
  results.set(node.id, { value: undefined });  // ← BUG: Selalu undefined!
}
```

**Fix:** Connect formula nodes to FORMULA_COMPUTATION_MAP:
```typescript
} else if (node.kind === 'formula') {
  const formula = formulaMap.get(node.formulaId);
  const computation = formula?.computation ?? FORMULA_COMPUTATION_MAP[node.formulaId];
  
  if (computation && computation.requiresParams.length > 0) {
    // Check if all required params available in varScope
    const missingParams = computation.requiresParams.filter(p => !(p in varScope));
    
    if (missingParams.length === 0) {
      try {
        const paramValues: Record<string, number> = {};
        for (const p of computation.requiresParams) {
          paramValues[p] = varScope[p];
        }
        const steps = computation.steps(paramValues);
        const finalStep = steps[steps.length - 1];
        
        results.set(node.id, {
          value: finalStep.result !== undefined ? Number(finalStep.result) : undefined,
          valueDisplay: finalStep.result?.toString(),
          simplified: formula?.latex ?? '',
        });
      } catch (err) {
        results.set(node.id, { error: 'Kesalahan komputasi' });
      }
    } else {
      results.set(node.id, { 
        error: `Parameter diperlukan: ${missingParams.join(', ')}`,
        simplified: formula?.latex,
      });
    }
  } else {
    // No computation config - formula just shows LaTeX
    results.set(node.id, { 
      value: undefined,
      simplified: formula?.latex,
    });
  }
}
```

## 3. Core Features

### 3.1 Connection Dataflow System

**Problem:** Connections hanya visual, tidak mengalirkan nilai.

**Solution:** Implement topological execution engine.

**New File:** `frontend/src/components/formula-studio/studio/dataflowEngine.ts`

**Algorithm:**
1. Build adjacency list dari connections
2. Topological sort (Kahn's algorithm)
3. Execute nodes in order
4. Propagate values through connections
5. Detect cycles and mark as error

**Data Flow:**
```
Input Node (n=4) ──► Formula Node (dj-classical-bound) ──► Expression Node
         │                      │
         │                      ▼
         │              Computation: 2^(4-1)+1 = 9
         │                      │
         └────────── Propagate: {n: 4} ─┘
                            │
                            ▼
                    Output: {result: 9, bound: 9}
```

## 4. Example Scenarios

### 4.1 Phase Kickback Scenario

**File:** `frontend/src/components/formula-studio/scenarios/data.ts`

**Canvas Layout:**
```
   [h-gate-matrix] ──implements──► [plus-minus-states]
          │                              │
          │                              │
          ▼                              ▼
   [oracle-unitary] ──implements──► [dj-phase-kickback] ⭐
          │                                    │
          │                                    │
          ▼                                    ▼
   [pauli-z-matrix] ──related──► [Expression: (-1)^f(x)]
```

**Dataflow:**
- Input: f(x) = 0 or 1
- Compute: (-1)^f(x) → phase encoding
- Show: phase_0 = 0, phase_1 = π

**Guide Steps:**
1. Hadamard creates superposition
2. |−⟩ is eigenstate of Z with eigenvalue -1
3. Oracle applies XOR to ancilla
4. Phase kickback: U_f|x⟩|−⟩ = (−1)^f(x)|x⟩|−⟩
5. Interference via second Hadamard
6. Measurement reveals function type

### 4.2 Deutsch-Jozsa Scenario

**Canvas Layout:**
```
Row 1: [dj-oracle-function] ──implements──► [oracle-unitary] ──implements──► [dj-phase-kickback]

Row 2: [h-gate-matrix] ◄──implements── [plus-minus-states] ──implements──► [pauli-z-matrix]

Row 3: [dj-classical-bound] ──implements──► [complexity-classical] ──contrast-with──► [complexity-quantum]

Input Nodes: [n=4], [queries=5]
Expression: 2^(n-1)+1, speedup comparison
```

**Dataflow:**
```
n=4 → dj-classical-bound → 2^3+1 = 9 → classical = 10 queries
n=4 → dj-phase-kickback → quantum = 1 query
→ speedup = 10x (for n=4)
```
