# SPEC 009: VQE Webapp Implementation

## Prompt

Bangun halaman VQE yang menggantikan placeholder `/vqe`, membaca dataset `datasets/vqe/*.json`, dan menampilkan benchmarking Variational Quantum Eigensolver (VQE) versus Full Configuration Interaction (FCI). Halaman harus tetap memakai 1024 shots, menampilkan ansatz dan Hamiltonian H2, serta memperlihatkan loop optimisasi hybrid secara jelas.

## Purpose

Menjadikan VQE sebagai studi kasus algoritma hybrid yang memperlihatkan interaksi prosesor kuantum dan klasik. Visualisasi harus membantu membaca hubungan antara ansatz, Hamiltonian, pengukuran nilai ekspektasi, optimizer klasik, dan konvergensi energi menuju ground state.

## Why

1. Proposal menempatkan VQE sebagai wakil kategori simulasi dan aproksimasi.
2. Pembanding klasiknya adalah FCI, sehingga akurasi energi harus terlihat jelas.
3. Route `/vqe` masih placeholder dan belum memanfaatkan dataset JSON yang sudah ada.
4. Dataset sudah memuat `molecule`, `qubits`, `ansatz`, dan `hamiltonian`, jadi backend dapat dibangun tanpa hardcoding.
5. VQE adalah algoritma hybrid, jadi UI perlu menonjolkan loop optimisasi dan bukan hanya sirkuit kuantumnya.

## Changes Planned

### 1. Case discovery tetap data-driven

Backend membaca seluruh file `VQE-*.json` dari folder dataset. Case harus muncul otomatis tanpa menambah daftar manual di kode.

### 2. Pembangunan ansatz dari dataset

Backend membaca `ansatz.type` dan `ansatz.n_layers`. Tipe `ry_linear` dan `ry_circular` harus menghasilkan pola entanglement yang sesuai. Qubit dan koefisien Hamiltonian dibaca dari JSON, bukan ditulis ulang di kode.

### 3. Benchmark VQE versus FCI

Sisi klasik menghitung energi referensi FCI atau diagonalization eksak dari Hamiltonian yang sama. Sisi kuantum menjalankan ansatz, mengukur term Pauli, lalu menghitung energi ekspektasi pada setiap iterasi optimizer. Hasil benchmark harus memuat `time_complexity`, `circuit_depth`, `gate_count`, `shots`, `iterations`, dan `accuracy` berbasis selisih energi terhadap referensi.

### 4. Loop optimisasi hybrid

VQE harus menampilkan loop: parameter ansatz -> pengukuran -> evaluasi energi -> update parameter -> ulangi sampai konvergen. Optimizer klasik yang dipakai harus konsisten antar framework dan memakai kriteria berhenti yang sama.

### 5. Frontend VQE page

Halaman VQE mengikuti pola DJ: case selector, tab klasik/kuantum, panel metrik, visualisasi ansatz, panel Hamiltonian, chart konvergensi energi, dan section perbandingan. Tab klasik menampilkan referensi FCI dan skala bebannya, sedangkan tab kuantum menampilkan ansatz, basis measurement, dan iterasi optimizer.

### 6. Endpoint yang dibutuhkan

- `GET /api/vqe/cases`
- `POST /api/vqe/benchmark`
- `GET /api/vqe/circuit-image/<case_id>`
- `GET /api/vqe/trace/<case_id>`
- `GET /api/vqe/dataset/<case_id>`

## Logic / Pseudocode

```text
scan datasets/vqe/*.json
load selected case
extract molecule, qubits, ansatz, and hamiltonian terms

classical side:
  compute FCI / exact reference energy
  derive classical scaling summary

quantum side:
  build parameterized ansatz from ansatz.type and n_layers
  apply Pauli measurement groups for Hamiltonian terms
  run optimizer loop with 1024 shots per evaluation
  stop when energy change is below threshold or max iterations reached

benchmark result:
  report energy error, depth, gate count, iterations, and accuracy
  return per-framework summary
```
