# SPEC 041 — Shared HadamardGate Component

**Date:** 2026-04-21
**Status:** COMPLETE

## Prompt

Create shared HadamardGate component that all 3 algorithms (DJ, QFT, QAOA) use for Hadamard gate rendering. QFT currently uses sphereGeometry (GateNode) — change to boxGeometry (DJ style). QAOA uses cylinderGeometry (GateDisc) — change to boxGeometry. DJ already uses boxGeometry (GateTile) — optionally migrate H to HadamardGate.

## Goal

Unified Hadamard gate visual across all algorithm animations. All H gates render as blue box plate (boxGeometry) with consistent active/inactive states and color.

## Research Findings (No Hallucination)

### Real Qiskit Hadamard

**DJ Backend** (`backend/services/dj_service.py`):
- `qc.h(qr_in[:])` — batch Hadamard on ALL input qubits at start
- `qc.h(qr_anc[0])` — single Hadamard on ancilla
- `qc.h(qr_in[:])` — batch Hadamard at end (interference)

**QFT Backend** (`backend/services/qft_service.py`):
- Sequential `qc.h(qr[j])` per qubit (j=n-1→0), interleaved with CP gates
- Each qubit gets one H gate as first step per processing row

**QAOA Backend**:
- Hadamard creates superposition at start of quantum circuit
- Applied to all qubits simultaneously (layer)

### Animation State Mapping

| Algorithm | Data Trigger | Backend Alignment |
|-----------|-------------|-------------------|
| DJ | `wire_markers: {"0":"H", "1":"H", ...}` | Batch H via `qc.h(qr_in[:])` ✅ |
| QFT | `timeline[step].phase === 'hadamard' && step.target_qubit === j` | Sequential H via `qc.h(qr[j])` ✅ |
| QAOA | `activeStep.phase === 'superposition'` | Batch H on all qubits ✅ |
