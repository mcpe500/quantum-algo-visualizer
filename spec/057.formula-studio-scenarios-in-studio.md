# SPEC 057: Formula Studio Scenario Loader in Studio Tab

## Status: COMPLETED

**Tanggal:** 2026-04-26  
**Project:** quantum-algo-visualizer  
**Area:** Formula Studio (`/formulas`)  
**Scope:** Menempatkan skenario siap pakai di tab **Studio**, bukan di Explore atau Stories.

---

## 1. Prompt

User menegaskan bahwa skenario yang sudah dibuat harus tersedia langsung di Formula Studio, khususnya pada tab **Studio** di halaman `http://localhost:5173/formulas`. Skenario tidak boleh hanya hidup sebagai narasi di `Explore` atau `Stories`, karena tujuan utamanya adalah memuat canvas siap pakai yang dapat diedit, dihubungkan, dan dikomputasi di workspace Studio.

---

## 2. Tujuan

Tujuan perubahan adalah menambahkan pemuat skenario di toolbar Studio agar pengguna dapat memilih skenario algoritma dan langsung memuat graph formula ke canvas. Skenario ini harus berisi node formula, node input, node ekspresi, dan koneksi yang sudah disusun sehingga Studio tidak lagi dimulai dari canvas kosong ketika pengguna ingin mendemonstrasikan alur algoritma tertentu.

---

## 3. Alasan Perubahan

Sebelum perubahan ini, Formula Studio sudah memiliki tab `Stories` berisi narasi langkah algoritma, serta tab `Explore` untuk melihat katalog formula. Namun kedua tab tersebut tidak memenuhi kebutuhan kerja Studio, karena pengguna tidak dapat langsung memanipulasi graph canvas dari skenario tersebut. Dengan menempatkan skenario pada toolbar Studio, alur kerja menjadi lebih langsung: pilih skenario, muat ke canvas, lalu edit node, sambungkan ulang koneksi, jalankan komputasi input, atau ambil screenshot.

---

## 4. Codebase Context

Formula Studio berada di folder `frontend/src/components/formula-studio`. Struktur relevan:

- `FormulaStudioPage.tsx` mengatur tab `Explore`, `Studio`, dan `Stories`.
- `studio/StudioCanvas.tsx` mengatur canvas, node, koneksi, inspector, toolbar, dan persistence.
- `studio/CanvasToolbar.tsx` berisi aksi utama Studio seperti Input, Ekspresi, Connect, Auto, Screenshot, dan Clear.
- `studio/canvas-types.ts` mendefinisikan `CanvasState`, `CanvasNodeData`, dan `CanvasConnection`.
- `studio/useCanvasReducer.ts` sudah memiliki action `LOAD_CANVAS`, sehingga pemuatan skenario dapat memakai reducer yang sudah ada.
- `stories/data.ts` sudah menyimpan narasi algoritma, tetapi tidak memuat canvas state yang bisa diedit di Studio.

---

## 5. Implementasi Logis

Perubahan logis yang diterapkan:

1. Menambahkan module `scenarios` khusus untuk data skenario Studio.
2. Mendefinisikan tipe `FormulaStudioScenario` yang berisi metadata skenario dan `CanvasState` siap pakai.
3. Menyediakan empat skenario sesuai cakupan proyek:
   - Deutsch-Jozsa: Oracle → Phase Kickback
   - QFT: DFT → Twiddle Factor → Gate Count
   - VQE: Hamiltonian → Energi Variational
   - QAOA: Max-Cut → Ansatz → Acceptance
4. Menambahkan dropdown skenario dan tombol `Muat` di toolbar Studio.
5. Menghubungkan pilihan skenario ke `LOAD_CANVAS` pada reducer.
6. Membersihkan mode koneksi ketika skenario baru dimuat agar state interaksi tidak bocor dari canvas sebelumnya.
7. Memperbaiki error TypeScript Formula Studio yang menghalangi build, termasuk union error code engine, import tidak terpakai, dan typing `dataflowEngine`.

---

## 6. Pseudocode

```text
on StudioToolbar render:
  tampilkan select skenario
  jika user memilih skenario:
    simpan scenarioId lokal
  jika user klik Muat:
    panggil onLoadScenario(scenarioId)

onLoadScenario(scenarioId):
  cari scenario di FORMULA_STUDIO_SCENARIOS
  jika tidak ditemukan:
    stop
  clone scenario.state agar data konstanta tidak termutasi
  dispatch LOAD_CANVAS dengan state hasil clone
  set connectionMode ke idle
  set connectionSourceId ke null
```

## 8. Manual Testing Plan

9. Klik `Auto` untuk memastikan layout masih dapat disusun ulang.
