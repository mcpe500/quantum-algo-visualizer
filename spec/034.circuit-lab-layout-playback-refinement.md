## Tujuan

Merapikan implementasi `Circuit Lab` agar lebih dekat ke struktur referensi visual tanpa keluar dari bahasa visual repo. Fokus session ini adalah memperbaiki layout yang masih collapse, menambahkan playback interaktif yang benar-benar bisa di-scrub, dan menyambungkan preview, trace, serta grid ke langkah aktif yang sama.

## Kenapa

Versi sebelumnya sudah fungsional, tetapi belum terasa seperti laboratorium sirkuit yang matang. Setelah layout dirombak pertama kali, screenshot menunjukkan kolom utama masih terlalu sempit dan hierarchy visual gagal. Karena itu perbaikannya tidak cukup pada styling permukaan, tetapi harus menyentuh distribusi ruang halaman, model playback, dan sinkronisasi state per langkah.

## Scope

- Tetap di route `/playground`
- Tetap di tab `Circuit Lab`
- Tidak menambah dependency baru
- Menggunakan referensi HTML hanya sebagai acuan struktur dan interaction pattern
- Menjaga `State Lab` tetap berjalan seperti sebelumnya

## Perubahan Utama

### 1. Playback berbasis snapshot langkah

Hook `useCircuitBuilder` ditambah data turunan baru:

```ts
interface CircuitPlaybackFrame {
  stepIndex: number;
  placementId: string | null;
  label: string;
  summary: string;
  ket: string;
  column: number | null;
  phenomena: string[];
  statevector: Complex[];
  blochData: BlochData[];
  blochCards: Array<BlochData & { label: string }>;
}
```

Isi frame dimulai dari:

- `step 0` = initial state `|00...0⟩`
- `step 1..n` = snapshot sesudah tiap gate berdasarkan urutan replay `column`, lalu `row`

Dengan ini UI tidak lagi menebak state aktif, tetapi membaca langsung snapshot terstruktur dari hook.

### 2. Playback bar interaktif pada Circuit Lab

`CircuitLabTab` diperbarui dengan:

- tombol `previous`, `play/pause`, `next`
- slider langkah
- indikator `STEP x/y`
- panel ringkasan langkah aktif berisi:
  - label step
  - summary
  - ket notation
  - badge fenomena pada langkah aktif

### 3. Trace sinkron dengan playback

Trace kini membaca `playbackFrames`, sehingga:

- initial state ikut muncul sebagai langkah pertama
- setiap item trace bisa diklik untuk lompat ke langkah terkait
- langkah aktif diberi highlight visual
- ket notation tampil pada setiap langkah

### 4. Preview sinkron dengan langkah aktif

Panel `Live Qubit Preview` tidak lagi selalu menampilkan final state. Preview sekarang mengikuti slider playback, sehingga user bisa melihat keadaan qubit pada setiap langkah replay.

### 5. Connector visual untuk gate dua qubit

Canvas sirkuit ditingkatkan agar gate `CNOT`, `CPhase`, dan `SWAP` tidak lagi tampil seperti dua sel yang terpisah. Sekarang ada:

- garis vertikal control-target
- simbol control / target yang lebih dekat ke notasi sirkuit
- badge secondary role seperti `TARGET`, `PHASE`, dan `SWAP`

### 6. Layout utama dibuat deterministik

Masalah terbesar dari screenshot adalah kolom utama masih collapse. Karena itu layout diganti dari pembagian span generik ke distribusi lebar yang eksplisit:

- kiri: `280px`
- tengah: `minmax(0, 1fr)`
- kanan: `320px`

Selain itu, halaman `Qubit Playground` diberi wrapper lebih lebar saat tab `Circuit Lab` aktif:

- `State Lab` tetap `max-w-[1280px]`
- `Circuit Lab` menjadi `max-w-[1600px]`

Ini memastikan workspace sirkuit memperoleh ruang horizontal yang cukup untuk palette, canvas, trace, dan preview.

### 7. Preview component dibuat mendukung mode sempit

`QubitStatePreview` sekarang punya prop `compact` agar panel preview di sidebar kanan tetap terbaca dan tidak memaksa grid internal menjadi terlalu padat.

## Manual Testing Plan

1. Buka `/playground`
2. Masuk ke tab `Circuit Lab`
3. Pastikan layout utama terbagi jelas menjadi palette kiri, canvas tengah, dan preview/inspector kanan
4. Drag `H` ke `q0` kolom 1
5. Pastikan preview kanan berubah saat slider berada di step `1`
6. Klik `play` pada playback bar dan pastikan step bergerak otomatis hingga langkah terakhir
7. Klik item trace mana pun dan pastikan slider, highlight grid, serta preview ikut pindah ke langkah yang sama
8. Drag `CNOT` ke `q0` kolom berikutnya pada mode 2 qubit
9. Pastikan connector control-target muncul jelas di canvas
10. Klik gate parametrik lalu ubah angle di inspector
11. Pastikan trace, preview, dan final state ikut diperbarui
12. Klik `Reset`
13. Pastikan playback kembali ke initial state dan seluruh placement hilang

## Catatan

1. Layout menggunakan ukuran kolom eksplisit karena pembagian `3/6/3` sebelumnya masih menghasilkan workspace yang terlalu sempit.
2. Playback tetap memakai engine simulasi yang sama dengan `State Lab`; yang berubah hanya cara snapshot diekspos ke UI.
3. Referensi HTML dipakai pada level struktur informasi dan interaction pattern, tetapi visual tetap diselaraskan dengan kartu terang dan border halus yang sudah dipakai repo.
