# SPEC 029: QFT Quantum Animation 3D — Signal-to-Spectrum Visualization

## Prompt

Implementasikan animasi 3D Three.js untuk QFT (Quantum Fourier Transform) yang menampilkan transformasi sinyal domain waktu ke domain frekuensi dengan visualisasi phase cascade yang jelas. Animasi harus mengikuti pola circuit-diagram seperti DJ, dengan horizontal wires, phase rotation landmarks, dan signal bars 3D. Panel kanan harus menjelaskan transformasi signal → phase cascade dengan phase wheels yang dinamis.

## Goal

User non-teknis harus langsung paham:

1. mana sinyal input (domain waktu) dalam bentuk bar chart 3D,
2. mana proses inisialisasi amplitude ke statevector,
3. mana proses Hadamard (membuka superposisi),
4. mana phase cascade (controlled rotations R_k yang akumulasi phase),
5. mana SWAP operations (bit reversal),
6. mana output spektrum frekuensi (domain frekuensi),
7. bagaimana phase angles berkembang step-by-step (phase wheel visualization).

## Purpose

Menyediakan visualisasi end-to-end QFT yang menjelaskan:
- Bagaimana sinyal diskret (amplitudo real) di-encode ke statevector kuantum
- Bagaimana QFT circuit mengubah basis komputasi ke basis Fourier melalui phase manipulation
- Mengapa QFT memiliki kompleksitas O(log² N) vs FFT O(N log N)
- Hubungan antara phase accumulation dan frequency detection

## Why

1. Proposal menempatkan QFT sebagai wakil kategori aljabar dan teori bilangan dengan pembanding FFT.
2. QFT adalah algoritma fundamental yang menjadi inti dari Shor's algorithm dan banyak aplikasi quantum.
3. QFT memiliki visualisasi unik (phase rotation) yang berbeda dari oracle-based algorithms seperti DJ.
4. Route QFT sudah memiliki webapp (008) dan signal topography (017), tetapi belum memiliki quantum animation 3D.
5. Dataset QFT bersifat signal-based, memungkinkan visualisasi yang intuitif (sinyal masuk → spektrum keluar).

### Backend: QFT Circuit Builder with Timeline

```python
def build_qft_animation_timeline(signal_vector, n_qubits):
    """
    Build step-by-step timeline untuk QFT animation.
    Setiap step mencakup statevector snapshot dengan phase angles.
    """
    timeline = []
    steps_count = 0
    
    # Phase 1: Signal Initialization (2-3 steps)
    # Step: Initialize statevector dengan signal amplitudes (normalized)
    state = normalize(signal_vector)  # complex amplitudes
    timeline.append({
        'step': steps_count,
        'phase': 'init',
        'operation': 'Initialize Statevector',
        'description': f'Load {len(signal_vector)} signal amplitudes into quantum state',
        'statevector': state,
        'qubit_phases': [0.0] * n_qubits  # semua phase 0
    })
    steps_count += 1
    
    # Phase 2: Hadamard Layer (n steps)
    for j in range(n_qubits):
        # Apply Hadamard ke qubit j
        state = apply_hadamard(state, j, n_qubits)
        timeline.append({
            'step': steps_count,
            'phase': 'hadamard',
            'operation': f'H on qubit {j}',
            'target_qubit': j,
            'description': 'Create superposition for frequency analysis',
            'statevector': state,
            'qubit_phases': extract_phases(state, n_qubits)
        })
        steps_count += 1
    
    # Phase 3: Phase Cascade (n(n-1)/2 steps)
    for j in range(n_qubits):
        for k in range(j + 1, n_qubits):
            angle = pi / (2 ** (k - j))  # R_{k-j+1} rotation
            # Apply controlled-phase: control=k, target=j
            state = apply_controlled_phase(state, control=k, target=j, angle=angle)
            timeline.append({
                'step': steps_count,
                'phase': 'phase_cascade',
                'operation': f'CPHASE({angle:.4f}) on q{j} controlled by q{k}',
                'target_qubit': j,
                'control_qubit': k,
                'rotation_angle': angle,
                'description': f'Add phase rotation π/2^{k-j}',
                'statevector': state,
                'qubit_phases': extract_phases(state, n_qubits)
            })
            steps_count += 1
    
    # Phase 4: SWAP Network (n/2 steps)
    for i in range(n_qubits // 2):
        state = apply_swap(state, i, n_qubits - i - 1)
        timeline.append({
            'step': steps_count,
            'phase': 'swap',
            'operation': f'SWAP qubit {i} and {n_qubits - i - 1}',
            'description': 'Bit reversal ordering correction',
            'statevector': state,
            'qubit_phases': extract_phases(state, n_qubits)
        })
        steps_count += 1
    
    # Phase 5: Measurement (1 step)
    probabilities = compute_measurement_probabilities(state)
    timeline.append({
        'step': steps_count,
        'phase': 'measurement',
        'operation': 'Measure all qubits',
        'description': 'Collapse to frequency spectrum',
        'probabilities': probabilities,
        'qubit_phases': extract_phases(state, n_qubits)
    })
    
    return timeline


def extract_phases(statevector, n_qubits):
    """
    Extract accumulated phase angle φ untuk setiap qubit dari statevector.
    Untuk QFT, phase accumulation adalah inti dari algoritma.
    """
    phases = []
    for j in range(n_qubits):
        # Compute reduced density matrix untuk qubit j
        rho = partial_trace(statevector, j, n_qubits)
        # Extract phase dari off-diagonal elements
        phase = atan2(rho[0,1].imag, rho[0,1].real)
        phases.append(phase)
    return phases
```

### Frontend: Phase Cascade Derivation (like DJ's oracle-construction.ts)

```typescript
// frontend/src/components/qft/animation/phase-cascade.ts

interface PhaseCascadeRow {
  qubitIndex: number;
  stepNumber: number;
  intro: string;
  subSteps: {
    phase: 'init' | 'hadamard' | 'controlled_r' | 'swap';
    gate: string;
    angle: number;  // in radians
    controlQubit?: number;
    detail: string;
  }[];
  accumulatedPhase: number;  // total phase angle
  gateSequence: string;
}

interface PhaseCascadeModel {
  nQubits: number;
  signalType: string;
  intro: string;
  overviewBlock: {
    label: string;
    totalGates: number;
    complexity: string;
  };
  rows: PhaseCascadeRow[];
  activeQubit: number | null;
}

function buildPhaseCascadeModel(
  data: QFTAnimationPayload,
  activeQubit: number | null
): PhaseCascadeModel {
  const n = data.n_qubits;
  const rows: PhaseCascadeRow[] = [];
  
  // Build phase cascade explanation per qubit
  for (let j = 0; j < n; j++) {
    const subSteps = [];
    let accumulatedPhase = 0;
    
    // Step 1: Initialize
    subSteps.push({
      phase: 'init',
      gate: 'Load',
      angle: 0,
      detail: `Initialize qubit ${j} with signal amplitude`
    });
    
    // Step 2: Hadamard
    subSteps.push({
      phase: 'hadamard',
      gate: 'H',
      angle: 0,
      detail: 'Create superposition (rotation around Y-axis by 90°)'
    });
    
    // Step 3: Controlled rotations
    for (let k = j + 1; k < n; k++) {
      const angle = Math.PI / Math.pow(2, k - j);
      accumulatedPhase += angle;
      subSteps.push({
        phase: 'controlled_r',
        gate: `CR_${k-j+1}`,
        angle: angle,
        controlQubit: k,
        detail: `Controlled rotation by π/2^${k-j} = ${(angle * 180 / Math.PI).toFixed(1)}°`
      });
    }
    
    // Step 4: SWAP (if needed for bit reversal)
    if (j < n / 2) {
      subSteps.push({
        phase: 'swap',
        gate: 'SWAP',
        angle: 0,
        detail: `Swap with qubit ${n - j - 1} for bit reversal`
      });
    }
    
    const gateSequence = subSteps
      .map(s => s.gate)
      .join(' → ');
    
    rows.push({
      qubitIndex: j,
      stepNumber: j + 1,
      intro: `Qubit ${j} undergoes ${subSteps.length} operations with total phase accumulation of ${(accumulatedPhase * 180 / Math.PI).toFixed(1)}°`,
      subSteps,
      accumulatedPhase,
      gateSequence
    });
  }
  
  return {
    nQubits: n,
    signalType: data.signal_type,
    intro: `${n}-qubit QFT transforms ${Math.pow(2, n)} time-domain samples into frequency spectrum via phase cascade. Total gates: ${n} Hadamard + ${n*(n-1)/2} controlled-phase + ${Math.floor(n/2)} SWAP.`,
    overviewBlock: {
      label: `${n}-Qubit QFT`,
      totalGates: n + n*(n-1)/2 + Math.floor(n/2),
      complexity: `Quantum: O(log²N)=O(${n}²) vs Classical FFT: O(N log N)=O(${Math.pow(2,n)}·${n})`
    },
    rows,
    activeQubit
  };
}
```

## Catatan Penting

### Perbedaan Fundamental QFT vs DJ

| Aspek | DJ (Oracle-based) | QFT (Transform-based) |
|-------|-------------------|----------------------|
| **Input** | Truth table (f(x) → {0,1}) | Signal vector (amplitudes) |
| **Core Operation** | Oracle evaluation (X→MCX→X) | Phase rotation cascade |
| **Visualization** | Oracle box, MCX controls | Phase wheels, angle accumulation |
| **Output** | Konstanta/Balanced classification | Frequency spectrum |
| **Complexity** | O(n) vs O(2^n) | O(log²N) vs O(N log N) |
| **State Tracking** | Wire markers (0/1/+) | Phase angles (φ) |

### QFT-Specific Technical Details

1. **Phase Accumulation Formula**:
   - Untuk qubit j: φ_j = Σ_{k=j+1}^{n-1} π/2^{k-j}
   - Contoh n=3:
     - Qubit 0: π/2 + π/4 = 135°
     - Qubit 1: π/2 = 90°
     - Qubit 2: 0°

2. **Statevector Tracking**:
   - DJ: Track |0⟩/|1⟩ probabilities (pOne)
   - QFT: Track complex amplitudes dan extract phase angles φ
   - Perlu partial trace untuk reduced density matrix per qubit

3. **Bit Reversal**:
   - QFT output secara natural dalam bit-reversed order
   - SWAP gates mengembalikan ke natural order
   - Contoh: |001⟩ → |100⟩ (bit reversal 3-qubit)

4. **Signal Encoding**:
   - Classical input: real amplitudes a_j
   - Quantum state: |ψ⟩ = Σ a_j/√N |j⟩ (normalized)
   - Measurement gives frequency bin probabilities |FFT coefficient|²
