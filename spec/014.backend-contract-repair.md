# SPEC 014: Backend Contract Repair for QFT, VQE, QAOA

## Prompt

Fix runtime failures after backend refactor:

1. `QFTClassicTab` crash: `Cannot read properties of undefined (reading 'time_complexity')`
2. `VQEClassicTab` crash: `Cannot read properties of undefined (reading 'matrix_size')`
3. QAOA `500` on `/benchmark`, `/trace`, `/circuit-image` caused by edge shape mismatch (`list` vs `dict`)
4. QFT benchmark memory/runtime failure from wrong qubit-size interpretation.

## Tujuan

Stabilize all QFT/VQE/QAOA pages by restoring API payload contracts expected by frontend components, while preserving refactored backend structure (`routes` + `services`) and existing endpoint paths.

## Mengapa Perlu

Backend refactor changed service payload shape into simplified structures that no longer matched frontend types:

- `frontend/src/components/vqe/VQEClassicTab.tsx` expects `result.classical.matrix_size`
- `frontend/src/components/qft/QFTClassicTab.tsx` expects `result.fft.time_complexity`
- `frontend/src/components/qaoa/*` expects benchmark sections `exact/classical/quantum/comparison`

QAOA services also assumed edge objects (`{u,v,weight}`), while dataset stores `graph.edges` as arrays (`[u,v]`).

## Perubahan Implementasi

### 1) VQE service rebuilt to contract-compatible payload

Updated `backend/services/vqe_service.py`:

- Reintroduced full benchmark structure:
  - `classical`: `energy`, `matrix_size`, `time_complexity`, `execution_time_ms`, `note`
  - `quantum`: convergence history, optimal params, depth, gate count, iterations, accuracy
  - `comparison`: FCI vs VQE summary
- Added robust ansatz builders (`ry_linear`, `ry_circular`) with ParameterVector
- Added trace builder with `stages` + `partitions` format consumed by frontend
- Kept route function signatures unchanged (`run_vqe_payload`, `get_vqe_trace_payload`, etc.)

### 2) QFT service rebuilt to FFT/QFT comparison shape

Updated `backend/services/qft_service.py`:

- Restored benchmark payload with:
  - top-level: `input_signal`, `padded_signal`, `n_points_original`, `n_points_padded`, `n_qubits`
  - `fft`: `time_complexity`, `execution_time_ms`, `spectrum`, `dominant_bins`, `dominant_magnitudes`
  - `qft`: `num_qubits`, `counts`, `probabilities`, `input_amplitudes`, depth, gate count
  - `comparison`: classical/quantum complexity notes
- Fixed qubit sizing logic: use `log2(n_points_padded)` instead of using raw point count as qubit count
- Added consistent trace payload (`stages`, `partitions`) for QFT trace table

### 3) QAOA service rebuilt with edge normalization

Updated `backend/services/qaoa_service.py`:

- Introduced unified edge parser handling:
  - dict edge: `{u, v, weight}`
  - tuple/list edge: `[u, v]` or `[u, v, weight]`
- Centralized problem normalization in `_get_problem_from_case`
- Ensured all paths use normalized edges:
  - benchmark optimization
  - classical baseline
  - trace generation
  - circuit image generation
- Restored full benchmark sections expected by frontend:
  - `exact`, `classical`, `quantum`, `comparison`
- Returned trace payload with `n_nodes`, `n_edges`, `p_layers`, `stages`, `partitions`

## Logika Perubahan (Pseudocode)

### VQE benchmark

```text
load case -> build SparsePauliOp
classical: exact diagonalization (eigvalsh) -> FCI energy
quantum: COBYLA optimize ansatz parameters using Statevector expectation
compute energy_error + accuracy
return {classical, quantum, comparison} contract
```

### QFT benchmark

```text
load signal from dataset
resolve points -> pad to next power of 2
n_qubits = log2(padded_points)
run FFT on padded signal
prepare amplitude-encoded state -> run QFT circuit -> measure counts
return {fft, qft, comparison} contract
```

### QAOA benchmark

```text
load graph case
normalize edges from any supported shape
exact: brute force max-cut
classical: simulated annealing
quantum: optimize gamma/beta with COBYLA on expectation value
run final measured circuit
return {exact, classical, quantum, comparison}
```
