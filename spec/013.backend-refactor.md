# SPEC 013 — Backend Refactor

## Latar Belakang

Backend saat ini menempatkan terlalu banyak tanggung jawab di file modul route, terutama pada `backend/api/dj.py`. Di sana route Flask, akses dataset, pembentukan circuit, simulasi quantum/classical, generasi pseudocode, trace building, dan rendering image bercampur dalam satu file besar. Pola serupa muncul juga pada `qft.py`, `vqe.py`, dan `qaoa.py`.

Akibatnya:
- sulit melakukan test unit pada logic tanpa memanggil layer HTTP
- perubahan kecil di domain logic berisiko mempengaruhi route
- reuse helper menjadi buruk karena helper hidup di file endpoint
- file sangat panjang dan sulit dipelihara

## Tujuan

Refactor backend tanpa mengubah kontrak API yang sudah dipakai frontend:
- path endpoint tetap sama
- bentuk payload tetap semirip mungkin
- pemisahan concern menjadi lebih jelas
- bootstrap aplikasi lebih rapi dan mudah dijalankan/test

## Ruang Lingkup

Termasuk:
- refactor `backend/app.py` menjadi app factory
- menambah `backend/run.py`
- memecah route ke `backend/api/routes/*`
- memindahkan domain/service logic ke `backend/services/*`
- memindahkan helper API lintas-file ke `backend/api/shared/*`
- merapikan konfigurasi sederhana di `backend/config.py`

Tidak termasuk:
- perubahan endpoint frontend
- perubahan format dataset
- perubahan algoritma inti yang bersifat fungsional
- optimisasi performa berat
- penambahan test suite baru

## Desain Target

### 1. App Factory
`backend/app.py` diubah agar hanya bertugas membuat app Flask melalui `create_app()`.

`backend/run.py` ditambahkan sebagai entrypoint sederhana untuk local run.

### 2. Route Layer Tipis
Route dipindah ke:
- `backend/api/routes/dj.py`
- `backend/api/routes/qft.py`
- `backend/api/routes/vqe.py`
- `backend/api/routes/qaoa.py`

Prinsip:
- ambil input dari request
- validasi ringan/default value
- panggil service
- bentuk response HTTP

### 3. Service Layer
Logic domain dipindah ke:
- `backend/services/common.py`
- `backend/services/dj_service.py`
- `backend/services/qft_service.py`
- `backend/services/vqe_service.py`
- `backend/services/qaoa_service.py`

Prinsip:
- dataset loading dan listing berada di service/common
- per-algorithm logic berada di service masing-masing
- route cukup memanggil fungsi payload/service

### 4. Shared API Helpers
Helper yang dipakai lintas endpoint dipindah ke:
- `backend/api/shared/dj_partition.py`
- `backend/api/shared/plotting.py`

## Detail Perubahan

### Bootstrap
Sebelum:
- `backend/app.py` langsung membuat objek app global

Sesudah:
- `create_app()` menjadi satu-satunya factory
- `backend/run.py` membuat app instance runtime

### Config
Sebelum:
- `Config.DEBUG = True` hardcoded

Sesudah:
- `DEBUG` dibaca dari environment `FLASK_DEBUG`
- `get_config()` disediakan untuk dipakai app factory

### DJ
Sebelum:
- semua logic DJ berada di `backend/api/dj.py`

Sesudah:
- route DJ di `backend/api/routes/dj.py`
- logic DJ di `backend/services/dj_service.py`
- helper partition trace di `backend/api/shared/dj_partition.py`
- helper plotting phase-box dan base64 di `backend/api/shared/plotting.py`

### QFT / VQE / QAOA
Sebelum:
- masing-masing file route juga berisi seluruh logic

Sesudah:
- masing-masing dibagi menjadi route layer + service layer
- bentuk payload dipertahankan setara

## Dampak ke Frontend

Tidak ada perubahan endpoint yang diwajibkan. Frontend tetap bisa memanggil:
- `/api/dj/*`
- `/api/qft/*`
- `/api/vqe/*`
- `/api/qaoa/*`

## Risiko

- import path berubah, sehingga raw execution file tertentu bisa gagal bila ada script lama yang mengandalkan struktur modul lama
- jika ada test atau util eksternal yang import langsung dari `backend/api/dj.py`, maka perlu diarahkan ke service baru
- perilaku `DEBUG` kini mengikuti env var, bukan hardcoded selalu true

## Validasi

Checklist validasi minimal:
- import app dari `run.py` berhasil
- seluruh file Python hasil refactor lolos compile
- endpoint utama tetap resolve

## Langkah Adopsi

1. Tambahkan folder baru:
   - `backend/api/routes`
   - `backend/api/shared`
   - `backend/services`

2. Replace isi:
   - `backend/app.py`
   - `backend/config.py`
   - `backend/api/__init__.py`

3. Tambahkan file:
   - `backend/run.py`
   - seluruh file route/service/shared hasil refactor

4. Hapus modul lama:
   - `backend/api/dj.py`
   - `backend/api/qft.py`
   - `backend/api/vqe.py`
   - `backend/api/qaoa.py`

## Hasil Akhir yang Diinginkan

Backend memiliki struktur yang lebih stabil:
- HTTP layer tipis
- logic reusable
- helper terpusat
- bootstrap lebih clean
- kontrak API lama tetap aman
