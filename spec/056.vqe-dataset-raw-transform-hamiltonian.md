# SPEC 056: VQE Dataset RAW to TRANSFORM to Hamiltonian Visualization

## Prompt

User requested a stronger VQE dataset visualization that explains the data flow:

```text
RAW -> TRANSFORM -> Hamiltonian
```

The visualization must be based on the actual VQE JSON data, must not hallucinate chemistry details, and must be understandable for an informatics student. User explicitly requested subagent critic review and sequential thinking.

## Purpose

Make the VQE dataset page explain how raw case specification becomes the canonical Pauli Hamiltonian used by VQE. The page must separate:

1. raw molecule and experiment specification;
2. preprocessing/canonicalization boundary;
3. canonical Hamiltonian terms used by VQE.

## Why

VQE is harder to understand visually than DJ, QFT, or QAOA because the visible input is not directly the object optimized by the algorithm. Raw VQE JSON describes H2 and experiment settings, while VQE evaluates a qubit Hamiltonian expressed as Pauli strings. Without a clear visual bridge, users can misread the dataset as if raw JSON directly contains the Hamiltonian.

## Source of Truth

Primary runtime data:

- `datasets/vqe/VQE-01.json`
- `datasets/vqe/VQE-02.json`
- `datasets/vqe/canonical/VQE-01.canonical.json`
- `datasets/vqe/canonical/VQE-02.canonical.json`

Important correction:

- User mentioned `scripts/datasets/vqe`, but the current active app dataset path is `datasets/vqe`.
- Canonical VQE-02 currently uses `ry_linear`, 2 layers, 15 Pauli terms, including X/Y mixed terms.
- Older notes or stale drafts may mention `ry_circular`, 16 all-Z terms. That version must not drive the visualizer.

## Critic Findings Applied

The visualization must not claim that raw JSON directly generates Hamiltonian terms. Raw JSON only stores the case specification. The canonical Hamiltonian is produced or selected by preprocessing.

VQE-01 and VQE-02 must not be shown as identical transform paths:

- VQE-01 uses verified 2-qubit coefficients for H2/STO-3G.
- VQE-02 uses the preprocessing pipeline and Jordan-Wigner mapping into a 4-qubit Hamiltonian.

Pauli strings must be explained as computational operators, not literal chemical bonds or orbital pictures.

### Backend

Added `get_vqe_dataset_payload(case_id)` in `backend/services/vqe_service.py`.

Pseudo-flow:

```python
raw = load_case("vqe", case_id)
canonical = load_case_canonical("vqe", case_id)

payload = canonical
payload["raw_spec"] = {
    "problem_type": raw["problem_type"],
    "molecule_spec": raw["molecule_spec"],
    "preprocessing": raw["preprocessing"],
    "experiment": raw["experiment"],
}
payload["transform"] = {
    "source": "verified_2q_coefficients" or "jordan_wigner_mapping",
    "mapping": raw["preprocessing"]["mapping"],
    "target_qubits": raw["preprocessing"]["target_qubits"],
    "canonical_terms": len(canonical["hamiltonian"]["terms"]),
    "note": "...",
}
```

Updated `GET /api/vqe/dataset/<case_id>` to return this enriched dataset payload.
