# SPEC 003: Fix DJ Classical Visualization Layout for TA Book

## Prompt

Fix layout visualisasi DJ klasik (brute force) yang jelek: tiga panel (INPUTS, ORACLE, HASIL) tidak sejajar, dead space masif, dan aspek rasio 16:9 tidak cocok untuk buku A4 portrait. Empat reviewer setuju layout ini tidak pantas masuk TA.

## Purpose

Membuat figur visualisasi DJ klasik yang:
- Proporsional untuk buku TA format A4 portrait
- Tidak ada dead space / "?" boxes yang membuang ruang
- Tiga panel align-top (bukan float di tengah)
- Informasi density tinggi (data-ink ratio > 0.8)
- Konsisten antar DJ-01 s/d DJ-04

## Why

Reviewer feedback unanimous:
1. Panel HASIL punya 25-78% dead space ("?" boxes)
2. Panel INPUTS menampilkan semua 2^n input padahal cuma pakai 3-9
3. Panel ORACLE punya fixed height h-[180px] yang memaksa scroll/waste
4. `items-center` membuat panel pendek "melayang" di tengah
5. Header memakan 30% tinggi canvas
6. Rasio 16:9 landscape tidak cocok untuk A4 portrait

## Codebase Context

**Frontend React + Tailwind:**
- `frontend/src/components/dj/ClassicalVisualization.tsx` — main layout container
- `frontend/src/components/dj/InputsPanel.tsx` — panel kiri (daftar input)
- `frontend/src/components/dj/OraclePanel.tsx` — panel tengah (query log)
- `frontend/src/components/dj/ResultPanel.tsx` — panel kanan (output)
- `frontend/src/components/dj/types/classical.ts` — TypeScript types

**Backend (unchanged):**
- `backend/api/dj.py` — endpoint `/api/dj/classic-run` returns stepped results

## Changes Made

### 1. ClassicalVisualization.tsx

| Before | After | Why |
|--------|-------|-----|
| `items-center` | `items-start` | Align top, no floating |
| `gap-4` | `gap-6` | Proportional spacing for book |
| `pt-16` | `pt-4` | Compress whitespace |
| `pt-8 lg:pt-10` header | `pt-4 pb-2` | Compress header |
| `mt-12 lg:mt-16` | `mt-6 lg:mt-8` | Tighter flow |
| `mt-16` footer | `mt-8` | Less dead space |
| `max-w-6xl` | `max-w-4xl` | Narrower = better for A4 |
| Arrow 48px | Arrow 40px | Proportional |
| Arrow `self-center` | Arrow `self-center mt-6` | Align with content |
| Footer text vague | Footer text precise | Shows query count + classification |

### 2. InputsPanel.tsx — Compact Ellipsis Pattern

**Before:** Show all 2^n inputs (5 visible + rest faded ghost text)
**After:** Show first 3 inputs + vertical ellipsis "⋮" + summary "+N input lainnya"

| Before (n=4) | After (n=4) |
|---------------|-------------|
| 16 rows (5 visible + 11 faded) | 3 rows + ⋮ + 1 summary = ~5 rows |

Removed: unused import `ClassicalResult`

### 3. OraclePanel.tsx — Dynamic Height

**Before:** `h-[180px] overflow-y-auto` (hardcoded, causes scroll/waste)
**After:** `space-y-1` (auto height, grows with content)

- DJ-02 (2 queries): was 180px with dead space → now ~80px
- DJ-03 (9 queries): was scrollable → now fully visible ~280px

### 4. ResultPanel.tsx — No More "?" Boxes

**Before:** Shows actual results + "?" placeholder boxes (25-78% dead space)
**After:** Shows ONLY actual results + summary line

| Case | Before | After |
|------|--------|-------|
| DJ-01 CONSTANT | 5 blue + 3 "?" = 8 rows | 5 blue + summary |
| DJ-02 BALANCED | 1 orange + 1 blue + 5 "?" = 7 rows | 1 orange + 1 blue + summary |
| DJ-03 CONSTANT | 9 blue + 3 "?" = 12 rows | 9 blue + summary |
| DJ-04 BALANCED | 1 orange + 2 blue + 11 "?" = 14 rows | 1 orange + 2 blue + summary |

Added: counter badge "steps/totalInputs" in header
Added: informative summary line ("5 query → semua sama → CONSTANT")

### 5. TypeScript Fixes

- Removed unused `ClassicalResult` import from InputsPanel
- Removed unused `useRef` from App.tsx
- Removed unused `useEffect` from useClassicalDJ.ts

## Logic / Pseudocode

### InputsPanel (compact ellipsis)
```
showCount = min(3, totalSteps)
remaining = totalInputs - showCount

render first showCount inputs with blue styling
if remaining > 0:
    render vertical ellipsis "⋮"
    render "+{remaining} input lainnya"
```

### ResultPanel (no "?" boxes)
```
isConstant:
    render all steps as blue boxes (actual data only)
    summary: "{count} query → semua sama → CONSTANT"

isBalanced:
    render diffStep as orange box
    render checked steps as blue boxes (max 4)
    summary: "Beda di query ke-{n} → BALANCED"
```

### Layout (items-start)
```
flex-row items-start (NOT items-center)
    INPUTS (160px, auto height, compact)
    → ARROW (40px)
    ORACLE (200px, auto height, no scroll)
    → ARROW (40px)
    HASIL (180px, auto height, no "?" boxes)
```
