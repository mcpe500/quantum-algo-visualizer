# SPEC 025: Frontend Quality Refactor Phase 1 (High-Impact, Low-Risk)

## Prompt

User meminta refactor frontend secara serius, terstruktur, dan tidak destruktif:

1. Tingkatkan kualitas visual, arsitektur, maintainability, dan consistency.
2. Kurangi code smell, magic numbers, dan ad-hoc inline values.
3. Perbaiki batas tanggung jawab komponen dan naming.
4. Pertahankan perilaku yang sudah berjalan.
5. Kerjakan bertahap, jangan blind rewrite.

Batch execution yang dipilih:

- **Stabil core, split ringan DJ animation**.
- Artinya batch ini fokus pada shared frontend architecture + konsistensi page/tab, lalu extraction kecil di DJ animation tanpa mengubah logika 3D inti.

## Tujuan

1. Membentuk fondasi UI shared yang konsisten untuk page, section, state, dan dataset link.
2. Menghilangkan duplikasi berulang pada page-level error/loading/empty states.
3. Menyatukan konfigurasi global yang berulang (shots default, capture id, html2canvas options).
4. Mengurangi magic numbers pada chart components yang sering dipakai lintas algoritma.
5. Menjaga semua fitur existing tetap jalan dan lolos lint/build.

## Kenapa Ini Lebih Baik

1. **Consistency naik:** page-state visual seragam lintas algoritma.
2. **Maintainability naik:** perubahan style/state behavior cukup di constants/shared components.
3. **Readability naik:** tab components tidak lagi penuh wrapper class duplikat.
4. **Reusability naik:** `SectionCard`, `PageState`, `DatasetLink` bisa dipakai di halaman lain.
5. **Magic numbers turun:** chart sizing/padding kini bernama.
6. **Behavior tetap:** seluruh endpoint, flow run benchmark, tab switching, export, dan rendering utama tetap dipertahankan.

## Dampak Arsitektur

1. Frontend sekarang punya boundary yang lebih jelas antara:
   - shared UI state/layout primitives,
   - algorithm-specific content,
   - constants/config global.
2. Route pages berfungsi lebih sebagai orchestrator ringan.
3. Struktur lebih siap untuk phase berikutnya (split lanjut DJ animation) tanpa perlu mengganti fondasi lagi.

## Out of Scope (Sengaja Ditunda)

1. Split penuh `DJQuantumAnimation` ke banyak modul rendering/export.
2. Perombakan visual total dataset topography pages.
3. Bundle/code-splitting optimization besar (dynamic import granular).

Item di atas ditunda untuk menjaga risiko regression tetap rendah pada batch pertama.
