# SPEC 031: Video Export Stabilization

## Status
Implemented in the shared animation export hook.

## Scope
This specification stabilizes browser-side WebM recording and MP4 conversion for the DJ, QFT, and QAOA animation pages. The implementation keeps a single shared export pipeline, captures a 1920x1080 compositor canvas at fixed FPS, restores animation state after export, and adds a conversion fallback when ffmpeg/WASM conversion fails.

## Implementation
- `frontend/src/shared/hooks/useVideoExport.ts` restores step, speed, playing state, renderer canvas reference, animation frame handle, and MediaStream tracks after each export attempt.
- MP4 conversion errors now download a WebM fallback artifact instead of losing the recording.
- `frontend/src/utils/videoConvert.ts` retains timeout protection for ffmpeg core loading and conversion execution.

## Verification
Static audit confirms the fallback export path exists and the boxed animation engine verification passes for DJ, QFT, and QAOA.
