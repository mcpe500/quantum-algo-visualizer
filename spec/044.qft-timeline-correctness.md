# SPEC-044: QFT Timeline Correctness and Dataset-Driven Bloch Visualization

## Prompt
Perbaiki visualisasi QFT yang tidak sesuai permintaan pengguna. Target utama:
1. Timeline animasi QFT harus identik dengan evolusi Qiskit yang dipakai pada proposal dan benchmark.
2. Nilai visual qubit pada Bloch sphere harus berasal dari state riil setiap langkah, bukan pseudo-reconstruction dari label frontend.
3. Implementasi harus tetap dinamis berdasarkan dataset QFT yang tersedia.
4. Validasi harus membuktikan kecocokan untuk kasus dataset QFT aktif.

## Tujuan
- Menghilangkan mismatch antara animasi QFT dan sirkuit Qiskit referensi.
- Menjamin probabilitas, statevector, dan visual Bloch qubit konsisten terhadap dataset.
- Menyelaraskan urutan narasi UI dengan urutan operasi QFT yang benar: amplitude encoding -> controlled phase -> hadamard -> swap -> measurement.
- Menambahkan validator backend agar koreksi dapat diuji ulang tanpa inspeksi manual.

## Codebase Context
- Backend QFT: `backend/services/qft_service.py`
- Frontend QFT scene: `frontend/src/components/qft/animation/scene-primitives.tsx`
- Frontend phase narration: `frontend/src/components/qft/animation/phase-cascade.ts`, `frontend/src/components/qft/animation/narration.ts`
- Shared 3D qubit node: `frontend/src/shared/components/3d/gates/BlochQubitNode.tsx`
- Dataset-driven cases aktif: `QFT-01`, `QFT-02`

## Masalah yang Ditemukan
Implementasi timeline lama di backend menggunakan mutasi manual `phase_cascade` yang tidak ekuivalen dengan controlled-phase Qiskit. Dampaknya, statevector animasi menyimpang dari referensi bahkan sebelum tahap akhir. Frontend juga masih membangun pseudo-Bloch state dari label dan `p_zero`, sehingga orb QFT dapat terlihat meyakinkan tetapi tidak benar secara fisika kuantum.

## Perubahan yang Dilakukan

### Phase 1 - Backend Timeline Diselaraskan dengan Qiskit
`build_qft_animation_timeline()` diubah agar setiap langkah berasal dari evolusi `Statevector` menggunakan `QuantumCircuit` Qiskit:
- `cp(angle, control, target)` untuk semua phase cascade
- `h(qubit)` setelah semua rotasi terkontrol pada qubit target selesai
- `swap(i, n - 1 - i)` untuk koreksi bit-reversal

Langkah awal juga diubah menjadi amplitude encoding yang langsung memuat sinyal dataset yang telah dipadding dan dinormalisasi.

### Phase 2 - Ringkasan Qubit Riil untuk Frontend
Backend sekarang menurunkan `qubit_summaries` dari reduced density matrix setiap qubit. Ringkasan ini memuat:
- `p_zero`, `p_one`
- komponen Bloch `bx`, `by`, `bz`
- `radius` dan `coherence`
- `phase`
- `body_color`
- `label`

Dengan kontrak ini, frontend tidak perlu lagi menebak orientasi Bloch dari label heuristik.

### Phase 3 - Frontend QFT Menjadi Data-Driven
Frontend QFT diubah agar langsung memakai `activeStep.qubit_summaries`:
- `scene-primitives.tsx` tidak lagi merekonstruksi pseudo-Bloch state dari probabilitas semata.
- `BlochQubitNode.tsx` mendukung `bodyColor` dan `coherence` agar visual shell tetap mengikuti preferensi gaya DJ tetapi isi orb membawa informasi fase QFT.
- `QFTAnimationStep` ditambah field `qubit_summaries`, dan summary type diperluas agar kompatibel dengan payload backend.

### Phase 4 - Narasi dan Urutan Substep Diperbaiki
Urutan penjelasan UI diselaraskan dengan urutan operasi yang benar:
- amplitude encoding sebagai state awal, bukan `|0...0>`
- controlled-phase terjadi sebelum hadamard
- hadamard dijelaskan sebagai pengubah akumulasi fase menjadi interferensi Fourier
