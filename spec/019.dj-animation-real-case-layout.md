# SPEC 019: DJ Animation Real-Case Timeline Layout

## Prompt

Perbaiki animasi Deutsch-Jozsa Three.js karena susunan circuit masih terasa kurang. User meminta animasi yang lebih manusiawi, lebih detail, dan benar-benar mengikuti dataset JSON asli. Visual oracle tidak boleh sekadar template hardcoded. Font juga diminta sedikit lebih besar, tetapi tetap rapi, terbaca, dan tidak overlap.

Prompt inti yang dipahami:

1. Animasi harus memakai data asli dari dataset JSON.
2. Susunan circuit harus real case, bukan scaffold tetap.
3. Visual perlu lebih banyak detail agar kualitas animasi naik.
4. Tetap gunakan Three.js.
5. Typography dibesarkan sedikit, tetapi layout harus tetap aman.

## Tujuan

Mengubah animasi DJ dari scene bertipe template-phase menjadi scene bertipe timeline sirkuit nyata, sehingga:

1. setiap kolom visual berasal dari trace aktual,
2. oracle terlihat mengikuti case yang dipilih,
3. dataset JSON benar-benar ikut dirender sebagai bagian narasi visual,
4. ukuran teks naik secukupnya tanpa merusak keterbacaan.

## Kenapa Perlu Diubah

Masalah utama implementasi sebelumnya bukan pada warna atau efek, tetapi pada struktur visual. Walaupun backend sudah memakai `truth_table` asli untuk membangun circuit dan menjalankan simulator, scene Three.js masih menggambar pola tetap: `X -> H -> U_f -> H -> M`. Akibatnya DJ-01 dan DJ-04 masih terasa memakai kerangka yang sama, padahal kompleksitas oracle keduanya berbeda jauh.

Masalah kedua adalah sumber kebenaran animasi terpecah. Komponen lama membaca `trace` dan `snapshots` secara terpisah, lalu menyelaraskan keduanya dengan `currentStep` yang sama. Pendekatan ini rawan mismatch karena panjang timeline dan makna tiap langkah tidak benar-benar identik. Hasilnya, detail visual bisa terasa artifisial walaupun datanya berasal dari backend.

Masalah ketiga adalah dataset JSON hanya tampil sebagai token contoh yang dipotong. Untuk permintaan user, pendekatan tersebut belum cukup karena belum menunjukkan bahwa oracle benar-benar dibentuk dari seluruh isi dataset.

## Perubahan Logika

### 1. Satu sumber kebenaran animasi

Payload `/api/dj/animation/<case_id>` sekarang menjadi sumber utama animasi. Backend tidak lagi hanya mengirim snapshot generik, tetapi juga mengirim:

1. `timeline` per langkah,
2. `partitions` per phase,
3. `oracle_summary`,
4. `focus_input_bits` untuk langkah oracle tertentu.

Dengan kontrak ini, frontend tidak perlu lagi menebak-nebak posisi gate dari phase template.

### 2. Trace untuk constant case diperbaiki

`build_trace_by_layer()` sebelumnya hanya merinci oracle untuk balanced case. Sekarang constant case juga punya kolom oracle nyata:

1. `Oracle I` untuk constant-zero,
2. `Oracle X(anc)` untuk constant-one.

Perubahan ini penting agar circuit layout tetap valid bila nanti dataset baru berisi constant-one case.

### 3. Timeline animasi dibentuk dari trace aktual

Backend kini mereplay setiap stage trace ke circuit tanpa measurement, lalu mengambil `Statevector` sesudah setiap langkah. Dengan begitu:

1. panjang timeline sama dengan jumlah kolom trace,
2. probabilitas qubit selaras dengan langkah visual yang aktif,
3. phase partition dan susunan oracle mengikuti case nyata.

### 4. Layout Three.js kini kolom-dinamis

Frontend tidak lagi memakai posisi gate tetap. Posisi X setiap kolom dihitung dari jumlah langkah aktual `timeline.length`. Phase band, gate glyph, MCX control line, orb position, dan result board mengikuti layout dinamis ini.

### 5. Dataset JSON ditampilkan penuh

Token oracle 3D yang sebelumnya hanya mengambil `slice(0, 8)` diganti dengan panel dataset HTML yang menampilkan seluruh `truth_table`. Untuk langkah oracle tertentu, row aktif pada dataset juga ikut di-highlight melalui `focus_input_bits`.

### 6. Typography dibesarkan dengan layout baru

Font HTML dan label penting diperbesar sedikit. Agar tetap aman:

1. detail stage dipindah ke panel HTML,
2. canvas dibersihkan dari overlay berlebih,
3. tinggi canvas dibuat lebih longgar untuk case padat,
4. label 3D dibatasi pada elemen penting saja.

## Pseudocode Implementasi

```text
backend:
  load case dari dataset JSON
  build trace dari truth_table aktual
  partition trace by phase
  replay setiap stage ke circuit tanpa measurement
  ambil statevector setelah tiap stage
  kirim timeline + partition + oracle summary + measurement

frontend:
  load animation payload
  hitung posisi X berdasarkan panjang timeline
  render phase band dari partitions
  render marker kolom dari wire_markers aktual
  render orb state dari probabilities langkah aktif
  render truth table penuh dari JSON
  highlight row truth table jika focus_input_bits ada
  render measurement dan amplitude summary di panel detail
```
