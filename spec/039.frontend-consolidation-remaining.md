# SPEC 039: Remaining Frontend Consolidation

## Prompt

> "untuk kita mulai refactor codebase"

## Tujuan

Menyelesaikan konsolidasi frontend yang tersisa dari SPEC 037. Mengekstrak shared panel components, migrasi halaman DJ dan VQE ke pola AlgorithmPageShell, dan membersihkan dead code.

## Konteks

SPEC 037 (Phases 1-4) membuat shared infrastructure:
- `shared/utils/animation-helpers.ts`
- `shared/components/PhaseStepper.tsx` + `DetailCard.tsx`
- `shared/hooks/useAlgorithmBenchmark.ts`
- `shared/components/AlgorithmPageShell.tsx`

Tetapi:
- DJCombinedPage (301 baris) masih pakai inline state + manual layout
- VQECombinedPage pakai layout lama (AlgorithmPageLayout + ClassicQuantumTabs)
- ReadingGuideCard diduplikasi 3x di DJ/QFT/QAOA
- ProbabilityBarList diduplikasi di DJ MeasurementPanel + QFT FrequencySpectrumPanel
- QFT helpers.ts hanya 7 baris pure re-exports
- Empty `components/shared/` directory
- Canvas utils (roundedRect, wrapText) hanya di DJ helpers

## Catatan

1. **StepDetailList NOT extracted:** Row content terlalu berbeda antara DJ OracleConstruction dan QFT PhaseCascade. Hanya CSS pattern yang sama — insufficient justification untuk abstraction.

2. **QAOA formatPercent/formatRadians NOT changed:** QAOA's `formatPercent` selalu `toFixed(1)` dan `formatRadians` outputs radians (bukan degrees). Ini intentional domain-specific behavior.

3. **Factory pattern NOT applied:** useQFT/useQAOA/useVQE tetap sebagai standalone hooks. Factory (useAlgorithmBenchmark) hanya dijadikan opsi untuk future hooks. Overhead migrasi > benefit saat ini.

4. **DJ isVideoExporting handling:** AlgorithmPageShell tidak punya `disabled` prop. Solusi: CSS wrapper dengan `pointer-events-none opacity-90` saat video exporting. Simple, no shell modification needed.

5. **VQE captureId changed:** Dari `CAPTURE_IDS.vqe` ke `"vqe-combined"`. Harus verifikasi CAPTURE_IDS constant masih konsisten.
