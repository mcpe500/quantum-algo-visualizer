# SPEC 033: Circuit Lab Drag-and-Drop Tab

## Prompt

> "saya tujuannya sekarang ada baru lagi jadi bisa main cirquit"
> "cirquit nya bisa diatur atur jadi saya bisa drag and drop"
> "ini tapi di tab baru saja"
> "qubit nya bisa divisualisasi dengan drag and drop gate nya"
> "bisa keliatan dalemnya"
> "misalkan saya nyusun gimana nanti bisa otomatis tau phases kickbacknya atau hal hal lain"

## Tujuan

Menambahkan mode baru di halaman `/playground` berupa **Circuit Lab**. Mode ini memungkinkan user menyusun sirkuit kuantum dengan drag-and-drop, melihat preview Bloch/statevector secara live, membaca trace langkah demi langkah, dan memperoleh insight otomatis seperti superposition, phase rotation, entanglement, serta phase kickback heuristic.

## Kenapa

Mode sebelumnya cocok untuk manipulasi state secara langsung per gate. User sekarang membutuhkan cara berpikir yang lebih dekat ke sirkuit kuantum asli: gate ditempatkan di kolom dan row, lalu sistem mereplay rangkaian dari kiri ke kanan. Karena itu solusi yang tepat bukan route baru, tetapi **tab baru** di halaman yang sama agar eksperimen state-level dan circuit-level tetap berdekatan.

## Scope

- Route tetap: `/playground`
- Tab baru: `Circuit Lab`
- Tab lama tetap ada: `State Lab`
- Tidak ada backend baru
- Tidak ada dependency drag-and-drop baru, cukup HTML5 drag-and-drop

## Rule Penting

### Penempatan gate

- Gate 1-qubit menempati satu row
- Gate 2-qubit menempati row yang dipilih dan row tepat di bawahnya
- Jika drop gate 2-qubit di row terakhir, placement ditolak
- Placement baru membersihkan konflik pada kolom dan row yang sama

### Replay

- Semua placement diurutkan berdasarkan `column`, lalu `row`
- Replay selalu dimulai dari `|00...0⟩`
- Matrix embedding memakai engine `embedGate1Q` / `embedGate2Q` yang sama dengan mode State Lab

## Insight Heuristics

### Superposition

Jika ada qubit dengan `p(|0⟩)` dan `p(|1⟩)` sama-sama signifikan, insight superposition ditampilkan.

### Phase Rotation

Jika rangkaian memuat `Z`, `S`, `T`, `Rz`, atau `CPhase`, insight phase rotation ditampilkan.

### Entanglement / Correlation

- Untuk 2 qubit: dipakai uji eksak state murni `|ad - bc| > ε`
- Untuk 3 qubit: dipakai heuristik penyusutan norm vektor Bloch setelah interaksi dua qubit

### Phase Kickback

Untuk `CNOT` atau `CPhase`, sistem memeriksa apakah:

- control qubit sudah berada di superposition
- target qubit mendekati `|-⟩`

Jika iya, insight `phase kickback` ditampilkan.

## Manual Testing Plan

1. Buka `/playground`
2. Pastikan tab `State Lab` dan `Circuit Lab` muncul
3. Masuk `Circuit Lab`
4. Drag `H` ke `q0`, kolom 1 → preview harus berubah menjadi superposition
5. Drag `CNOT` ke `q0`, kolom 2 pada mode 2 qubit → preview harus menunjukkan korelasi/entanglement jika sesuai
6. Klik gate parametrik (`Rz` / `CPhase`) lalu ubah angle di inspector → preview dan trace harus ikut berubah
7. Drag gate yang sudah terpasang ke kolom lain → gate harus berpindah
8. Klik gate → inspector harus menampilkan detail gate tersebut
9. Klik `Remove selected gate` → gate hilang dari grid
10. Klik `Clear circuit` → semua placement terhapus dan state kembali ke `|00...0⟩`
11. Gunakan 3 qubit lalu drop gate 2-qubit di row terakhir → placement harus ditolak

## Catatan

1. Implementasi drag-and-drop memakai HTML5 native API, jadi tidak menambah library baru.
2. Heuristic insight sengaja ringan dan edukatif, bukan symbolic proof engine.
3. Gate 2-qubit saat ini dibatasi pada pasangan row bersebelahan (`row` dan `row + 1`) agar UI drag-and-drop tetap sederhana dan stabil.
