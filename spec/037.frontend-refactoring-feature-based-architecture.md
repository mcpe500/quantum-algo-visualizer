## Tujuan

Merefaktor frontend React dari struktur flat-hierarchical menjadi feature-based architecture yang rapi, scalable, dan maintainable. Fokus pada struktur code, reusability, separation of concerns, dan naming clarity tanpa menambah auth, login, atau fitur baru.

## Scope

### Dalam Scope

- Restrukturir folder `frontend/src` ke approach feature-based
- Extract animation helpers yang duplicated ke shared
- Extract/generic-ize `PhaseStepper` dan `DetailCard` components
- Buat `useAlgorithmBenchmark` hook factory
- Buat `AlgorithmPageShell` component

### Di luar Scope

- TIDAK menambah auth, login, register, session, token
- TIDAK mengubah business logic behaviour
- TIDAK menambah library baru
- TIDAK merefactor backend
- TIDAK membuat fitur baru
- CircuitLab decomposition (Phase 5, 6, 7) dibatalkan karena terlalu kompleks dan tidak ada ROI signifikan untuk research TA

## Prinsip Refactor

1. **Don't break behavior** - Refactor hanya structural, behavior dipertahankan
2. **Don't over-abstract** - Shared hanya untuk yang benar-benar generic dan reusable
3. **Feature owns its domain** - Components, hooks, types stay close to domain
4. **Single Responsibility** - Pecah file besar jadi component focused
5. **Consistent naming** - Nama harus jelas dan konsisten

## Langkah Refactor

### Phase 1: Extract Shared Animation Utilities

**Alasan:** Pure functions, zero domain dependency, highest ROI dengan 200+ lines duplikat dihapus.

**File baru:**
- `frontend/src/shared/utils/animation-helpers.ts`

**Fungsi yang di-extract:**
- `clamp()`, `wait()`, `waitForAnimationFrame()`, `waitForAnimationFrames()`
- `waitForCanvasReady()`, `formatPercent()`, `getLaneYs()`, `getColumnLayout()`
- `formatRadians()`, `formatComplex()`, `radiansToDegrees()`, `degreesToRadians()`, `normalizeAngle()`

**Update:**
- `frontend/src/components/dj/animation/helpers.ts` в†’ re-export dari shared + local overrides
- `frontend/src/components/qft/animation/helpers.ts` в†’ re-export dari shared + local overrides
- `frontend/src/components/qaoa/animation/helpers.ts` в†’ re-export dari shared + keep QAOA-specific

### Phase 2: Extract Shared Animation Components

**Alasan:** PhaseStepper dan DetailCard hampir identik di dj, qft, qaoa. Variasi hanya di warna dan sedikit prop.

**File baru:**
- `frontend/src/shared/components/PhaseStepper.tsx` - generic dengan color prop + startKey prop
- `frontend/src/shared/components/DetailCard.tsx` - single identical implementation

**Update:**
- `frontend/src/components/dj/animation/panels.tsx` в†’ re-export PhaseStepper, DetailCard dari shared

### Phase 3: Create Shared Hook Factory

**Alasan:** Pattern `selectedCaseId, availableCases, benchmarkResult, circuitImage, trace, handleRun, handleDownload` identik di semua algorithm pages.

**File baru:**
- `frontend/src/shared/hooks/useAlgorithmBenchmark.ts` - hook factory dengan generic types

**Interface:**
```typescript
interface AlgorithmBenchmarkConfig {
  algorithmId: string;
  defaultCaseId: string;

```

**Usage:**
```typescript

```

## Catatan

1. **Backward Compatibility:** API endpoint dan response format TIDAK berubah. Hanya internal structure yang di-refactor.

2. **Incremental Migration:** Tiap phase bisa dieksekusi independently tanpa break phase lain.

3. **No Over-Engineering:** Jika satu component hanya dipakai di satu tempat, biarkan di feature. Shared hanya jika proven reusable.

4. **Maintain Behavior:** Setiap refactor step harus menghasilkan output yang functionally identical dengan input.

5. **Phase 5-7 Dibatalkan:** CircuitLab decomposition (Phase 5, 6, 7) dibatalkan karena:
   - Terlalu kompleks dengan 962-line component
   - Tidak ada ROI signifikan untuk research TA
   - Original files tetap utuh dan berfungsi

6. **Dukungan PhaseStepper untuk DJ/QFT:** Shared PhaseStepper sekarang mendukung `startKey` prop untuk menangani perbedaan `start_col` (DJ) vs `start` (QFT) pada partition interface.

7. **Dead Code Restoration:** Files yang awalnya ditandai untuk dihapus (GateButtons.tsx, StateInfoPanel.tsx, useQubitState.ts) dikembalikan karena QubitPlaygroundPage masih menggunakan State Lab tab yang membutuhkan component tersebut.
