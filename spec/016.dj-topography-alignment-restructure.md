# SPEC 016: DJ Topography Alignment Restructure

## Prompt

User feedback: konten sudah benar, tetapi susunan visual halaman topography terasa tidak nyaman dilihat. Fokus perbaikan diminta pada alignment/layout, bukan warna.

## Tujuan

Merapikan ritme layout agar pembacaan visual stabil:

1. Menghilangkan efek "konten melayang" pada kartu 3-qubit.
2. Menghindari campuran kepadatan 8-state dan 16-state dalam satu baris yang sama.
3. Menjaga style warna/branding tetap sama.

## Akar Masalah

1. Body matriks sebelumnya memakai `items-center`, membuat grid kecil berada di tengah vertikal dengan dead space besar.
2. Semua kartu dipaksa dalam satu grid 4 kolom sehingga kartu 3-qubit dan 4-qubit bertabrakan ritme kepadatan.

## Perubahan Implementasi

File: `frontend/src/components/dj/OracleTopography.tsx`

1. **Top alignment untuk area matriks**
   - `items-center` → `items-start`
   - tambah `content-start` pada grid

2. **Grouping layout by qubit count**
   - Cases dikelompokkan berdasarkan `n_qubits`.
   - Render menjadi 2 section:
     - `3 Qubit Oracle Cases` (DJ-01, DJ-02)
     - `4 Qubit Oracle Cases` (DJ-03, DJ-04)
   - Tiap section memakai grid 2 kolom (`xl:grid-cols-2`) agar ritme antar kartu setara.

3. **Numeric alignment refinement**
   - Tambah `tabular-nums` pada label input dan output untuk stabilitas kolom angka.

4. **Order stability**
   - Truth table tetap disortir biner asc.
   - Kasus per group disortir `case_id` asc.

## Verifikasi

- Build frontend: `npm run build` ✅ PASS
- Tidak ada perubahan kontrak API atau struktur dataset.

## Dampak

1. Kartu tidak lagi terasa jomplang secara vertikal.
2. Perbandingan CONSTANT vs BALANCED lebih enak dibaca per level qubit.
3. Komposisi keseluruhan lebih konsisten tanpa mengubah tema warna.
