# SPEC 054: VQE Hybrid Split View + Step Flow Visualization

## Prompt

Implementasi visualisasi alur hybrid VQE menggunakan pendekatan **Split Separator View (Classical | Quantum)** yang dikombinasikan dengan **Step Flow Diagram** 6 langkah. Visualisasi menunjukkan interaksi iteratif antara optimizer klasik dan sirkuit kuantum, dengan gambar sirkuit asli dari Qiskit renderer pada setiap checkpoint iterasi. Dataset VQE JSON tidak berubah.

## Purpose

Menyediakan representasi visual yang jelas dan akademis tentang sifat **hybrid** VQE: bagaimana domain klasikal (CPU) dan domain kuantum (QPU/simulator) berinteraksi dalam loop iteratif sampai konvergensi. Reviewer thesis dapat melihat evolusi sirkuit dan parameter secara real, bukan mockup.

## Why

1. VQE adalah algoritma **hybrid** — pemisahan classical vs quantum adalah konsep inti yang harus terlihat jelas.
2. Split View (dua kolom) menunjukkan state pada iterasi aktif. Step Flow (6 langkah dengan panah) menunjukkan alur proses.
3. Kombinasi keduanya memberikan pemahaman UTUH: "apa yang terjadi" (step flow) dan "apa yang sedang terjadi sekarang" (split view).
4. Proposal mensyaratkan eksperimen VQE yang lengkap termasuk visualisasi proses.
5. Split view lebih akademis dan edukatif untuk audience thesis.

## Design Decisions

### 1. Step Flow Diagram (6 Steps)

6 langkah VQE dengan **color coding** per domain:
- Steps ①, ⑤, ⑥ — **Classical (Blue)** — Initialize, Update Parameters, Check Convergence
- Steps ②, ③, ④ — **Quantum (Purple)** — Build Circuit, Prepare State, Measure Energy

**Layout desktop:** 2-row snake pattern
```
① Init θ⁽⁰⁾ → ② Build U(θ) → ③ Prepare |ψ(θ)⟩
⑥ Check Conv? ← ⑤ Update θ ← ④ Measure E(θ)
       ↓
   No → loop back to ②
   Yes → Done
```

**Layout mobile:** Vertical stack

**Fitur:**
- Arrows: horizontal (→, ←), vertical (↓), curved loop back
- Active step highlight based on `activeCheckpoint`
- "Done" badge untuk exit condition
- Unique SVG marker IDs via `useId()`

### 2. Split Separator View (Classical | Quantum)

Dua kolom dengan batas visual:
- **LEFT (Classical/CPU)**: Optimizer info, parameter vector, energy from quantum
- **RIGHT (Quantum/QPU)**: Circuit image (Qiskit render), state prep formula, Hamiltonian terms, energy measurement formula

**Boundary arrows:**
- θ→ (blue, Classical→Quantum)
- E← (purple, Quantum→Classical)

### 3. Checkpoint Timeline

5 checkpoint button row:
- Start, 25%, 50%, 75%, Final
- Active state styling (purple highlight)
- Shows iteration number and energy per checkpoint

### 4. Section Order (UX-Critique Based)

Urutan yang logis untuk thesis reviewer:
1. Metrics Grid (summary)
2. **Comparison: FCI vs VQE** (anchor hasil utama — dipindah ke atas)
3. Step Flow Diagram (konsep alur kerja)
4. Convergence Chart (bukti agregat)
5. Checkpoint Timeline (kontrol navigasi — sebelum detail)
6. Hybrid Split View + Energy Cards (detail per checkpoint)
7. Circuit Display (final circuit monumental)
8. Optimal Parameters
9. Shot Evaluation
10. Trace Table (technical, collapsible candidate)

### 5. Dataset Immutability

File `datasets/vqe/canonical/*.json` **tidak berubah**. Semua data snapshot dihasilkan runtime di `vqe_service.py`.

**Change 2: Checkpoint Selection**
```python
def _select_checkpoints(history, n=5):
    if not history: return []
    m = len(history)
    if m <= n: return list(range(m))
    return [min(m - 1, int(i * (m - 1) / (n - 1))) for i in range(n)]
```

**Change 3: Circuit Image Generation**
```python
def _generate_snapshot_circuit_image(n_qubits, ansatz_type, n_layers, theta_values):
    qc, params = build_ansatz_circuit(n_qubits, ansatz_type, n_layers)
    if len(theta_values) != len(params): return None
    bound = qc.assign_parameters({params[i]: float(theta_values[i]) for i in range(len(params))})
    fig = circuit_drawer(bound, output='mpl')
    try:
        return figure_to_base64(fig)
    finally:
        plt.close(fig)
```

## Pseudocode

### Backend — Snapshot Generation

```text
after VQE optimization:
  checkpoints = _select_checkpoints(vqe_result['history'])
  snapshots = []
  for idx in checkpoints:
    entry = history[idx]
    img = _generate_snapshot_circuit_image(n_qubits, ansatz_type, n_layers, entry['params'])
    if img:
      snapshots.append({iteration: idx, energy: entry['energy'], parameters: entry['params'], circuit_image: img})
  return snapshots
```

### Frontend — Section Order

```text
VQEQuantumTab:
  if !result: emptyState

  snapshots = result.quantum.iteration_snapshots
  activeCheckpoint = state(0)
  currentSnapshot = snapshots[safeActive]

```
