# SPEC 049: QAOA Adjacency Matrix JSON Schema Update

## Prompt

Ubah JSON dataset QAOA supaya berisi field `adjacency_matrix` secara langsung. Saat ini JSON hanya berisi `graph.nodes` dan `graph.edges`, lalu frontend menghitung matrix secara dinamis. Setelah perubahan, JSON harus menyediakan matrix secara langsung, dan semua komponen yang mengonsumsi data harus diupdate.

## Purpose

Menjadikan adjacency matrix sebagai **first-class data** dalam JSON dataset, bukan computed value di frontend. Ini memastikan:
1. Data matrix bisa di-serve langsung dari backend
2. Frontend tidak perlu recompute setiap render
3. Backend bisa memvalidasi konsistensi antara edges dan matrix
4. API response juga menyertakan matrix

## Why

1. User meminta JSON QAOA berubah sehingga adjacency matrix tersedia langsung
2. Semua handling yang bergantung pada JSON harus ikut berubah
3. Menghilangkan redundant computation di frontend
4. Data-driven: matrix didefinisikan di dataset, bukan dihitung ulang

## Changes Planned

### 1. Dataset JSON

Tambahkan field `adjacency_matrix` di dalam `graph`:

**QAOA-01 (K3 triangle):**
```json
"graph": {
  "nodes": [0, 1, 2],
  "edges": [[0, 1], [0, 2], [1, 2]],
  "adjacency_matrix": [
    [0, 1, 1],
    [1, 0, 1],
    [1, 1, 0]
  ]
}
```

**QAOA-02 (K4 complete):**
```json
"graph": {
  "nodes": [0, 1, 2, 3],
  "edges": [[0, 1], [0, 2], [0, 3], [1, 2], [1, 3], [2, 3]],
  "adjacency_matrix": [
    [0, 1, 1, 1],
    [1, 0, 1, 1],
    [1, 1, 0, 1],
    [1, 1, 1, 0]
  ]
}
```

### 2. TypeScript Types

Update 3 interfaces:

- `QAOAGraph`: Tambah `adjacency_matrix: number[][]`
- `QAOABenchmarkResult`: Tambah `adjacency_matrix: number[][]`
- `QAOAAnimationPayload`: Tambah `adjacency_matrix: number[][]`

### 3. AdjacencyMatrixGrid Component

- Tambah prop opsional `adjacencyMatrix?: number[][]`
- Logika: `const matrix = adjacencyMatrix ?? buildAdjacencyMatrix(nodes, edges)`
- Fallback ke computation jika matrix tidak tersedia (backward compatible)

### 4. QAOADatasetVisualizer

- Pass `data.graph.adjacency_matrix` ke `AdjacencyMatrixGrid`

### 5. Backend qaoa_service.py

**_normalize_problem:**
- Baca `graph.get('adjacency_matrix')` dari JSON
- Validasi: pastikan NxN, semua baris length sama
- Jika tidak valid atau tidak ada, compute dari edges (fallback)
- Sertakan `adjacency_matrix` dalam return dict

**Payload functions:**
- `run_qaoa_payload`: Tambah `'adjacency_matrix': problem['adjacency_matrix']`
- `get_qaoa_animation_payload`: Tambah `'adjacency_matrix': problem['adjacency_matrix']`
- `get_qaoa_classical_payload`: Tambah `'adjacency_matrix': problem['adjacency_matrix']`

## Logic / Pseudocode

```text
_normalize_problem(case):
  graph = case['graph']
  raw_matrix = graph.get('adjacency_matrix')
  
  if raw_matrix exists and is valid NxN:
    adjacency_matrix = raw_matrix  # Use from JSON
  else:
    # Compute from edges
    adjacency_matrix = NxN zeros
    for [u, v] in edges:
      matrix[u][v] = 1
      matrix[v][u] = 1
  
  return {
    ...existing_fields,
    'adjacency_matrix': adjacency_matrix,
  }

AdjacencyMatrixGrid:
  if adjacencyMatrix prop provided:
    use it directly
  else:
    compute from nodes + edges (fallback)
```
