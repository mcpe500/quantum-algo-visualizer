## ISSUE 2: QAOA HybridFlowPanel Redesign

### Problem

Current `HybridLoopPanel` shows OUTPUT (parameter values, cut scores) but NOT the PROCESS:
- No visualization of optimizer → circuit → evaluation → update cycle
- No pseudocode showing step-by-step
- No arrows showing data flow
- "No arrows" was wrong diagnosis — the problem is conceptual

### Solution: HybridFlowPanel with Pseudocode + Flow Diagram

```
┌──────────────────────────────────────────────────────────────┐
│  HYBRID LOOP — QAOA                                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│   │   CLASSICAL   │◄──►│   QUANTUM    │◄──►│   MEASURE    │  │
│   │   (COBYLA)   │    │   (Circuit)  │    │   (Shots)    │  │
│   └──────────────┘    └──────────────┘    └──────────────┘  │
│                                                              │
│   PSEUDOCODE:                                                │
│   ──────────────────────────────────────────────────────     │
│   1. Initialize γ₀, β₀ (random)                              │
│   2. → Build QAOA circuit with γ, β                         │  ← active
│   3. → Statevector simulation → ⟨H_C⟩ expected cut           │
│   4. ← COBYLA receives -⟨H_C⟩, updates γ, β                 │
│   5. → Repeat until convergence                              │
│   6. → Final: AerSimulator(shots=1024) → bitstring          │
│                                                              │
│   PARAMETERS: γ=[0.5, 1.2] β=[0.8, 0.3]                     │
│   BEST CUT: 12.5                                            │
└──────────────────────────────────────────────────────────────┘
```

### Phase-to-Line Mapping

### HybridFlowPanel Features

1. **Flow Diagram**: 3 boxes (Classical/Quantum/Measure) with bidirectional arrows
   - Active box highlighted with algorithm color
   - Feedback arrow (dashed) from Measure back to Classical

2. **Pseudocode Block**: 6 numbered lines
   - Line 1: Initialize parameters
   - Lines 2-3: Quantum circuit + evaluation
   - Line 4: Classical optimizer update
   - Line 5: Repeat
   - Line 6: Final measurement
   - Active line highlighted in blue based on current phase

3. **Phase-to-Line Mapping**:
   | activePhase | Active Line |
   |-------------|-------------|
   | `optimizer` | 4 (COBYLA update) |
   | `superposition` | 2 (Build circuit) |
   | `cost` | 2 (Build circuit) |
   | `mixer` | 2 (Build circuit) |
   | `measurement` | 6 (Final shots) |
   | `update` | 4 (COBYLA update) |

4. **Parameters Display**: Shows γ and β values for active checkpoint
