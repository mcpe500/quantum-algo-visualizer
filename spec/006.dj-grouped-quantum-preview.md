# SPEC 006: DJ Phase-Boxed Circuit Image

## Prompt

Tambahkan phase boxes pada gambar circuit DJ menggunakan matplotlib. Box harus mengelompokkan circuit berdasarkan phase trace `init / prep / oracle / interference / measure`, dan tetap dinamis mengikuti JSON case aktif.

## Purpose

Menampilkan circuit DJ dengan overlay kotak phase yang memperjelas alur algoritma. Box ditambahkan langsung pada matplotlib image, bukan sebagai komponen frontend terpisah.

## Why

1. Gambar circuit yang ada sudah benar, jadi box harus ditambahkan di atasnya.
2. Box harus dinamis - mengikuti struktur circuit dari JSON case aktif.
3. Jika dataset bertambah (`DJ-05`, `DJ-06`, dll), box tetap mengikuti circuit structure.
4. Box harus muncul di live preview dan PNG download (karena baked into image).

## Changes Made

### 1. Backend phase box drawing

Added `_add_phase_boxes_to_figure(fig, qc)` helper function:

Behavior:
- Analyzes `qc.data` to find barrier positions
- Splits circuit into 5 phases: init, prep, oracle, interference, measure
- Calculates proportional x-spans for each phase
- Draws colored Rectangle patches on matplotlib axes
- Adds phase labels with number badges at top

Phase detection:
- `init`: first gate (X on ancilla)
- `prep`: gates before first barrier, after init
- `oracle`: gates between barriers
- `interference`: H gates after second barrier
- `measure`: measure gates at end

### 2. New boxed circuit endpoint

Added `/api/dj/circuit-image-boxed/<case_id>`:
- Same as regular circuit-image but calls `_add_phase_boxes_to_figure`
- Returns PNG with phase boxes baked in

### 3. Frontend boxed image display

`DJCombinedPage.tsx` changes:
- Loads boxed circuit image via `getCircuitImageBoxed()`
- Displays boxed image below regular quantum solution
- Regular circuit image stays unchanged (no boxes)
- Both images inside `capture-area` for PNG export

## Logic / Pseudocode

### Phase box drawing (backend)

```python
def _add_phase_boxes_to_figure(fig, qc):
    ax = fig.axes[0]
    x_min, x_max = ax.get_xlim()
    y_min, y_max = ax.get_ylim()

    # Find barriers in qc.data
    barrier_indices = [i for i, item in enumerate(qc.data) if item.operation.name == 'barrier']

    # Split into sections
    section0 = indices before barrier[0]
    section1 = indices between barriers
    section2 = indices after barrier[1]

    # Further split
    init = section0[0:1]
    prep = section0[1:]
    oracle = section1
    interference = [i for i in section2 if not measure]
    measure = [i for i in section2 if measure]

    # Calculate weights proportional to instruction count
    for phase in [init, prep, oracle, interference, measure]:
        weight = len(phase) or 0.5  # min weight for empty
        phase_width = (weight / total_weight) * total_x_width

        # Draw Rectangle patch
        rect = Rectangle((current_x, y_min), phase_width, height, ...)
        ax.add_patch(rect)

        # Draw phase label
        ax.text(center_x, top_y, f'{num}. {label}', ...)

    return fig
```

## Dynamic Behavior

The phase boxes are fully dynamic:

- **Gate count per phase**: derived from actual `qc.data`, not hardcoded
- **Box widths**: proportional to instruction count per phase
- **Colors**: fixed per phase (blue/init, green/prep, amber/oracle, violet/interference, red/measure)
- **Labels**: always show "1. Inisialisasi", "2. Persiapan", etc.
- **Oracle handling**: shows even if empty (constant-0 case), with minimum visual weight

When JSON case changes:
- Backend builds new circuit from `truth_table`
- `_add_phase_boxes_to_figure` recalculates based on new gate structure
- Box positions and widths adjust automatically
