# SPEC 006: DJ Grouped Quantum Preview

## Prompt

Tambahkan live preview kuantum berkelompok di bawah solusi kuantum DJ yang sudah ada. Preview baru harus berbentuk kotak-kotak grouped berdasarkan phase trace `init / prep / oracle / interference / measure`, dan tetap dinamis mengikuti JSON case aktif.

## Purpose

Menampilkan versi terkelompok dari solusi kuantum untuk memperjelas alur DJ tanpa mengubah solusi kuantum utama yang sudah ada. Preview ini dipakai untuk live view dan ikut masuk ke PNG download.

## Why

1. Solusi kuantum yang ada sudah benar, jadi tidak perlu diganti.
2. Dibutuhkan tampilan boxed/grouped agar alur phase trace lebih mudah dibaca.
3. Grouping harus mengikuti data case aktif, bukan hardcoded per case.
4. Jika dataset bertambah seperti `DJ-05`, `DJ-06`, sampai `DJ-10`, preview tetap mengikuti JSON aktif.

## Changes Made

### 1. New grouped preview component

Created `QuantumPhaseGroupedPreview.tsx` to render the quantum trace as grouped cards.

Behavior:
- reads `trace.partitions`
- slices `trace.stages` by phase
- renders cards for `init`, `prep`, `oracle`, `interference`, and `measure`
- shows stage steps, operations, input qubits, and ancilla state

### 2. Integrated below existing quantum solution

`DJCombinedPage.tsx` now renders the grouped preview below the existing `QuantumVisualization` card.

Rules:
- existing quantum solution stays unchanged
- grouped preview appears as a second live section
- both sections stay inside `capture-area`, so the PNG download includes them

### 3. Dynamic sizing

The preview does not depend on a fixed dataset count.

It adapts to:
- the selected JSON case
- `n_qubits`
- the current trace stages
- the returned phase partitions

## Logic / Pseudocode

### Grouped preview generation

```text
trace = GET /api/dj/trace/<case_id>
groups = []

for each partition in trace.partitions:
  stages = trace.stages.slice(partition.start, partition.end)
  if stages not empty:
    groups.push({ label, stages })

render each group as a boxed card
render each stage inside its group card
```

### Live page flow

```text
load selected case
load benchmark result
load circuit image
load quantum trace

render existing quantum solution
render grouped quantum preview under it
render trace table below
```
