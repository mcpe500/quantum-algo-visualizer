# SPEC 024: DJ Animation — Step Comments, Simpler Narrative, Better Value Display

## Prompt

User revisi:
1. "value setiap step kalau bisa di tampilkan lebih baik"
2. "Penjelasan juga lebih dibuat sederhana dan dapat dipahami manusia"
3. "Mungkin perlu dikomentari step2 nya. Singkat saja."

Constraint tambahan: **JANGAN HARD CODE** — semua harus berjalan dinamis untuk semua tipe dataset DJ.

## Tujuan

1. Setiap step animasi menampilkan **nilai qubit** (probabilitas P(|0⟩)) secara informatif.
2. Semua teks narasi ditulis ulang agar sederhana, langsung, bisa dipahami manusia tanpa latar belakang quantum.
3. Setiap step punya **comment singkat** (1 baris) yang ditampilkan di UI.
4. Perbaikan bug: teks Rusia "состояние" (2 lokasi), kata corrupt "1abelle", parameter salah `step.step` vs `totalSteps`.

## Constraint: Dynamic / No Hard-Coding

Semua perubahan berjalan dinamis:
- Qubit count: loops over `data.total_qubits` / `step.bloch_states.length`
- Step count: scales via `getColumnLayout(data.timeline.length)`
- Oracle patterns: dispatch on `kind` (exhaustive for DJ phase space), fallback untuk kind unknown
- Narratives: dispatch on `step.phase` + `step.operation`, keduanya dari backend
- Probabilities: computed from `blochState.bz` via formula `(1+bz)/2`
- Comments: generated from `kind` + `focus_bits` (dinamis), fallback untuk kind unknown

Dynamic trace proof:

| Dataset | Qubits | Steps | Oracle Pattern | Dynamic? |
|---------|--------|-------|----------------|----------|
| DJ-01 (3q, constant-zero) | 4 | 8 | oracle-identity ×1 | Yes |
| DJ-02 (3q, balanced) | 4 | ~18 | flip/mcx/restore ×4 | Yes |
| DJ-03 (3q, constant-one) | 4 | 8 | oracle-constant-one ×1 | Yes |
| DJ-04 (4q, balanced) | 5 | ~31 | flip/mcx/restore ×8 | Yes |

## Keputusan Teknis

### A. Backend: `_get_step_comment(kind, focus_bits)`

Fungsi baru di `dj_service.py`. Mapping `kind` → comment string. Dinamis karena:
- `kind` dihasilkan dari struktur sirkuit (bukan hard-coded per dataset)
- `focus_bits` di-inject ke comment untuk oracle steps
- Fallback generik untuk kind yang tidak dikenali

Comment table:
| kind | comment |
|------|---------|
| pre-init | Keadaan awal \|0⟩ |
| init-register | Ancilla → \|1⟩, input → superposisi |
| prep-ancilla | Ancilla → \|−⟩ untuk phase kickback |
| oracle-identity | Oracle constant: tidak ada operasi |
| oracle-constant-one | Oracle constant: flip ancilla |
| oracle-flip | Balik kontrol agar cocok |
| oracle-mcx | MCX: evaluasi f({focus_bits}) |
| oracle-restore | Pulihkan kontrol ke semula |
| interference | H akhir: fase → amplitudo |
| measure | Ukur input → baca klasifikasi |
