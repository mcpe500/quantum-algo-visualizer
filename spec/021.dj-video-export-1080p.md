# SPEC 021: DJ Video Export 1080p Resolution Fix

## Prompt

User meminta fitur `export as video` tetap ada, tetapi hasilnya harus punya resolusi yang benar karena kualitas video sebelumnya terlihat jelek. Default target yang dipilih adalah `1080p 16:9`.

## Tujuan

Memastikan export video DJ menghasilkan output yang benar-benar tajam dan stabil dengan karakteristik berikut:

1. resolusi final tetap `1920x1080`,
2. aspect ratio tetap `16:9`,
3. kualitas tidak lagi bergantung pada ukuran preview di halaman,
4. caption penjelasan tetap ikut masuk ke video.

## Masalah Implementasi Sebelumnya

Export video sesi sebelumnya masih mengambil source frame dari canvas preview yang tampil di UI. Secara visual ini memang berfungsi, tetapi kualitas output ikut tergantung pada ukuran layout halaman, yang biasanya hanya sekitar tinggi `520` sampai `560` piksel dan lebar dinamis dari grid layout.

Akibatnya, recorder tidak benar-benar bekerja dari surface video standar. Saat file diputar, hasilnya terlihat seperti preview yang dibesarkan, bukan render video yang memang dibuat untuk resolusi tinggi. Masalah utamanya bukan hanya bitrate, tetapi sumber render yang salah. Jika hanya menaikkan ukuran kompositor atau bitrate tanpa mengganti source canvas, hasil tetap akan blur karena sebenarnya yang direkam hanyalah preview kecil yang di-upscale.

## Keputusan Teknis

### 1. Pisahkan preview renderer dan export renderer

Preview canvas tetap dipakai untuk interaksi normal user. Untuk export, komponen sekarang membuat renderer kedua yang khusus dipakai saat export berjalan.

Renderer export ini:

1. dirender di luar viewport,
2. memiliki ukuran tetap `1920x1080`,
3. memakai `dpr={1}` agar ukuran drawing buffer tepat,
4. memiliki `preserveDrawingBuffer: true` agar aman untuk frame capture.

Dengan cara ini, source untuk recorder tidak lagi berasal dari preview halaman.

### 2. Recorder memakai source 1080p tetap

`handleExportVideo()` sekarang menunggu canvas export 1080p benar-benar siap sebelum recorder dimulai. Setelah siap, kompositor menggambar source dari canvas export, bukan dari canvas preview.

### 3. Bitrate dinaikkan untuk 1080p

Karena output kini benar-benar 1080p, bitrate export juga dinaikkan dari nilai sebelumnya menjadi `16 Mbps`. Nilai ini dipilih agar gate line, orb, dan teks overlay tetap tajam tanpa menambah kompleksitas preset kualitas pada sesi ini.

### 4. Latar gradient video dibuat di kompositor

Canvas preview sebelumnya mengandalkan background CSS pada elemen DOM. Untuk export, background dasar kini ikut digambar langsung di kompositor agar kualitas tampilan video konsisten dan tidak bergantung pada style DOM canvas.

## Perubahan Code

### `frontend/src/components/dj/DJQuantumAnimation.tsx`

Perubahan utama:

1. tambah konstanta resolusi export:
   - `EXPORT_VIDEO_WIDTH = 1920`
   - `EXPORT_VIDEO_HEIGHT = 1080`
   - `EXPORT_VIDEO_BITRATE = 16_000_000`
2. tambah helper `waitForCanvasReady()` untuk menunggu hidden export canvas siap,
3. ganti source recorder dari preview canvas ke export canvas,
4. preview canvas tidak lagi memakai `preserveDrawingBuffer`,
5. hidden export canvas dirender hanya saat export aktif,
6. tombol export dan keterangan UI sekarang menyebut `1080p` secara eksplisit,
7. kompositor menggambar gradient dasar sebelum menggambar frame dari canvas export.

## Pseudocode Perubahan

```text
user click export video
  stop playback biasa
  set export active
  mount hidden canvas 1920x1080
  wait until hidden canvas width/height ready
  start MediaRecorder from compositor

  each frame:
    fill compositor background gradient
    draw export canvas 1080p
    draw caption overlay

  stop recorder
  download .webm
  unmount hidden export canvas
  restore previous playback state
```

## Kenapa Ini Solusi yang Benar

Perubahan ini memperbaiki akar masalah, bukan gejalanya. Akar masalahnya adalah source video sebelumnya berasal dari preview kecil. Setelah dipisah menjadi renderer preview dan renderer export, video kini punya surface yang memang ditujukan untuk output `1920x1080`. Dengan kata lain, kualitas tajam sekarang berasal dari render source yang benar, bukan dari scaling sesudah fakta.
