# SPEC 005: Remove Hardcoding in DJ Backend and Case Selector

## Prompt

Refactor Deutsch-Jozsa implementation so case discovery, oracle construction, and case selection are fully data-driven. New files like `DJ-05.json`, `DJ-06.json`, or `DJ-10.json` must work without code changes.

## Purpose

Make the DJ flow robust against future dataset growth. The backend must discover cases directly from `datasets/dj/`, the quantum circuit must build the oracle from `truth_table`, and the frontend selector must read available cases from the API instead of a fixed list.

## Why

1. `range(1, 5)` only exposes DJ-01 to DJ-04.
2. Balanced oracle handling was incomplete when the oracle section had no actual gates.
3. Literal step numbers make the trace fragile and hard to extend.
4. A static case list in the UI blocks new JSON cases from being selectable.

## Codebase

**Backend:**
- `backend/api/dj.py`

**Frontend:**
- `frontend/src/pages/DJCombinedPage.tsx`

## Changes Made

### 1. Dynamic case discovery

`get_all_cases()` now scans `datasets/dj/` and loads every file matching `DJ-*.json`.

Behavior:
- ignores non-JSON files
- ignores generated result files
- sorts cases numerically by case id

### 2. Real oracle construction

`create_dj_circuit()` now builds the oracle from the dataset truth table.

Rules:
- constant zero oracle stays identity
- constant one oracle applies `X` on ancilla
- balanced oracle applies multi-controlled `X` for each input where `f(x)=1`

Barriers remain in the required order:
- beginning
- barrier
- oracle
- barrier
- measure/output

### 3. Counter-based tracing

`build_trace_from_truth_table()` now uses an incrementing counter for step numbers.

Current trace flow:
1. Inisialisasi awal
2. X pada ancilla
3. H pada input register
4. H pada ancilla
5. Oracle
6. H kedua pada input register
7. Measurement

### 4. Dynamic frontend selector

`DJCombinedPage.tsx` now loads available cases from `GET /api/dj/cases` on mount.

Effects:
- the select options follow the dataset folder
- adding `DJ-05.json` immediately exposes `DJ-05` in the UI
- default selection stays valid even when case order changes

### 5. Safety fixes

- `run_quantum_dj()` now checks the measured zero state with a set comparison.
- `run_classic_brute_force_stepped()` now uses safe table lookups for missing keys.
- `derive_trace_from_circuit()` was removed because it was unused and relied on hardcoded gate indices.

## Logic / Pseudocode

### Case discovery

```text
scan datasets/dj/
for each file:
  if filename matches DJ-*.json:
    load JSON
sort cases by numeric id
return cases
```

### Oracle building

```text
profile truth_table
if constant zero:
  keep identity oracle
if constant one:
  apply X on ancilla
if balanced:
  for each input where f(x)=1:
    flip zero-bit controls
    apply MCX to ancilla
    undo flips
```

### Dynamic selector

```text
on page mount:
  fetch /api/dj/cases
  map response to case ids
  sort ids numerically
  populate select options
  keep current selection if still valid
```
