# SPEC 005: Remove Hardcoding — Dynamic DJ Case Discovery & Balanced Oracle

## 壹　Prompt

Refactor `backend/api/dj.py` to remove ALL hardcoding:
- `get_all_cases()` must auto-discover DJ-XX.json files from directory scan
- Step numbers must use incrementing counter, NOT literal numbers
- Balanced oracle must be built from truth_table using multi-controlled X gates
- Must work for DJ-05 through DJ-99 without code changes

## 貳　Purpose

Make DJ backend fully data-driven. Add DJ-05.json → code handles it automatically. No `range(1,5)`, no magic numbers.

## 叁　Why

1. **Case discovery**: `range(1,5)` + `f'DJ-0{i}'` — DJ-05, DJ-10 invisible
2. **Step literals**: `'step': 1, 2, 3...` hardcoded — user said "nggak pake counter. tapi pake magic number"
3. **Balanced oracle**: `pass` for balanced — no gates built, circuit WRONG for balanced cases
4. **Dead code**: `derive_trace_from_circuit()` has `if i == 0/1/2/3...` hardcoded

## 伍　Changes

### 1. `get_all_cases()` — Dynamic Directory Scan

**Before:**
```python
def get_all_cases():
    cases = []
    for i in range(1, 5):        # MAGIC NUMBER 5
        case_id = f'DJ-0{i}'     # Assumes 2-digit zero-padded
        case = load_case(case_id)
        if case:
            cases.append(case)
    return cases
```

**After:**
```python
def get_all_cases():
    cases = []
    if not os.path.exists(DATASETS_DIR):
        return cases
    for fname in sorted(os.listdir(DATASETS_DIR)):
        if not (fname.startswith('DJ-') and fname.endswith('.json')):
            continue
        case_id = fname[:-5]
        case = load_case(case_id)
        if case:
            cases.append(case)
    def case_sort_key(c):
        try:
            return int(c['case_id'].split('-')[1])
        except:
            return 0
    cases.sort(key=case_sort_key)
    return cases
```

## 陸　Logic / Pseudocode

### Dynamic Case Discovery
```
scan DATASETS_DIR for all *.json files
filter filenames starting with 'DJ-' and ending with '.json'
strip .json extension → case_id
load_case(case_id) → add to list if valid
sort by numeric part of case_id (DJ-1 vs DJ-10)
return sorted list
```

### Balanced Oracle Construction
```
num_ones = sum(truth_table.values())
total = len(truth_table.values())
is_balanced = (num_ones > 0) and (num_ones < total)

if not is_balanced:
    if num_ones == 0: pass  # identity
    else: qc.x(qr_anc[0])  # f(x)=1 constant
else:
    for each (input_str, output_val) in truth_table:
        if output_val == 1:
            # input_str e.g. "010" for n=3
            zero_indices = [i for i, bit in enumerate(input_str) if bit == '0']
            for i in zero_indices: qc.x(qr_in[i])
            qc.mcx(list(qr_in), qr_anc[0])
            for i in zero_indices: qc.x(qr_in[i])
```

### Step Counter
```
step = 0
for each stage definition:
    step += 1
    append {'step': step, ...}
```

## 玖　Dataset Schema

```json
{
  "case_id": "DJ-01",
  "n_qubits": 3,
  "expected_classification": "CONSTANT",
  "oracle_definition": {
    "truth_table": {
      "000": 0,
      "001": 0
    }
  }
}
```

- `case_id`: string, format `DJ-XX` (X can be 1-99, zero-padded optional)
- `n_qubits`: integer, 1-10
- `expected_classification`: "CONSTANT" or "BALANCED"
- `oracle_definition.truth_table`: object mapping binary string → 0 or 1
