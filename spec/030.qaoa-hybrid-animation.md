# SPEC 030: QAOA Hybrid Animation 3D

## Prompt

Implementasikan animasi 3D untuk QAOA yang mengikuti pola animasi QFT, tetapi tetap benar secara konsep karena QAOA adalah algoritma hybrid quantum-classical. Animasi harus dinamis dari dataset `datasets/qaoa/*.json`, menampilkan graph Max-Cut, Hamiltonian Ising, loop optimizer klasik, cost layer, mixer layer, measurement, dan update objective.

## Goal

User non-teknis harus langsung paham:

1. graph input berasal langsung dari dataset,
2. graph dipetakan ke Hamiltonian Ising untuk Max-Cut,
3. QAOA bukan alur satu arah seperti QFT, tetapi loop hybrid,
4. parameter `gamma` dan `beta` berubah antar iterasi optimizer,
5. cost layer bekerja per edge,
6. mixer layer bekerja per qubit,
7. measurement menghasilkan distribusi bitstring dan nilai cut,
8. hasil akhirnya dibandingkan dengan Exact dan Simulated Annealing.

## Proposal Alignment

Implementasi harus selaras dengan `proposal.md`:

1. QAOA diposisikan sebagai algoritma hybrid untuk optimisasi kombinatorial.
2. Studi kasus adalah Max-Cut pada graph 3 dan 4 node.
3. Formulasi masalah ditampilkan sebagai Hamiltonian Ising.
4. Jumlah shots tetap 1024.
5. Metrik yang tampak harus relevan dengan proposal: circuit depth, gate count, expected cut, best cut, dan approximation ratio.
6. Pembanding klasik utama yang dipakai adalah Simulated Annealing, sedangkan brute force ditampilkan sebagai optimum referensi.

   - `_counts_to_cut_distribution()`
   - `_evaluate_qaoa_checkpoint()`
4. Builder checkpoint hybrid:
   - `_build_qaoa_animation_checkpoints()`
5. Builder timeline animasi:
   - `_build_qaoa_checkpoint_timeline()`
6. Endpoint payload builder:
   - `get_qaoa_animation_payload(case_id, shots=1024)`

### Timeline Hybrid

Timeline dibentuk dari checkpoint optimizer nyata:

1. `initial`
2. `middle`
3. `best`

Setiap checkpoint memuat urutan:

1. `optimizer`
2. `superposition`
3. `cost` per edge
4. `mixer` per qubit
5. `measurement`
6. `update`

Jumlah langkah `cost` dan `mixer` sepenuhnya dinamis:

- `cost = edge_count * p_layers`
- `mixer = n_nodes * p_layers`

Perbaikan:

## Visual Design

### Track A: 3D Scene

Canvas dibagi menjadi tiga zona:

1. **Graph Input**
   - node dan edge asli dari dataset
   - edge aktif menyala saat `cost` phase
2. **Quantum Circuit**
   - wire horizontal `q0...q{n-1}`
   - gate visual untuk `H`, `ZZ`, `Rx`, `M`
   - orb qubit bergerak mengikuti phase aktif
3. **Cut Output**
   - graph hasil partisi dari bitstring aktif
   - warna node mengikuti hasil partition
