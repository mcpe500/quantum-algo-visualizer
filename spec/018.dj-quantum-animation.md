# SPEC 018: DJ Quantum Animation 3D — Circuit-Diagram Style

## Prompt

Perbaiki total visualisasi DJ 3D: hilangkan overlap, tampilkan circuit-diagram style layout dengan qubit orbs yang bergerak sepanjang wire, gate Python dari backend trace jelas terlihat, compact 1 layar.

## Goal

User non-teknis harus langsung paham:

1. mana input (wire kiri),
2. mana proses superposition (H gates),
3. mana oracle (U_f box),
4. mana output (measurement),
5. gate Python yang sedang aktif (phase stepper),
6. token 0/1 itu apa (optional, default off).

## Architecture

### Track A: Circuit-Diagram 3D Scene

Canvas 3D menampilkan **physical circuit layout**:

- **Horizontal wires** (`q0..anc`) dari kiri ke kanan
- **Gate landmarks** di posisi tetap sepanjang wire:
  - `X` gate pada ancilla wire (x=-8)
  - `H` gates round 1 pada semua wire (x=-4)
  - `U_f` Oracle box spanning semua wire (x=0)
  - `H` gates round 2 pada input wire (x=4)
  - `M` measurement pada input wire (x=8)
- **Qubit orbs** bergerak smoothly sepanjang wire:
  - X position = `computeOrbX(step, snapshots)` — interpolasi dalam phase range
  - State ditentukan dari real Qiskit `probabilities` via `getQubitP1()`
  - Superposition visual: wireframe + opacity turun + glow + bobbing
  - Ring color = phase color
- **Gate highlight**: gates brighten saat orb pass through
- **Result board**: muncul di measurement position saat step terakhir
- **Token oracle**: optional ribbon, follows orbX position

#### Orb Position Logic (`computeOrbX`)

```
PHASE_X_RANGE:
  init:         [-10, -6]
  prep:         [-6, -2]
  oracle:       [-2, 2]
  interference: [2, 6]
  measure:      [6, 10]

Within phase: interpolate linearly based on step-within-phase / total-steps-in-phase
Last step: orbX = GATE_X.measure (8)
```

#### Superposition Visual

- `pOne < 0.15` → state `0` (blue, solid)
- `pOne > 0.85` → state `1` (orange, solid)
- Otherwise → superposition `0|1` (violet, wireframe, opacity 0.65, glow, bobbing)

### Track B: Compact Phase Stepper (HTML)

Di atas canvas, tampil gate yang berasal dari Python trace:

- **Phase stepper**: clickable badges, active highlighted
- **Active stage detail**: operation + wire markers inline
- Total height: ~60px

### Data Sources

- `GET /api/dj/animation/<case_id>?shots=1024` → `DJAnimationPayload` (snapshots, measurement, truth_table)
- `GET /api/dj/trace/<case_id>` → `DJQuantumTrace` (stages with wire_markers)
- Dataset JSON files: `datasets/dj/DJ-01.json` through `DJ-04.json`

## Component Design

### `DJQuantumAnimation.tsx`

#### 3D Components:
1. `CircuitLayout` — wires + gate landmarks + labels (replaces ZonePanels + LaneGrid)
2. `GateBox` — single gate landmark (transparent box + wireframe + label)
3. `OracleGateBox` — tall oracle box spanning all wires with rotating core
4. `MeasureBox` — measurement indicator on input wires
5. `QubitOrbNode` — animated sphere with wireframe/opacity for superposition + ring + glow
6. `OracleTokenRibbon` + `TokenCard3D` — optional tokens following orbX
7. `ResultBoard` — classification result at measurement position
8. `CameraRig` — fixed/orbit camera modes
9. `StoryScene` — orchestrator, computes orbX + qubit states

#### HTML Components:
1. `CompactGateTimeline` — phase stepper + active stage detail
2. `MarkerBadge` — wire marker badges

#### Helpers:
- `computeOrbX()` — maps step index → X position on circuit
- `getLaneYs()` — wire Y positions
- `getQubitP1()` — probability of |1⟩ from statevector
- `groupStagesByPhase()` — groups trace stages
