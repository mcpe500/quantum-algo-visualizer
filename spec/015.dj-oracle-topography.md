# SPEC 015: DJ Oracle Topography Visualization

## 壹　Task

Create "Quantum Oracle Topography" visualization — a page displaying all 4 Deutsch-Jozsa oracle cases (DJ-01 through DJ-04) as truth table grids in an elegant card layout. Based on user's React component example with Tailwind styling, lucide-react icons (Compass/Scale), and mount animations.

## 貳　Prompt Summary

User provided exact React component example and asked to "buatkan di quantum-algo-visualizer/" — integrate into existing project. The example shows:
- Header: "Quantum Oracle Topography" with Binary icon
- 4 cards in responsive grid (1 col mobile, 2 col lg, 4 col 2xl)
- Each card: case_id header, n_qubits label, Compass (CONSTANT) or Scale (BALANCED) icon, 4-column truth table grid, classification footer
- Mount animation with staggered delays

## 叁　Why

Current DJCombinedPage is a complex benchmark runner. This new "Oracle Topography" page serves as a **dataset overview** — lets users quickly see all oracle patterns at a glance before diving into detailed benchmarking. Useful for thesis presentation/demo.

## 肆　Codebase Understanding

**Stack:** React 19 + Vite + Tailwind + lucide-react + react-router-dom

**Existing patterns:**
- Components live in `frontend/src/components/{dj,qft,vqe,qaoa}/`
- Pages in `frontend/src/pages/`
- Routing in `frontend/src/App.tsx`
- API calls via `frontend/src/services/api.ts` → `djApi.getCases()`, `djApi.getDataset(caseId)`
- Types defined in `frontend/src/types/dj.ts`

**Data structure:**
```typescript
interface DJCase {
  case_id: string;          // "DJ-01"
  n_qubits: number;          // 3 or 4
  expected_classification: 'CONSTANT' | 'BALANCED';
  oracle_definition: {
    truth_table: Record<string, number>;  // e.g. {"000": 0, "001": 1, ...}
  };
}
```

## 捌　Visual Design (from user's example)

- **Background:** `bg-blue-50/50`
- **Header:** centered, Binary icon + "Quantum Oracle Topography" title
- **Card grid:** `grid-cols-1 lg:grid-cols-2 2xl:grid-cols-4 gap-8`
- **Card:** white bg, rounded-2xl, border-slate-200, shadow on hover
- **Truth table cells:**
  - State 1 (output=1): `bg-blue-500 border-blue-500 text-white`
  - State 0 (output=0): `bg-white border-slate-200`
- **Footer classification:** blue dot + classification text
- **Animation:** translate-y-12 → 0, opacity 0 → 1, transition-delay 100ms * index

## 玖　Edge Cases

- Empty cases array → show "No cases available" message
- Missing truth_table entries → render only available entries
- API fetch failure → show error state with retry button
