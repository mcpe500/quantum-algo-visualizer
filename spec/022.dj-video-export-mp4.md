# SPEC 022: DJ Video Export MP4 via FFmpeg.wasm

## Prompt

User meminta fitur export video dalam format MP4, bukan hanya WebM. Browser tidak bisa langsung merekam MP4 via MediaRecorder API, sehingga diperlukan konversi dari WebM ke MP4 di sisi client menggunakan FFmpeg.wasm.

## Tujuan

Menambahkan opsi export MP4 pada animasi Deutsch-Jozsa, dengan dua tombol terpisah:
1. **WebM** ‚Äî instant, langsung download hasil rekaman MediaRecorder
2. **MP4** ‚Äî rekam WebM lalu konversi menggunakan FFmpeg.wasm di browser

## Kenapa Perlu Diubah

Export sebelumnya hanya menghasilkan WebM. Format MP4 lebih universal dan kompatibel dengan lebih banyak player serta platform presentasi. FFmpeg.wasm dipilih karena self-contained di client-side tanpa perubahan backend.

## Keputusan Teknis

### 1. FFmpeg.wasm untuk konversi client-side

Library `@ffmpeg/ffmpeg` dan `@ffmpeg/util` dimuat secara lazy ‚Äî hanya download ~25MB WASM ketika user pertama kali klik export MP4. Setelah itu instance di-cache untuk pemakaian berikutnya.

### 2. COOP/COEP headers di Vite

FFmpeg.wasm memerlukan `SharedArrayBuffer` yang hanya tersedia jika server mengirim header:
- `Cross-Origin-Opener-Policy: same-origin`
- `Cross-Origin-Embedder-Policy: require-corp`

Header ini ditambahkan di `vite.config.ts` untuk dev server dan harus juga dikonfigurasi di production server.

### 3. Dua tombol terpisah

- Tombol WebM (violet): instant, sudah ada sebelumnya
- Tombol MP4 (emerald): rekam + konversi, tambahan baru

### 4. Error handling

Jika FFmpeg.wasm gagal load atau konversi gagal, ditampilkan pesan error. Tidak ada fallback ke WebM.

## Perubahan Code

### `frontend/src/utils/videoConvert.ts`

Modul baru yang berisi:
- `loadFFmpeg()` ‚Äî lazy-load FFmpeg instance, cache setelah load pertama
- `convertWebmToMp4(webmBlob)` ‚Äî konversi WebM ke MP4 menggunakan ffmpeg CLI:
  - `ffmpeg -i input.webm -c:v libx264 -preset fast -crf 23 -pix_fmt yuv420p -movflags +faststart output.mp4`
- `isFFmpegSupported()` ‚Äî cek apakah `SharedArrayBuffer` tersedia di browser

### `frontend/vite.config.ts`

Tambah konfigurasi:
- `optimizeDeps.exclude` ‚Äî exclude FFmpeg dari Vite pre-bundling
- `server.headers` ‚Äî COOP/COEP untuk SharedArrayBuffer

### `frontend/src/components/dj/DJQuantumAnimation.tsx`

Perubahan:
1. Import `convertWebmToMp4` dan `isFFmpegSupported`
2. Tambah state `isConverting` dan computed `ffmpegReady`
3. Tambah `handleExportMp4()` ‚Äî mirip `handleExportVideo` tapi setelah WebM selesai, lakukan konversi ke MP4 lalu download
4. Tambah tombol kedua (emerald) "MP4 1080p" di toolbar
5. Tambah pesan warning jika SharedArrayBuffer tidak tersedia
6. Perbaiki preview canvas ‚Äî gunakan `cameraMode` user (bukan locked fixed)
7. Update PhaseStepper disabled state termasuk `isConverting`

## Pseudocode

```text
user klik "MP4 1080p"
  cek MediaRecorder support ‚Üí gagal? error
  cek SharedArrayBuffer ‚Üí gagal? error
  lock controls, set isExporting

  setup compositor canvas 1920x1080
  setup MediaRecorder dari compositor stream
  mount hidden export renderer

  draw intro overlay
  start recorder
  play through all steps
  draw outro overlay
  stop recorder ‚Üí dapat WebM blob

  set isConverting = true
  load FFmpeg.wasm (lazy, ~25MB first time)
  write input.webm ke virtual FS
  exec ffmpeg convert ‚Üí output.mp4
  read output.mp4 dari virtual FS
  cleanup virtual FS
  download .mp4 file

  unlock controls
  restore previous state
```

## Risiko

1. FFmpeg.wasm core di-load dari unpkg CDN ‚Äî jika CDN down, MP4 export gagal
2. ~25MB WASM download pertama kali ‚Äî bisa lambat di koneksi lambat
3. SharedArrayBuffer tidak tersedia di semua environment (misal beberapa sandbox iframe)
4. Konversi bisa 5-15 detik untuk video 1080p di mesin lambat
